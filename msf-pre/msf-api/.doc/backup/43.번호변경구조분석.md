예약 가능한 전화번호 조회 구조 분석
1. 전체 흐름 요약
[화면] 번호예약 영역 (뒷 4자리 입력) 
   → [JS] searchNumber() 
   → [WAS] /appform/searchNumberAjax.do (searchNumber)
   → [서비스] getPhoneNoList(resNo, "NU1") 
   → [OSST 연동] inqrOsstSvcNoInfo (전화번호 조회)
   → [응답] 예약 가능 번호 리스트
   → [화면] 모달(searchNumDialog)에 라디오 목록 표시
   → 사용자 선택 후 [JS] setNumber → /appform/setNumberAjax.do (번호 예약 확정)
2. 프론트엔드 (Portal Web)
2-1. 진입점·UI
파일: mcp-portal-web/src/resources/js/portal/appForm/appForm.js
버튼: #btnSearchNumber (action="RSV"일 때 조회)
입력: #reqWantNumberN — 희망 번호 뒷 4자리 (010-****-XXXX)
검증: validator.config['reqWantNumberN'] = 'isNumFix4' (4자리 숫자)
2-2. 조회 함수: searchNumber()
URL: POST /appform/searchNumberAjax.do
전송 데이터:
reqWantNumber: #reqWantNumberN 값 (뒷 4자리)
requestKey: pageObj.requestKey
성공 시:
jsonObj.RESULT_OBJ_LIST: 번호 목록
jsonObj.SEARCH_CNT: 현재까지 조회 횟수
#searchCnt: "조회 가능 횟수 N회" (20 - SEARCH_CNT)
#searchNumDialog ._list에 라디오 버튼 목록 생성 후 모달 오픈
번호 한 건 데이터 (라디오/라벨용):
tlphNo, tlphNoView(표시용), tlphNoStatCd, tlphNoOwnCmncCmpnCd, encdTlphNo
에러 처리:
0001: 선택 가능한 번호 없음
0004: 조회 가능 횟수 초과 → 신청서 다시 작성 유도
기타: "번호 조회에 실패하였습니다."
2-3. 번호 확정: #btnConfirmNum 클릭
URL: POST /appform/setNumberAjax.do
전송: 선택한 라디오의 tlphNo, tlphNoStatCd, tlphNoOwnCmncCmpnCd, encdTlphNo
성공 시 화면에 선택 번호 표시, 버튼 문구를 "번호취소"로 변경
2-4. JSP/모달
Portal: mcp-portal-was/.../views/portal/appForm/appForm.jsp
번호예약: #inpReservNumHead, #reqWantNumberN, #btnSearchNumber, #searchCnt
모달: #searchNumDialog, #searchNumTitle("예약가능 번호 리스트"), ._list, #btnConfirmNum
Mobile: .../mobile/appForm/appForm.jsp 에도 동일 ID/역할로 구성
3. 백엔드 (Portal WAS)
3-1. 컨트롤러: 전화번호 조회
파일: mcp-portal-was/.../appform/controller/AppformController.java
매핑: @RequestMapping("/appform/searchNumberAjax.do"), searchNumber(McpRequestDto mcpRequestDto)
처리 순서:

세션/환경 분기

LOCAL: 세션 조회 횟수 19회 초과 시 예외; resNo = "3860" 고정 후 OSST 조회 횟수(requestOsstCount) 19회 초과 시 예외.
STG: 동일하게 resNo = "3860", 조회 횟수 19회 제한.
운영: SessionUtils.getAppformSession()에서 sessAppformReqDto 조회, null이면 예외. 같은 예약에 대해 OSST 조회 횟수 24회 초과 시 예외.
희망번호 저장 (운영만)

mcpRequestDto.setRequestKey(sessAppformReqDto.getRequestKey())
appformSvc.updateMcpRequest(mcpRequestDto) → DB에 reqWantNumber 등 반영
번호 조회

appformSvc.getPhoneNoList(sessAppformReqDto.getResNo(), EVENT_CODE_SEARCH_NUMBER)
EVENT_CODE_SEARCH_NUMBER = "NU1"
응답

성공·목록 있음: RESULT_CODE=성공, RESULT_OBJ_LIST, SEARCH_CNT
목록 없음: RESULT_CODE=0001
OSST 실패: RESULT_CODE=0002, RESULT_XML, ERROR_MSG
기타: 9997(응답 null), 9999(SocketTimeout), 9998(SelfServiceException)
3-2. 서비스·연동
인터페이스: AppformSvc.getPhoneNoList(String resNo, String eventCd)
구현: AppformSvcImpl.getPhoneNoList
param: appEventCd = "NU1", resNo
mplatFormOsstServerAdapter.callService(param, mPhoneNoListXmlVO, 100000) 호출
OSST 이벤트 코드: Constants.EVENT_CODE_SEARCH_NUMBER = "NU1"
3-3. OSST 어댑터
파일: mcp-portal-was/.../common/mplatform/MplatFormOsstServerAdapter.java
역할:
LOCAL/STG: appEventCd == "NU1"이면 inqrOsstSvcNoInfoResponse 형태의 고정 XML 반환(테스트용 번호 3개).
운영: osstSimpleOpenUrl로 param(appEventCd, resNo 등) 전송 후 받은 XML을 CommonXmlVO 계열로 파싱.
MCP 매퍼: MPlatformMapper.xml에서 appEventCd = 'NU1' → 서비스/오퍼레이션 osst:inqrOsstSvcNoInfo, VO InqrOsstSvcNoInfoInVO 사용.
3-4. 방어 로직 (조회 횟수 제한)
세션: SessionUtils.getCntSession() — 세션 기준 조회 횟수
DB: appformSvc.requestOsstCount(mcpRequestOsstDto)
McpRequestOsstDto: mvnoOrdNo = resNo, prgrStatCd = EVENT_CODE_SEARCH_NUMBER("NU1")
AppformDao.requestOsstCount → AppformMapper.requestOsstCount로 해당 예약의 NU1 건수 조회
LOCAL: 세션 19회 + OSST 19회 초과 시 예외
운영: 세션 19회 + OSST 24회 초과 시 예외
4. 데이터 구조
4-1. 응답 VO: MPhoneNoListXmlVO / MPhoneNoVo
파일:
mcp-portal-was/.../common/mplatform/vo/MPhoneNoListXmlVO.java
MPhoneNoVo.java
파싱: 응답 XML의 outDto → svcNoList 리스트를 MPhoneNoVo 리스트로 매핑.
MPhoneNoVo 주요 필드:
tlphNo: 전화번호
tlphNoView: StringUtil.getDivPhoneNum(tlphNo) (표시용 구분)
encdTlphNo: 암호화 전화번호 (예약 확정 시 사용)
tlphNoStatCd, tlphNoOwnCmncCmpnCd: 상태/소유 통신사
기타: asgnAgncId, fvrtnoAqcsPsblYn, openSvcIndCd, rsrvCustNo 등
4-2. 요청 DTO
조회 요청: McpRequestDto — requestKey, reqWantNumber(뒷 4자리) 등
DB 반영: AppformMapper.updateMcpRequest로 MCP_REQUEST 등 테이블에 희망번호 저장
5. 번호 예약 확정 (setNumber) 흐름
URL: /appform/setNumberAjax.do → setNumber(McpRequestOsstDto)
이벤트: EVENT_CODE_NUMBER_REG = "NU2" (전화번호 예약)
처리: 세션의 AppformReqDto·OsstDto 확인 후 appformSvc.insertMcpRequestOsst → appformSvc.sendOsstService(resNo, "NU2", ...) 로 OSST resvOsstTlphNo 호출하여 해당 번호 예약 확정.
