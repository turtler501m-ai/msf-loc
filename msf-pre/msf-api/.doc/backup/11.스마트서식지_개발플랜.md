## 2. 개발 범위

### 2.1 담당 업무 (3종)

| 신청서 | 화면ID | mform-web 컴포넌트 | 스텝 수 |
|--------|--------|--------------------|---------|
| 서비스변경 | S102xxxxx | change/ | 5단계 (가입유형→고객→상품→동의→신청완료) |
| 명의변경 | S103xxxxx | ident/ | 4단계 (가입유형→고객→상품→동의→신청완료) |
| 서비스해지 | S104xxxxx | cancel/ | 4단계 (가입유형→고객→상품→동의→신청완료) |

### 2.2 기술 스택 (고정)

- **언어**: JavaScript (TypeScript 미사용)
- **프론트**: Vue 3, Vue Router 5, Pinia, Vite 7, Tailwind CSS 4
- **백엔드**: 신규 개발 (연동 API는 M포털과 동일 스펙 구현)

### 4.1 폴더 구조 (추가/변경)

```
mform-web/src/
├── api/                    # (신규) API 호출

│   ├── serviceChange.js   # 서비스변경 API

│   └── cancel.js          # 서비스해지 API
├── stores/
│   ├── service_change_form.js   # (신규) 서비스변경 폼 상태
│   ├── ident_form.js            # (신규) 명의변경 폼 상태
│   └── cancel_form.js           # (신규) 서비스해지 폼 상태
├── composables/            # (선택) useServiceChange 등
├── constants/              # (선택) ServiceChangeCode, MutuallyExclusiveGroup
└── components/
    ├── change/
    │   ├── ChangeType.vue    # S102010101
    │   ├── ChangeCust.vue    # S102020101
    │   ├── ChangeProd.vue    # S102030101
    │   └── ChangeAgree.vue   # S102040101
    ├── ident/
    ├── cancel/
    └── commons/              # 우편번호, 약관 팝업 등
```

### 4.2 스텝 플로우 (McpStepService)

- `mcp_menu.js`에 이미 서비스변경: 가입유형→고객→상품→동의 정의됨
- 신청완료는 별도 뷰 또는 ChangeAgree 내 완료 영역으로 처리
- `mcp_menu_step.js`와 연동하여 `stepSubMenuUrls`로 현재 스텝 추적

// api/serviceChange.js 예시
import { mformApi } from './mform'
