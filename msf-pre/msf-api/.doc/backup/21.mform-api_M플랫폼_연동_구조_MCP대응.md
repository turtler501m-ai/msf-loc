# mform-api M플랫폼 연동 구조 (MCP 대응)

**목적**: 기존 M포털(MCP) 운영자가 mform-api 유지보수 시 동일한 개념·설정키·클래스명으로 이해할 수 있도록 구조를 맞춤.

---

## 1. 설정키 (MCP 와 동일)

| 설정키 | MCP (mcp-portal-was) | mform-api | 비고 |
|--------|----------------------|-----------|------|
| **juice.url** | MplatFormServerAdapter 호출 URL (msc-prx) | 동일 | 미설정 시 X20 은 Mock |
| **SERVER_NAME** | LOCAL 이면 getVo() Mock, 아니면 실제 호출 | 동일 | LOCAL / DEV / STG / 운영 등 |
| **mplatform.user-id** | (MCP 는 세션 sesUserId()) | getParamMap 용 userId | 선택 |

---

## 3. 호출 흐름 (X20 기준, MCP 와 동일)

```
[Controller] additionCurrent(req)
    → MplatFormService.getAddSvcInfoDto(ncn, ctn, custId)
        → getParamMap(ncn, ctn, custId, "X20")   // ncn, ctn, custId, userId, appEventCd
        → SERVER_NAME == "LOCAL" ?
            → getVo(20, vo)   // Mock XML 세팅 후 vo.toResponseParse()
            : mplatFormServerAdapter.callService(param, vo)
                → saveMplateSvcLog(param)   // ip, url, mdlInd 보강(선택)
                → getURL(param)   // key=value&... UTF-8 인코딩
                → POST juice.url (form: getURL=...)
                → vo.setResponseXml(responseXml); vo.toResponseParse()
    → vo.getList() (List<MpSocVO>) → API 응답으로 변환
```

---

## 4. 차이점 (REST 환경 반영)

| 항목 | MCP | mform-api |
|------|-----|------------|
| HTTP 클라이언트 | HttpClientUtil.post (NameValuePair) | RestTemplate (form-urlencoded) |
| 세션 userId | SessionUtils.getUserCookieBean() | mplatform.user-id 설정값 |
| saveMplateSvcLog | HttpServletRequest 에서 ip, url, mdlInd | mplatform.log.ip / log.url / log.mdlInd (선택) |
| XML 파싱 | JDOM (XmlParse, Element) | javax.xml (DocumentBuilder) + XmlParseUtil |
| getVo Mock | MplatFormService 내 getVo(20) | 동일 (동일 Mock XML 조각 사용) |

---

## 5. 운영 시 참고

- **로컬/개발**: `SERVER_NAME=LOCAL` 로 두면 juice.url 없이 X20 Mock 응답.
- **운영 연동**: `juice.url=http://.../mPlatform/serviceCall.do`, `SERVER_NAME` 은 환경값(비 LOCAL)으로 설정.
- **트러블슈팅**: MCP 에서 X20 이 실패할 때 보는 것과 동일하게, juice.url 로 가는 요청·응답 로그와 vo.isSuccess(), resultCode 를 보면 됨.
