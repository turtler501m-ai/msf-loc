# 서비스해지 ASIS 분석

**참조**: [`10.서식지프로젝트.md`](./10.서식지프로젝트.md), [`60.서비스해지신청서_UI설계서_분석.md`](./60.서비스해지신청서_UI설계서_분석.md), [`15.개발기능목록_ASIS_TOBE_기능분석명세서.md`](./15.개발기능목록_ASIS_TOBE_기능분석명세서.md)

**목적**: 서비스해지 신청서 TO-BE 개발 시 참조할 ASIS M포탈·M플랫폼 소스 정리.

---

## 1. ASIS 관련 소스 위치

### 1.1 해지 상담 (Cancel Consult)

| 구분 | 경로 | 설명 |
|------|------|------|
| 프론트 | mcp-portal-web/src/resources/js/portal/mypage/cancelConsult.js | 해지 상담 화면: 해지신청 번호 입력, 본인인증, cancelConsultAjax.do |
| 백엔드 | mcp-portal-was/.../mypage/controller/CancelConsultController.java | 해지 상담 컨트롤러 |
| 백엔드 | mcp-portal-was/.../mypage/service/CancelConsultSvcImpl.java | 해지 상담 서비스 |
| DTO | mcp-portal-was/.../mypage/dto/CancelConsultDto.java | cancelMobileNo(해지신청 번호) 등 |

- **용도**: 해지 **상담** 신청(해지 신청서와 별도 플로우). TO-BE 서비스해지 **신청서**는 대리점 스마트서식지 플로우(가입유형→고객→상품→동의→완료)로 구현.

### 1.2 신청서 공통 (Appform)

| 구분 | 경로 | 설명 |
|------|------|------|
| 컨트롤러 | mcp-portal-was/.../appform/controller/AppformController.java | appForm.do 메인, saveAppformDb() 서식지 저장 |
| 서비스 | mcp-portal-was/.../appform/service/AppformSvcImpl.java | 서식지 저장·이미징·M플랫폼 연동 |
| DAO | mcp-portal-was/.../appform/dao/AppformDaoImpl.java | marketRequest 등 API 연동 |

- **marketRequest.do**: 오픈마켓 신청용 진입점(서비스해지 신청서와는 별도).
- 서비스해지 **신청서** 저장·전송은 appForm 공통 저장 흐름(saveAppformDb, 리포트 이미징, M플랫폼 전송) 참조.

### 1.3 M플랫폼 연동

| 구분 | 설명 |
|------|------|
| Y04 | 계약정보 유효성체크 — 휴대폰 인증(이름+ctn → ncn, custId 조회 후 호출). 서비스해지 고객 단계 동일 사용. |
| X01 | 가입정보조회 — 고객 정보·계약 정보. join-info 응답 연계. |
| X22/X67 | 잔여 요금·위약금·분납금 조회(IT부서 확인 후 연동). TO-BE cancel/remain-charge에서 Mock → 실연동 예정. |

## 2. TO-BE 매핑 요약

**백엔드**: `msf/formSvcCncl` (서비스해지 전용). join-info/remain-charge는 formSvcChg.SvcChgInfoSvc 재사용.

| 기능 | ASIS | TO-BE (mform-web / mform-api) |
|------|------|--------------------------------|
| 가입유형 선택 | (대리점 선택) | **20260310**: CancelCust.vue **하단** 섹션, cancel_form.agencyCode |
| 고객 정보·휴대폰 인증 | Y04·M전산 조회 | CancelCust.vue → POST /v1/cancel/join-info (formSvcCncl.CancelRestController) |
| 해지 가능 여부 | (해지 상담 시 검증) | POST /v1/cancel/eligible (Mock) |
| 잔여 요금 조회 | X22/X67 예정 | CancelProd.vue → POST /v1/cancel/remain-charge |
| 신청서 등록 | appForm saveAppformDb·이미징·M전산 | CancelAgree.vue → submitCancelApply() → POST /v1/cancel/apply (formSvcCncl.CancelApplySvc) |
| 신청완료 | 완료 화면 | ServiceCompleteView.vue (domain=cancel), lastApplicationNo 표시 |

---

## 3. 개발 완료 항목 (참고)

- **백엔드**: POST /api/v1/cancel/join-info, /cancel/eligible, /cancel/remain-charge, **/cancel/apply** (CancelApplyReqDto, CancelApplyVO)
- **프론트**: cancel_form.js(lastApplicationNo), cancel.js(submitCancelApply), CancelType/CancelCust/CancelProd/CancelAgree, ServiceCompleteView(접수번호·주의 문구)
- **플로우**: 가입유형 → 고객(인증) → 상품(사용여부·잔여요금) → 동의(서명·작성완료) → 완료(접수번호·열람/톡/SMS/다운로드 안내)
