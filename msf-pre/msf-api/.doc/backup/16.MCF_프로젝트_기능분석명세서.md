# 스마트서식지 MCF — 미구현 TO-BE API 분석

본 문서는 `30.개발기능목록_담당.md`, `15.개발기능목록_ASIS_TOBE_기능분석명세서.md` 와
현재 구현된 `mform-web`, `mform-api` 소스를 기준으로,
**아직 MCF(Tobe)에서 구현되지 않은 기능**에 대해
임시 이름(Controller / Service / 메서드)을 정의하고 구조를 정리한 것이다.

> 실제 개발 시 본 문서를 기준으로 패키지·클래스·메서드명을 통일한다.

## 1. 서비스변경(formSvcChg) 미구현 기능

### 1.1 서비스변경 동시 처리 불가 체크 (C08, REQ_010201)

- **기능**: 번호변경 / 분실복구·일시정지해제 / 요금상품 변경 / 기변 등
  동시 처리 불가 조합을 사전 체크.
- **현황**: 기능목록에는 `SvcChgInfoRestController.concurrentCheck()` 언급, 실제 클래스 미구현.

#### 제안 구조

- **패키지**
  - `com.ktmmobile.msf.formSvcChg.controller`
  - `com.ktmmobile.msf.formSvcChg.service`
  - `com.ktmmobile.msf.formSvcChg.dto`
  - `com.ktmmobile.msf.formSvcChg.mapper`

- **Controller**
  - `SvcChgInfoRestController`
    - `POST /api/v1/service-change/concurrent-check`
      - 메서드명: `concurrentCheck(@RequestBody SvcChgConcurrentCheckReqDto req)`
      - 설명: 선택된 서비스변경 대상 업무코드(SVC_TGT_CD) 목록을 받아,
        동시 처리 불가 조합이 있는지 체크하고 에러코드를 반환.

- **Service**
  - 인터페이스: `SvcChgInfoSvc`
    - `SvcChgConcurrentCheckResDto concurrentCheck(SvcChgConcurrentCheckReqDto req)`
  - 구현체: `SvcChgInfoSvcImpl`

- **DTO**
  - `SvcChgConcurrentCheckReqDto`
    - `String ncn`
    - `String ctn`
    - `String custId`
    - `List<String> svcTgtCdList`  (선택한 서비스변경 업무코드)
  - `SvcChgConcurrentCheckResDto`
    - `String resultCode`  (SUCCESS / ERROR)
    - `String resultMsg`
    - `List<String> conflictCodes` (충돌나는 업무코드 목록)

- **Mapper (선택사항)**
  - `SvcChgInfoMapper`
    - 동시변경 불가 조합을 코드테이블(MSF_CD_DTL 등)에 정의하는 경우,
      해당 목록을 조회하는 쿼리를 제공.

### 1.2 번호변경 (REQ_010206 — C24~C26)

- **기능**:
  - 신규 희망번호 검색(NU2)
  - 조회 가능 횟수 카운팅/증가
  - 번호변경 처리(X32)
- **현황**: 신규/변경 신청서(regist 도메인)에서만 사용.
  서비스변경 신청서 영역에서 별도 API 없음.

#### 제안 구조

- **패키지**
  - `com.ktmmobile.msf.formSvcChg.controller`
  - `com.ktmmobile.msf.formSvcChg.service`
  - `com.ktmmobile.msf.formSvcChg.dto`

- **Controller**
  - `SvcChgNumberController`
    - `POST /api/v1/service-change/number/search`
      - 메서드: `searchNewNumber(@RequestBody NumberSearchReqDto req)`
      - 설명: NU2 이벤트 기반 신규 희망번호 목록 및 현재 조회 횟수 반환.
    - `POST /api/v1/service-change/number/count-increase`
      - 메서드: `increaseSearchCount(@RequestBody NumberSearchReqDto req)`
      - 설명: 세션/계약 기준으로 조회 횟수 증가 처리.
    - `POST /api/v1/service-change/number/change`
      - 메서드: `changeNumber(@RequestBody NumberChangeReqDto req)`
      - 설명: X32 번호변경 실행.

- **Service**
  - 인터페이스: `SvcChgNumberSvc`
    - `NumberSearchResDto searchNewNumber(NumberSearchReqDto req)`
    - `NumberSearchResDto increaseSearchCount(NumberSearchReqDto req)`
    - `NumberChangeResDto changeNumber(NumberChangeReqDto req)`
  - 구현체: `SvcChgNumberSvcImpl`

- **DTO (예시)**
  - `NumberSearchReqDto`
    - `String ncn`, `String ctn`, `String custId`
    - 검색 조건(선호 국번, 패턴 등)
  - `NumberSearchResDto`
    - `List<String> numberList`
    - `int searchCount`
  - `NumberChangeReqDto`
    - `String ncn`, `String ctn`, `String custId`
    - `String newCtn`
  - `NumberChangeResDto`
    - `String resultCode`, `String resultMsg`

- **MplatForm 연동 (MplatFormSvc 내 추가 메서드)**
  - `List<String> requestPhoneNoList(String resNo, String eventCd)`
    - NU2 신규번호 목록 조회.
  - `boolean numChgeChg(String ncn, String ctn, String custId, String newCtn)`
    - X32 번호변경.
