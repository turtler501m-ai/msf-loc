# 단말안심 보험 가입신청 구조 분석

## 1. 개요

**단말안심 보험**(휴대폰 안심 서비스)은 셀프개통/서식지 가입 플로우의 **STEP4(부가서비스)** 구간에서 **선택**으로 신청할 수 있는 이동통신단말기보험(DB손해보험) 연동 상품이다.  
가입 시 **보험 상품 선택** → **약관 동의** → **휴대폰 본인인증(NICE)** 을 거친 뒤, 서식지 저장/개통 요청 시 **인증 정보와 동의 플래그**가 함께 저장·검증된다.

---

## 2. 전체 흐름 요약

```
[STEP4] 안심 보험 서비스 영역 (_insrProdForm)
  → [안심보험서비스 신청/변경] 클릭 → insrProdDialog 오픈
  → getInsrProdInfo() → /appform/selectInsrProdListAjax.do (rprsPrdtId, reqBuyType)
  → API: selectInsrProdList → MSP_INTM_INSR_MST / MSP_INTM_INSR_REL 기반 가입 가능 보험 목록 조회
  → 사용자가 보험 선택 후 [확인] → clauseInsrProdFlag 동의 체크 → [휴대폰 인증] 클릭
  → NICE 휴대폰 인증 (popNiceAuth sAuthType=M) → param_r3 = INSR_PROD 로 구분
  → 인증 성공 시 SessionUtils.saveNiceInsrRes() 저장
  → 서식지 저장(saveAppformDb) 또는 개통 요청 시
      - insrProdCd 있으면 getNiceInsrResCookieBean() 확인
      - 본인인증 CI와 안심보험 인증 CI 동일 여부 검증
      - insrAuthInfo = "ReqNo:xxx, ResNo:xxx", clauseInsrProdFlag = "Y" 설정
  → MCP_REQUEST 테이블에 INSR_PROD_CD, CLAUSE_INSR_PROD_FLAG, INSR_AUTH_INFO 저장
  → 스캔 서버 연동 시 CLAUSE_INSR_PROD_FLAG=Y 이면 이동통신 단말기 보험 상품 설명서 페이지(12/14) 메타로 PDF 입력
```

## 4. 백엔드 구조

### 4.1. 보험 상품 목록 조회

#### 4.1.1. Portal WAS

- **URL**: `POST /appform/selectInsrProdListAjax.do`
- **Controller**: `AppformController.selectInsrProdList(IntmInsrRelDTO intmInsrRelDTO)`
- **파라미터**: `IntmInsrRelDTO`  
  - `reqBuyType`: **필수**. `"MM"`(단말 구매), `"UU"`(유심 단독)  
  - `rprsPrdtId`: 대표 상품 ID (단말 구매 시 단말별 가입 가능 보험 필터용)
- **동작**: `apiInterfaceServer + "/appform/selectInsrProdList"` 로 REST 호출 후 `IntmInsrRelDTO[]` 반환.

#### 4.1.2. MCP-API

- **URL**: `POST /appform/selectInsrProdList` (또는 `/appform/insrProdList`)
- **Controller**: `AppformController.selectInsrProdList` / `insrProdList`
- **Mapper**: `AppformMapper.selectInsrProdList(IntmInsrRelDTO)`
- **쿼리**: `AppformMapper.xml` — `MSP_INTM_INSR_MST@DL_MSP`, `MSP_INTM_INSR_REL@DL_MSP`, `NMCP_CD_DTL`(IntmInsrRelNm)
  - `reqBuyType = 'UU'`: `INSR_TYPE_CD = '02'` (유심용)
  - `reqBuyType = 'MM'`: `MSP_INTM_INSR_REL`에서 `RPRS_PRDT_ID = #{rprsPrdtId}`, `APPL_YN = 'Y'` 인 `INSR_PROD_CD`만
  - 정렬: `CMPN_LMT_AMT DESC`
- **반환 컬럼**: `INSR_PROD_CD`, `CMPN_LMT_AMT`, `INSR_ENGG_CNT`, `INSR_PROD_NM`(DTL_CD_NM)

#### 4.1.3. IntmInsrRelDTO (요청/응답)

| 필드 | 설명 |
|------|------|
| insrProdCd | 보험상품코드 |
| rprsPrdtId | 대표 상품(단말) ID |
| insrProdNm | 보험상품명 |
| reqBuyType | MM(단말) / UU(유심) |
| cmpnLmtAmt | 보상한도금액 |
| insrEnggCnt | 보험약정기간 |

### 4.2. 휴대폰 인증 (안심보험 전용)

#### 4.2.1. 프론트

- `#btnAuthInsr` 클릭 시:
  - `validator`: `isAuth`, `clauseInsrProdFlag` 필수.
  - `getTimeInMillisAjax` 호출 후 `openPage('outLink2', '/nice/popNiceAuth.do?sAuthType=M', data)` 로 NICE 휴대폰 인증 팝업.
  - `pageObj.niceType = NICE_TYPE.INSR_PROD` 설정.

#### 4.2.2. NICE 인증 결과 처리

- **NiceCertifyController** (popNiceAuth 결과 수신):
  - 휴대폰 인증(`sAuthType=M`)이고 `param_r3`가 비어 있으면 **자동으로 `sReserved3 = INSR_PROD`** 로 간주.
  - `INSR_PROD`인 경우:
    - 기존 본인인증 세션(또는 DB 로그)의 CI(`seCiInfo`)와 현재 인증 CI(`connInfo`) 비교.
    - **동일하면** `param_r1 = "1"`, `SessionUtils.saveNiceInsrRes(niceResDtoSess)`.
    - 다르면 `param_r1 = "-1"` (동일 명의 휴대폰이 아니면 보험 인증 실패).
  - `insertNiceLogAjax` 등에서 `paramR3 == INSR_PROD`이면 동일하게 `SessionUtils.saveNiceInsrRes(niceResDtoSess)` 호출.

#### 4.2.3. 세션 키

- **Constants**: `INSR_PROD = "InsrProd"`.
- **저장**: `SessionUtils.saveNiceInsrRes(NiceResDto)`.
- **조회**: `SessionUtils.getNiceInsrResCookieBean()` → `NiceResDto` (reqSeq, resSeq, connInfo 등).

---

### 4.3. 서식지 저장 / 개통 시 안심보험 검증 및 저장

#### 4.3.1. saveAppformDb (서식지 저장)

- **위치**: `AppformController.saveAppformDb` 내부.
- **조건**: `appformReqDto.getInsrProdCd()` 가 비어 있지 않을 때만 처리.
- **로직**:
  1. `NiceResDto sessNiceInsrRes = SessionUtils.getNiceInsrResCookieBean()`.
  2. 전화상담이 아닌 경우(`telAdvice != "Y"`):
     - `sessNiceInsrRes == null` 이거나 **본인인증 CI(`sessNiceRes.getConnInfo()`)와 안심보험 인증 CI(`sessNiceInsrRes.getConnInfo()`)가 다르면**  
       이력 저장 후 `McpCommonJsonException("0005", NICE_CERT_EXCEPTION_INSR2)` 발생.
     - 통과 시:
       - `appformReqDto.setClauseInsrProdFlag("Y")`.
       - `appformReqDto.setInsrAuthInfo("ReqNo:" + sessNiceInsrRes.getReqSeq() + ", ResNo:" + sessNiceInsrRes.getResSeq())`.
  3. 전화상담(`telAdvice == "Y"`)이면 동의만 Y, 인증정보는 빈 문자열:  
     `setClauseInsrProdFlag("Y")`, `setInsrAuthInfo("")`.

#### 4.3.2. 개통 요청 시 (OSST 등 호출 직전)

- 동일하게 `insrProdCd`가 있으면 `getNiceInsrResCookieBean()`으로 검증.
- `sessNiceInsrRes == null` 이거나 본인인증 CI와 안심보험 인증 CI가 다르면 `NICE_CERT_EXCEPTION_INSR2` 예외.
- 통과 시 `clauseInsrProdFlag = "Y"`, `insrAuthInfo = "ReqNo:..., ResNo:..."` 설정 후 이후 개통 처리에 사용.

#### 4.3.3. DB 저장

- **테이블**: `MCP_REQUEST`.
- **컬럼**:  
  - `INSR_PROD_CD`: 단말보험코드(선택한 보험 상품 코드).  
  - `CLAUSE_INSR_PROD_FLAG`: 단말보험가입동의(Y/N).  
  - `INSR_AUTH_INFO`: 단말보험인증정보(ReqNo, ResNo 문자열).
- **매퍼**: `AppFormMapper.xml` insert/update 시 위 3개 컬럼 포함.

---

### 4.4. 스캔 서버 연동 (서식지 PDF)

- **ScanSvcImpl**: 서식지 XML 생성 시 `CLAUSE_INSR_PROD_FLAG == "Y"` 이면:
  - `INSR_PROD_CD`가 `PL214L310`, `PL214L312`, `PL214L316`, `PL214L317`, `PL214L319` 중 하나면  
    **이동통신 단말기 보험 상품 설명서 14페이지** 메타로 `getAppFormPageList("14")` 사용.
  - 그 외(예: 유심 중고형 등)면 **12페이지** 메타로 `getAppFormPageList("12")` 사용.
- 해당 메타의 좌표에 맞춰 `INPUT_DATA`에 보험 관련 값들이 채워져 PDF가 생성된다.

## 5. 프론트엔드 상세 (appForm.js)

### 5.1. getInsrProdInfo()

- **URL**: `POST /appform/selectInsrProdListAjax.do`
- **전송**: `rprsPrdtId`, `reqBuyType`.
- **성공 시**: `pageObj.insrProdList`에 목록 저장, `#divInsrProdList`에 라디오 카드 추가, `#insrProdCdTemp`가 있으면 해당 코드 선택 후 확인 트리거.

### 5.2. btnInsrProd 클릭

- `validator.validate()` (isAuth 등) 후 `insrProdList`가 없으면 `getInsrProdInfo()` 호출.
- `#insrProdDialog` 모달 오픈.

### 5.3. 보험 선택 확인 (#btnInsrProdConfirm)

- 선택한 라디오의 `data` → `pageObj.insrProdObj`, `pageObj.insrProdCd` 설정.
- `._insrProdList` 표시, `#insrProdView`에 `getRowTemplateInsrProdView(pageObj.insrProdObj)` 렌더링.
- 보험을 바꾼 경우 `#clauseInsrProdFlag` 체크 해제.

### 5.4. 제출 시

- step4 검증/저장/개통 요청 시 `pageObj.insrProdCd`가 있으면 `validator.config['clauseInsrProdFlag'] = 'isNonEmpty'` 설정.
- `pageObj.applDataForm.insrProdCd` / 전송 param 에 `insrProdCd` 포함.

### 5.5. 상품별 UI (dataFormConfig.js)

- **INSR_PROD_CONTENT**: 상품코드별 카드 HTML(제목, 월 이용료, 설명, 보상범위 등).
- **INSR_PROD_OBJ**: 상품코드별 `{ name, price }`.
- **INSR_PROD_DESC_LIST**, **INSR_PROD_ICO_LIST**: 상품코드별 설명/아이콘 리스트.
- 예시 코드: `PL214L310`(안드로이드 플래티넘), `PL214L312`(안드로이드 프리미엄), `PL214L316`(아이폰 프리미엄), `PL214L317`(아이폰 베이직), `PL214L319`(아이폰 플래티넘).  
- **Constants**: `INSR_PROD_OLD_CD = "USIMOLDFO"` (유심 중고형 코드).

## 7. 정리

- **단말안심 보험 가입신청**은 서식지 STEP4에서 선택 사항으로 동작하며, **전화상담 가입**인 경우에는 화면에서 숨겨진다.
- 가입 가능 보험 목록은 **reqBuyType**(MM/UU)과 **rprsPrdtId**(단말)에 따라 **MCP-API**의 `selectInsrProdList`로 조회하고, Portal WAS는 해당 API를 호출해 프론트에 전달한다.
- 보험 선택 후 **휴대폰 인증**은 NICE만 사용하며, **본인인증과 동일한 CI(동일 명의 휴대폰)**일 때만 성공 처리되어 `saveNiceInsrRes`로 세션에 적재된다.
- **서식지 저장** 및 **개통 요청** 시점에 `insrProdCd`가 있으면 반드시 위 인증 세션과 CI 일치 여부를 검사하고, 통과 시 `insrAuthInfo`(ReqNo/ResNo)와 `clauseInsrProdFlag=Y`를 설정한 뒤 `MCP_REQUEST`에 저장한다.
- 스캔 서버용 PDF에는 보험 가입 시 **이동통신 단말기 보험 상품 설명서** 12/14페이지 메타가 사용되며, 상품 코드에 따라 14페이지(자급제 신품 계열) 또는 12페이지(그 외)가 선택된다.
