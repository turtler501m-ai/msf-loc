# 명의변경 관리자 화면 분석

**대상**: MCP 관리자(mcp-admin-was) 명의변경관리 화면  
**참조**: `51.명의변경_ASIS_소스분석.md`, `53.명의변경_M전산_소스분석.md`

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **메뉴** | 고객센터관리 > 명의변경관리 (메뉴코드: AM007, AM707) |
| **역할** | 포털에서 접수된 명의변경 신청(양도인/양수인) 목록 조회, 상세 확인, 처리상태·메모 수정, 개인정보 삭제(처리완료) |
| **데이터** | NMCP_NFL_CHG_TRNS(양도인), NMCP_NFL_CHG_TRNSFE(양수인) |

---

## 2. 화면·URL 구조

| 구분 | URL | View 경로 | 설명 |
|------|-----|-----------|------|
| **목록** | `GET/POST /cs/transferList.do` | `/transfer/transferList` → `views/transfer/transferList.jsp` | 명의변경 신청 목록 + 검색·페이징 |
| **상세** | `GET/POST /cs/transferDetail.do` | `/cs/transfer/transferDetail` → `views/cs/transfer/transferDetail.jsp` | 양도인/양수인 정보 조회, 처리상태·메모 수정, 개인정보 삭제 |
| **저장** | `POST /cs/transferDetailSave.do` | - (redirect: `/cs/transferList.do`) | 처리상태(trnsStatusVal), 메모(confirmMemo) 저장 |
| **삭제** | `POST /cs/transferDetailDel.do` | - (redirect: `/cs/transferList.do`) | 개인정보 마스킹 후 처리완료(3)로 변경 |

※ View가 `transfer/` 와 `cs/transfer/` 두 위치에 중복 존재할 수 있음. 컨트롤러 반환값 기준으로는 리스트 `/transfer/transferList`, 상세 `/cs/transfer/transferDetail`.

---

## 3. 컨트롤러·서비스·DAO

| 계층 | 클래스 | 파일 | 역할 |
|------|--------|------|------|
| Controller | TransferController | cs/controller/TransferController.java | transferList, transferDetail, transferDetailSave, transferDetailDel |
| Service | TransferSvc, TransferSvcImpl | cs/service/TransferSvc(Impl).java | countTransferList, transferList, transferDetail, transferDetailSave, transferDetailDel |
| DAO | TransferDao, TransferDaoImpl | cs/dao/TransferDao(Impl).java | CsTransferMapper 호출 (namespace CsTransferMapper) |
| Mapper XML | CsTransferMapper | resources/sql/map/CsTransferMapper.xml | trnsStausEdit, countTransferList, transferList, transferDetail, transferDetailSave, transferDetailDel |

**TransferSvcImpl.transferList()**  
- 호출 시 **trnsStausEdit()** 선행 실행 → 7일 초과·상태 1(대기중) 건을 상태 4(부재중)로 일괄 변경 후 목록 조회.

## 4. 목록 화면 (transferList.jsp)

### 4.1 레이아웃·메뉴

- **타이틀**: 명의변경신청  
- **breadcrumb**: 홈 > 고객센터관리 > 명의변경관리  
- **메뉴 코드**: menuTopCode="AM007", menuSubCode="AM707"

### 4.2 검색

| 항목 | name | 설명 |
|------|------|------|
| 처리상태 | treatVal | 0:전체, 1:대기중, 2:신청완료, 3:처리완료, 4:부재중, 5:기한초과 |
| 검색구분 | multiVal | 0:전체, 1:작성자, 2:접수번호, 3:신청일, 4:처리일 |
| 검색어 | searchValue | multiVal 0,1,2 일 때 사용 (텍스트) |
| 기간 | startDt, endDt | multiVal 3(신청일), 4(처리일) 일 때 표시 (datepick) |

- **검색 버튼**: `searchBtn()` → multiVal 3/4 일 때 startDt, endDt 필수 체크 후 form submit.  
- **form**: `transferListForm`, action=`/cs/transferList.do`, method=POST. hidden: trnsApyNo, pageNo.

### 4.3 목록 테이블

| 컬럼 | 내용 |
|------|------|
| 번호 | boardNum (행 번호) |
| 접수번호 | trnsApyNo |
| 구분 | 양도인 / 양수인 (2행으로 표시) |
| 작성자 | 양도인: trnsNm, 양수인: trnsfeNm (양수인 미등록 시 '-') |
| 접수상태 | "신청" 고정 |
| 신청일 | 양도인: cretDt, 양수인: fcretDt |
| 처리상태 | trnsStatusVal (1~5 한글 표시) |
| 처리일 | 상세에서만 사용, 목록에서는 '-' |
| 개인정보삭제여부 | 목록에서는 '-' |

- **행 클릭**: `goDetail(trnsApyNo)` → trnsApyNo를 form hidden에 넣고 action을 `/cs/transferDetail.do`로 바꾼 뒤 submit (목록 검색 조건 유지).

### 4.4 페이징

- **한 페이지당**: 10건 (pageInfoBean.recordCount)  
- **컴포넌트**: `<nmcp:pageInfoAdmin pageInfoBean="${pageInfoBean}" function="transferListSubmit"/>`  
- **transferListSubmit(pageNo)**: pageNo 설정 후 form submit.

## 6. 백엔드 처리 (Mapper)

| ID | 용도 |
|----|------|
| trnsStausEdit | TRNS_STATUS_VAL='1' 이고 신청일(CRET_DT) 7일 초과 → '4'(부재중) 업데이트 (리스트 조회 전 실행) |
| countTransferList | treatVal, multiVal, searchValue, startDt, endDt 조건으로 NMCP_NFL_CHG_TRNS join TRNSFE 건수 |
| transferList | 동일 조건 + skipResult, maxResult로 목록 조회 (ORDER BY a.CRET_DT DESC) |
| transferDetail | trnsApyNo로 양도인·양수인 조인 상세 1건 |
| transferDetailSave | trnsSeq 기준 TRNS_STATUS_VAL, CONFIRM_MEMO, AMD_ID, AMD_DT 업데이트 |
| transferDetailDel | trnsSeq 기준 개인정보 필드 '삭제' 마스킹, TRNS_STATUS_VAL='3'(처리완료), AMD_ID, AMD_DT 업데이트 |

## 7. 정리·주의사항

1. **관리자 기능 범위**  
   - 목록/상세 조회, 처리상태·메모 수정, 개인정보 삭제(처리완료)만 수행.  
   - **M전산 가족관계 서류 확인·명의변경 실행(MP0 등) 호출 없음** (53.명의변경_M전산_소스분석.md §8 참고).

2. **상세 화면 버그**  
   - 양수인 **휴대폰번호** 표시 시 `trans.trnsMobileNo`(양도인 휴대폰)를 사용하고 있음.  
   - **trnsfeMobileNo**를 사용하도록 수정 필요 (views/cs/transfer/transferDetail.jsp 142~147행 부근).

3. **View 경로**  
   - 리스트: `/transfer/transferList`  
   - 상세: `/cs/transfer/transferDetail`  
   - 동일 내용 JSP가 `views/transfer/` 와 `views/cs/transfer/` 에 둘 다 있을 수 있으므로, 실제 사용 뷰는 ViewResolver 설정 확인 필요.

4. **처리상태 값**  
   - 1:대기중, 2:신청완료, 3:처리완료, 4:부재중, 5:기한초과.  
   - 상세에서 라디오는 1, 4, 3, 5만 제공(2는 양수인 등록 완료 시 자동이므로 변경 불가).
