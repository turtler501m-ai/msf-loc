# 일시정지 이력 조회 구조 분석

## 1. 개요

일시정지 이력은 **M-Platform X26(일시정지이력조회)** API를 통해 조회하며, 일시정지 첫 화면에서 **요약 정보**를 노출하고, **「일시정지 이력조회」** 버튼 클릭 시 **팝업**으로 상세 이력 목록을 보여준다.

---

## 2. 전체 흐름

```
[일시정지 화면] suspendView01 (회선 선택 → 조회)
    → X26 호출 → 요약(subStatus, susCnt, susDays, remainSusCnt, expectSpDate 등) 표시
    → [일시정지 이력조회] 버튼 클릭
        → suspenPosHisPop.do (strNcn = 선택 회선)
        → X26 재호출 → itemList(bkInfoDto 목록) 반환
        → pauseHistPop 팝업에 테이블로 표시 (시작일, 종료일, 구분, 정지일수)
```

---

## 3. 화면·URL

| 구분 | URL | 비고 |
|------|-----|------|
| 일시정지 첫 화면 (PC) | `/mypage/suspendView01.do` | 회선 선택 후 조회 |
| 일시정지 첫 화면 (모바일) | `/m/mypage/suspendView01.do` | 동일 기능 |
| 일시정지 이력 팝업 (PC) | `/mypage/suspenPosHisPop.do` | POST, strNcn 필수 |
| 일시정지 이력 팝업 (모바일) | `/m/mypage/suspenPosHisPop.do` | 동일 |

---

## 4. 백엔드 구조

### 4.1. PauseController

**파일**: `mcp-portal-was/.../mypage/controller/PauseController.java`

#### 4.1.1. 일시정지 첫 화면 — `suspendView01`

- **역할**: 회선 선택 후 조회 시, X26으로 **요약 정보**만 조회해 첫 화면에 뿌린다.
- **호출**: `mPlatFormService.suspenPosHis(ncn, ctn, custId, "1")`  
  - `termGubun = "1"` → **최근 1년** 구간.
- **모델에 넣는 값** (이력 요약용):
  - `subStatus` — 현재 상태 (A: 개통, S: 일시정지)
  - `susCnt` — 고객 정지 횟수(이용)
  - `susDays` — 일시정지 일수(이용)
  - `solSusDays` — 이용정지 기간 일수
  - `expectSpDate` — 일시정지 종료 예정일 (포맷: yyyy.mm.dd)
  - `remainSusCnt` — 잔여 정지 횟수
- **예외**: `SelfServiceException` → RESULT_CODE 001, `SocketTimeoutException` → 002.

#### 4.1.2. 일시정지 이력 팝업 — `suspenPosHisPop`

- **매핑**: `GET/POST`  
  `"/mypage/suspenPosHisPop.do"`, `"/m/mypage/suspenPosHisPop.do"`
- **파라미터**: `strNcn` (선택한 회선의 서비스 계약번호, 9자리).
- **처리 순서**:
  1. 로그인/정회원 체크.
  2. `cntrList`에서 `strNcn`과 일치하는 회선의 `custId`, `ctn` 추출.
  3. `mPlatFormService.suspenPosHis(strNcn, ctn, custId, "1")` 호출.
  4. `suspenPosHisVO.getItemList()` → `List<MpBkInfoDto>` 취득.
  5. 리스트 각 건에 대해:
     - `csaActivityRsnDesc`: `"고객요청"`, `"-"` 제거 후 저장.
     - `remainSusCnt`, `colSusDays`: `StringUtil.yyyymmddDot(..., ".")` 로 날짜 포맷.
  6. `suspenPosHisList`를 모델에 담아 뷰로 전달.
- **뷰**:
  - PC: `/portal/mypage/pause/pauseHistPop`
  - 모바일: `/mobile/mypage/pause/pauseHistPop`
- **예외**: SelfService / SocketTimeout / Exception 시 로그만 남기고, 빈 리스트로 팝업 표시.

---

### 4.2. MplatFormService — X26 연동

**파일**: `mcp-portal-was/.../common/mplatform/MplatFormService.java`

- **메서드**: `suspenPosHis(String ncn, String ctn, String custId, String termGubun)`
- **API**: **26. 일시정지이력조회(X26)**  
  - `ncn`: 서비스 계약번호 9자리 (하이픈 제외)  
  - `ctn`: 전화번호 11자리 (10자리면 앞에 0 추가)  
  - `custId`: 고객 ID  
  - `termGubun`: `"1"` = 최근 1년, `"A"` = 전체
- **동작**:
  - `getParamMap(ncn, ctn, custId, "X26")` 후 `param.put("termGubun", termGubun)`.
  - LOCAL이면 `getVo(26, vo)` 로 목 XML 반환, 아니면 `mplatFormServerAdapter.callService(param, vo)` 로 M-Platform 호출.
- **반환**: `MpSuspenPosHisVO` (공통 헤더 + 요약 필드 + 이력 리스트).

---

### 4.3. 응답 VO 구조

#### 4.3.1. MpSuspenPosHisVO

**파일**: `mcp-portal-was/.../common/mplatform/vo/MpSuspenPosHisVO.java`

- **상속**: `CommonXmlVO` (responseType, responseCode, body 파싱 공통).
- **요약 필드** (outDto 루트에서 파싱):

| 필드 | 설명 |
|------|------|
| subStatus | 현재 상태 (A: 개통, S: 일시정지) |
| reckonFromDate | 산정 시작일 |
| susCnt | 고객 정지 횟수(이용) |
| susDays | 일시정지 일수(이용) |
| remainSusCnt | 잔여 정지 횟수 |
| solSusDays | 이용정지 기간 일수 |
| remainOgDays | 착신정지 가능 일수(발신정지 가능 일수) |
| m2mRemainSusCnt | M2M 정지 잔여 횟수 |
| expectSpDate | 일시정지 종료 예정일 |

- **이력 리스트**: `outDto` 하위 `bkInfoDto` 리스트를 `MpBkInfoDto` 리스트로 파싱 → `getItemList()`.

#### 4.3.2. MpBkInfoDto (이력 1건)

**파일**: `mcp-portal-was/.../common/mplatform/vo/MpBkInfoDto.java`

| 필드 | XML 태그 | 용도 (화면) |
|------|----------|-------------|
| remainSusCnt | remainSusCnt | 시작일 |
| colSusDays | colSusDays | 종료일 |
| remainOgDays | remainOgDays | 정지일수 |
| csaActivityRsnDesc | csaActivityRsnDesc | 구분 (예: 고객요청 발신정지) |

- **참고**: DTO 주석상 `remainSusCnt`는 “시작일자”, `colSusDays`는 “종료일자”, `remainOgDays`는 “고객정지일수(건수)”로 사용된다.  
- **버그 가능성**: getter가 `getcolSusDays()`(소문자 c)로 되어 있어, 일부 환경에서 EL `colSusDays`와 불일치할 수 있음. 필요 시 `getColSusDays()`로 통일 권장.

---

## 5. M-Platform 요청/응답 (X26)

- **이벤트 코드**: `X26`
- **요청 파라미터**: `getParamMap` 공통 + `termGubun`  
  - ncn, ctn, custId, userId, appEventCd(X26), termGubun(1 또는 A)
- **LOCAL 목 응답** (case 26):  
  - `outDto` 하위에 `bkInfoDto` 다수 (colSusDays, csaActivityRsnDesc, remainOgDays, remainSusCnt)  
  - 루트에 요약 필드: subStatus, reckonFromDate, susCnt, susDays, remainSusCnt, solSusDays, remainOgDays, expectSpDate 등

## 6. 프론트엔드

### 6.1. 일시정지 첫 화면 (pauseInfo)

- **JSP**:  
  - PC: `.../portal/mypage/pause/pauseInfo.jsp`  
  - 모바일: `.../mobile/mypage/pause/pauseInfo.jsp`
- **스크립트**: `pauseInfo.js` (portal / mobile 각각)
- **동작**:
  - 회선 셀렉트 `#ctn` → 조회 버튼 클릭 시 `select()` → `POST /mypage/suspendView01.do` (또는 `/m/suspendView01.do`) with `ncn`.
  - 테이블: 일시정지 횟수 이용/잔여(`susCnt`/`remainSusCnt`), 일시정지 일수 이용/잔여(`susDays`/(180-solSusDays)), 종료 예정일(`expectSpDate`) 또는 “회선 사용 중”.
  - **「일시정지 이력조회」** 버튼: `#btnSuspenHis`  
    - 클릭 시 `strNcn = $("#ctn option:selected").val()` 로 보내기  
    - `openPage('pullPopupByPost', '/mypage/suspenPosHisPop.do', parameterData)` (모바일은 `/m/mypage/suspenPosHisPop.do`)  
    - 팝업으로 이력 테이블 로드.

### 6.2. 일시정지 이력 팝업 (pauseHistPop)

- **JSP**:  
  - PC: `.../portal/mypage/pause/pauseHistPop.jsp`  
  - 모바일: `.../mobile/mypage/pause/pauseHistPop.jsp`
- **테이블 컬럼**:
  - 시작일: `suspenPosHisList.remainSusCnt`
  - 종료일: `suspenPosHisList.colSusDays`
  - 구분: `suspenPosHisList.csaActivityRsnDesc`
  - 정지일수: `suspenPosHisList.remainOgDays`
- **데이터 없을 때**: “조회 결과가 없습니다.”
- **안내 문구**: “내역조회는 최근 1년간 최대 100건까지만 가능합니다.”

## 9. 참고 사항

- **조회 기간**: 현재 모두 `termGubun = "1"`(최근 1년)로만 호출. 전체 기간은 `"A"`.
- **첫 화면 vs 이력 팝업**: 동일 X26 API를 사용하며, 첫 화면은 요약 필드만, 이력 조회 버튼은 `getItemList()`(bkInfoDto 목록)만 사용한다.
- **Admin**: `mcp-admin-was`에도 `MplatFormService.suspenPosHis`, `MpSuspenPosHisVO` 등 동일 패턴이 있음.
