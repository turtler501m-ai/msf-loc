## 2. 전체 플로우 ※ v20260306

```
메인(S103050101) → 고객(S103010101) → 상품(S103020101) → 동의(S103030101) → 신청완료(S103040101)
```

- **처리 흐름**  
  신청서 작성 완료 → [작성 완료] → MC0 명의변경사전체크 → MP0 명의변경 → 리포트 이미징 → 이미징 시스템 전송 → K-NOTE(신청서·첨부서류).  
  신청서·첨부서류 전송으로 접수만 진행, 고객센터/상담사 확인 후 M전산에서 후속 처리(IT 프로세스 검토 예정).

## 8. 최신 개발 소스 기준 재검토 (2026.03 기준)

서식지 프로젝트 규칙(`10.서식지프로젝트.md`) 및 본 분석 문서를 기준으로, **현재 mform-web 개발 소스**와의 정합성을 검토한 결과입니다.

### 8.1 프로젝트 규칙 준수 현황

| 항목 | 규칙/분석 문서 | 현재 소스 (mform-web) | 비고 |
|------|----------------|------------------------|------|
| **개발 폴더** | `msf/formOwnChg` (백엔드), 프론트는 domain=ident | 라우트 `mobile/:domain/:service` → `ident/IdentTypeCust` 등 | ✅ 일치 |
| **API 명칭** | `api/msf.js`, msfPost/msfGet | `api/msf.js`, `api/ident.js`에서 msfPost 사용 | ✅ 적용됨 |
| **스토어** | msf_menu, msf_comp_loading | `stores/msf_menu.js`, `stores/msf_comp_loading.js` | ✅ 적용됨 |
| **서브메뉴** | 1.고객 2.상품 3.동의 | 명의변경 children: 고객 → 상품 → 동의 | ✅ 일치 |
| **화면 퍼블** | 왼쪽 라벨 + 오른쪽 입력, 라벨 폭 일치 | ident-form-row, ident-label 140px 등 공통 클래스 사용 | ✅ 유지 |
