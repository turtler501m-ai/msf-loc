# 가입자정보조회 X01 · Y04 ASIS 로직·구조 상세 분석

**목적**: 휴대폰 인증(가입자정보조회) 관련 ASIS의 X01(가입정보조회), Y04(계약정보 유효성체크) 로직 및 클래스/메소드 구조를 상세 정리  
**참조**: 15.개발기능목록_ASIS_TOBE_기능분석명세서.md, 53.명의변경_M전산_소스분석.md, 13.프로그램명세서.md §2.4

---

## 1. 개요

| 구분 | API/이벤트 | 용도 | 비고 |
|------|-------------|------|------|
| **X01** | 가입정보조회 (perMyktfInfo / moscPerInfo) | ncn, ctn, custId로 M플랫폼 호출 → addr, email, homeTel, initActivationDate 등 반환 | 휴대폰 인증 **이후** 이미 ncn/custId가 있을 때 가입정보 조회 |
| **Y04** | 계약정보 유효성체크 (moscSvcContValdChk) | clientIp, cIntUsrId, ncn, ctn, custId로 계약 일치 여부 검증 | 휴대폰 인증 **단계**에서 “이름+ctn ↔ ncn·custId” 일치 검증 |
| **M전산 조회** | 계약번호/고객번호 조회 | 이름+휴대폰 → ncn(계약번호), (선택) custId | X01·Y04 호출 **전**에 ncn/custId를 확보하기 위한 선행 단계 |

**전체 흐름 (설계서·기능목록 기준)**  
1. 고객 입력: 이름(name), 휴대폰(ctn)  
2. **M전산 조회**: 이름+ctn → ncn, custId (없으면 인증 실패)  
3. **Y04 호출**: 계약정보 유효성체크(ncn, ctn, custId 등) → 성공 시 휴대폰 인증 완료  
4. **X01 호출**(선택): 가입정보(주소, 이메일, 가입일 등) 조회 → 화면 표시용

## 2. ASIS – MCP 포털(mcp-portal-was · mcp-api)

### 2.1 M전산 조회 (계약번호 확보)

#### 2.1.1 호출 경로

| 계층 | 파일 | 메소드/ID | 설명 |
|------|------|-----------|------|
| Controller | `mcp-portal-was/.../mypage/controller/MypageUserController.java` | 회선 추가·비회원 인증 등 | 이름(name), 휴대폰(phone) 수신 후 `mypageUserService.selectContractNum(name, phone)` 호출 |
| Service | `mcp-portal-was/.../mypage/service/MypageUserServiceImpl.java` | `selectContractNum(String name, String phone)` | RestTemplate으로 **mcp-api** `apiInterfaceServer + "/mypage/contractNum"` 호출. params: `name`, `mobileNo`(phone). 반환: `String`(계약번호만) |
| API | mcp-api `MypageController` (추정) | `POST /mypage/contractNum` | M전산 조회 후 CONTRACT_NUM 반환 |
| Mapper | `mcp-api/.../mcp/mapper/MypageMapper.xml` | `selectContractNum` | M전산 테이블 조회 |

**MypageUserServiceImpl.java (254~273행)**  
- `HashMap`: `name`, `mobileNo`(phone)  
- `restTemplate.postForObject(apiInterfaceServer + "/mypage/contractNum", params, String.class)`  
- 반환: 계약번호 `String` (custId 미포함)

#### 2.1.2 M전산 쿼리 (MypageMapper.xml)

**파일**: `mcp/mcp-api/src/main/resources/mcp/mapper/MypageMapper.xml`

```xml
<select id="selectContractNum" parameterType="java.util.Map" resultType="java.lang.String">
    SELECT CONTRACT_NUM, SUBSCRIBER_NO AS MOBILE_NO
    FROM MSP_JUO_SUB_INFO@DL_MSP
    WHERE SUB_STATUS <![CDATA[<>]]> 'C'
      AND SUBSCRIBER_NO = TRIM(#{mobileNo})
      AND UPPER(REPLACE(SUB_LINK_NAME,' ','')) = UPPER(REPLACE(#{name},' ',''))
</select>
```

- **테이블**: `MSP_JUO_SUB_INFO@DL_MSP` (M전산 DB 링크)
- **조건**: SUB_STATUS <> 'C', 휴대폰(SUBSCRIBER_NO), 이름(SUB_LINK_NAME) 일치 (공백 제거·대소문자 무시)
- **반환**: resultType이 `java.lang.String` → **CONTRACT_NUM(ncn)만** 반환. CUSTOMER_ID(custId)는 조회하지 않음.

#### 2.1.3 Controller 사용처 (MypageUserController)

- **회선 추가 등**: 약 1320행 – `String contractNum = mypageUserService.selectContractNum(name, phone);`  
  - null이면 RESULT_CODE "0002", "사용자 정보가 일치하지 않습니다."  
  - 있으면 선납 여부 등 추가 검사 후 세션/인증 스텝 진행, `rtnMap.put("contractNum", contractNum)`  
- **회선 추가 처리**: 약 1480행 – `String contractNum = mypageUserService.selectContractNum(userVO.getName(), phone.toString());`  
  - 계약번호 확보 후 회선 등록 등 후속 처리

**정리**: ASIS MCP에서는 휴대폰 인증 시 **M전산 조회로 ncn(계약번호)만** 확보하며, custId는 이 쿼리에서 반환하지 않음.

### 2.2 X01 가입정보조회 (MplatFormService)

#### 2.2.1 진입점

**파일**: `mcp-portal-was/.../common/mplatform/MplatFormService.java`

- **메소드**: `perMyktfInfo(String ncn, String ctn, String custId)` (60~69행)
- **역할**: 이미 확보된 ncn, ctn, custId로 **X01** 이벤트 코드로 M플랫폼 호출 → 가입정보(주소, 이메일, 휴대폰, 가입일 등) 반환.

```java
public MpPerMyktfInfoVO perMyktfInfo(String ncn, String ctn, String custId) throws ... {
    MpPerMyktfInfoVO vo = new MpPerMyktfInfoVO();
    HashMap<String, String> param = getParamMap(ncn, ctn, custId, "X01");
    if ("LOCAL".equals(serverLocation)) {
        getVo(1, vo);   // Mock: case 1 가입정보조회
    } else {
        mplatFormServerAdapter.callService(param, vo);
    }
    return vo;
}
```

- **전제**: 호출 전에 **ncn, ctn, custId**가 이미 있어야 함. (MCP에서는 selectContractNum으로 ncn만 구하고 custId는 별도 경로 또는 빈값으로 X01 호출 가능.)

#### 2.2.2 getParamMap (공통 파라미터 생성)

**위치**: 동일 클래스 3148~3163행

```java
private HashMap<String, String> getParamMap(String ncn, String ctn, String custId, String eventCd) {
    HashMap<String, String> param = new HashMap<String, String>();
    String userId = this.sesUserId();  // 세션/쿠키 사용자 ID
    param.put("ncn", ncn);
    param.put("ctn", ctn);
    param.put("custId", custId);
    param.put("userid", userId);
    param.put("appEventCd", eventCd);
    return param;
}
```

- **X01 시**: `appEventCd = "X01"`.  
- **규격**: ncn 9자리(숫자만), ctn 11자리(10자리면 앞 0 추가) 등은 주석/규격서 기준.

#### 2.2.3 LOCAL Mock – getVo(1, vo) (case 1)

**위치**: 동일 클래스 3223~3225행 (switch case 1)

- **의미**: `SERVER_NAME=LOCAL` 일 때 실제 M플랫폼 대신 Mock XML로 응답.
- **응답 구조**: `moscPerInfoResponse` 래퍼 내 `<return>` → bizHeader(appEventCd X01), commHeader(responseType N), **outDto**:  
  `<addr>`, `<email>`, `<homeTel>`, `<initActivationDate>` 등 가입정보 필드.

```text
case 1: // 가입정보조회
  outDto: addr, email, homeTel, initActivationDate
  responseType: N (정상)
```

- **VO**: `MpPerMyktfInfoVO`가 이 XML을 파싱해 addr, email, homeTel, initActivationDate 등으로 세팅.

#### 2.2.4 운영 연동

- **LOCAL**이 아니면 `mplatFormServerAdapter.callService(param, vo)` 호출.
- **MplatFormServerAdapter**: getURL(param), M플랫폼 연동 URL 호출, 응답 XML을 vo에 세팅 후 `vo.toResponseParse()` 등으로 파싱.

**정리**: ASIS X01은 **ncn, ctn, custId + appEventCd=X01**로 M플랫폼을 호출해 가입정보(addr, email, homeTel, initActivationDate)를 조회하며, MCP에서는 M전산 조회로 ncn만 먼저 구한 뒤 이 메소드를 호출하는 구조.

### 2.3 Y04 계약정보 유효성체크 (ASIS 위치)

- **기능목록·설계서**: 휴대폰 인증 시 **Y04** 계약정보 유효성체크 사용.  
  - 인터페이스: IF_0179 / Y04  
  - M플랫폼 서비스: `MoscSvcContService.moscSvcContValdChk`
- **mcp-portal-was MplatFormService.java 내부**: **Y04 전용 메소드나 getParamMap(..., "Y04") 호출은 없음.**  
- **추정**:  
  - msc-prx 등 다른 레이어에서 Y04 호출, 또는  
  - M전산 조회(selectContractNum) 성공만으로 “계약 일치”로 간주하고, 별도 Y04 없이 X01만 호출하는 구성일 수 있음.  
- **TO-BE(mform-api)**: `MplatFormSvc.contractValdChk(clientIp, cIntUsrId, ncn, ctn, custId)`에서 **Y04**를 명시적으로 호출 (아래 §4 참조).

## 3. ASIS – MSP(ktisMvno) MplatformService

**파일**: `msp/ktisMvno/.../rcp/rcpMgmt/service/MplatformService.java`

- **역할**: 접수·상담 등 RCP에서 M플랫폼 호출 시 사용. X01은 **이미 ncn, ctn, custId가 있을 때** 가입정보 조회용.

### 3.1 perMyktfInfo (X01) (82~94행)

```java
/** 가입정보조회 (X01) */
public Map<String, Object> perMyktfInfo(String ncn, String ctn, String custId) {
    HashMap<String,String> param = new HashMap<String,String>();
    param.put("appEventCd", "X01");
    param.put("ncn", ncn);
    param.put("ctn", ctn);
    param.put("custId", custId);
    result = this.mplatformCall(param);
    return result;
}
```

- **mplatformCall**: serverName이 localhost 포함이면 **X01 Mock XML** 반환 (222~228행 부근).  
  - 성공 시: `<outDto><addr>...</addr><email>...</email><homeTel>...</homeTel><initActivationDate>...</initActivationDate></outDto>`, `responseType` N.  
  - 그 외: 실제 M플랫폼 URL 호출.

### 3.2 M전산 조회

- ktisMvno 쪽에서는 **MplatformService**가 M전산을 직접 조회하지 않음.  
- ncn, ctn, custId는 **호출하는 쪽**(예: AcenReceptionService 68~69행 – “가입정보조회 (X01)” 로 `mplatformService.perMyktfInfo(mpData.get("NCN"), mpData.get("CTN"), mpData.get("CUST_ID"))`)에서 이미 채워진 값을 넘김.

## 7. TO-BE Y04 처리 로직 ASIS 구조 정렬

TO-BE의 Y04 호출을 ASIS M플랫폼 연동 패턴과 동일한 구조로 맞춤.

### 7.1 ASIS 파라미터 구조

| 구분 | 내용 |
|------|------|
| **getParamMap** | ncn, ctn, custId, **userid**(세션), appEventCd |
| **Adapter saveMplateSvcLog** | param 에 **ip**(클라이언트IP), url, mdlInd 추가 후 getURL 인코딩 |
| **키 규칙** | userid 소문자 (MplatFormService 3156행) |

### 7.2 TO-BE 적용 사항

| 항목 | 변경 내용 |
|------|-----------|
| **MplatFormSvc.getParamMap** | param 키 `userId` → **userid** 로 변경 (ASIS와 동일) |
| **MplatFormSvc.contractValdChk** | 별도 HashMap 대신 **getParamMap(ncn, ctn, custId, "Y04")** 로 기본 param 생성 후 **ip(clientIp)** 추가. cIntUsrId 있으면 userid override. (X01/X20 등과 동일하게 getParamMap 기반) |
| **MplatFormServerAdapter.saveMplateSvcLog** | 서비스에서 이미 **ip** 를 넣은 경우(Y04) **덮어쓰지 않음**. 로그용 userid 키 참조. |

### 7.3 Y04 호출 시 최종 param 구조 (TO-BE)

- ncn, ctn, custId, **userid**, appEventCd=Y04, **ip** (clientIp)
- (선택) url, mdlInd 는 adapter 에서 설정값으로 보강

이 문서는 가입자정보조회 X01 및 Y04에 대한 ASIS 로직·명·구조를 상세히 분석한 내용이며, TO-BE의 join-info는 위 ASIS 흐름(M전산 → Y04 → X01 스펙 응답)을 하나의 API로 통합한 구조이다. Y04는 ASIS getParamMap + ip 보강 패턴과 동일하게 구현됨.

### 7.4 X01 가입정보조회 TO-BE 분리 (ASIS 동일)

| 항목 | ASIS | TO-BE (분리 후) |
|------|------|------------------|
| **X01** | MplatFormService.perMyktfInfo(ncn, ctn, custId) → getParamMap(..., "X01") → callService / getVo(1,vo) | MplatFormSvc.perMyktfInfo(ncn, ctn, custId) → getParamMap(..., "X01") → callService / getVo(1, vo) |
| **VO** | MpPerMyktfInfoVO (addr, email, homeTel, initActivationDate) | common/mplatform/vo/MpPerMyktfInfoVO.java 동일 스펙 |
| **join-info 흐름** | (MCP는 휴대폰 인증 후 별도 X01 호출) | M전산 조회 → Y04 → **X01(perMyktfInfo)** → JoinInfoResVO 에 addr/email/homeTel/initActivationDate 반영 |
