## 2. 신청서 작성 흐름

### 2.1 명의변경 신청서 작성 (1/2)

**[고객 단계 — 양도 고객]**

- **고객유형**: 내국인 / 미성년자(19세 미만) / 외국인(Foreigner) / 외국인(Foreigner) 미성년자 / 법인 / 공공기관
- **가입자 정보**: 이름, 생년월일/법인등록번호
- **명의변경 휴대폰번호** [인증]
- **법정대리인 정보** *(고객유형 '미성년자'인 경우만)*: 이름, 신청인과 관계, 주민등록번호, 휴대폰번호, 안내 및 동의사항

**[고객 단계 — 양수 고객]**

- **고객유형**: 내국인 / 미성년자(19세 미만) / 외국인(Foreigner) / 외국인(Foreigner) 미성년자 / 법인 / 공공기관
- **방문고객**: 본인 / 대리인
  - 법인/공공기관만 선택 / 성인 → `본인` 고정 / 미성년자 → `대리인` 고정
- **신분증 확인**: 주민등록증 / 운전면허증 / 모바일신분증 / 신원확인(안면인증)
  - 신분증 스캔 정보 K-NOTE 전송
- **가입자 정보**: 이름, 생년월일/법인등록번호
- **법정대리인 정보** *(고객유형 '미성년자'인 경우만)*: 이름, 신청인과 관계, 주민등록번호, 휴대폰번호, 안내 및 동의사항
- **실사용자 정보** *(법인/공공기관인 경우만)*: 이름, 생년월일
- **대리인 위임 정보** *(방문고객 '대리인'인 경우만)*: 이름, 생년월일, 신청인과 관계, 연락처
- **구비서류**: 스캔 정보 K-NOTE 전송 여부 및 전송 방식 확인
- **연락처 정보**: 연락처, 이메일주소, 주소, 국가/체류기간/비자 *(외국인만 해당)*
- **요금제**: 휴대폰/USIM 요금제 → 카테고리에서 선택
- **약관 동의**: 필수, 선택 (고객유형에 따라 해당하는 약관)

### 2.2 명의변경 신청서 작성 (2/2)

**[상품 단계]**

- USIM 정보 [체크]
- 납부 정보: 자동이체/신용카드/지로/통합청구, 본인납부/타인납부 [체크]
- 메모

**[동의 단계]**

- 고객 안내 사항
- 신청서 확인: 녹취, 가입정보 확인, 서명(가입자(대리인) 서명)
- 팝업 화면으로 최종 정보 확인 및 녹취
- 신청서 및 첨부서류 K-NOTE 전송 여부 확인
- 신청서 처리: 신청서 등록 → 리포트 이미징 처리 → 이미징 시스템 리포트 전송 → M전산 데이터 전송 → 신청서/첨부서류 K-NOTE 전송 *(별도 작성)*

**[완료 단계]**

- 신청서 교부: 열람, 발송 *(신규/변경 신청서와 동일)*

### 2.3 명의변경 신청서 처리 흐름

```
스마트 서식지
  → 신청서 작성 완료
  → 신청서 등록 (신청서)
  → 이미징 시스템: 리포트 이미징 처리 (리포트 서식)
  → 리포트 이미징 전송 → M전산 / M플랫폼
  → 명의변경 전송
  → K-NOTE: 신청서 및 첨부서류 전송
  → KOS: 신청서 / 첨부서류

※ 신청서 및 첨부서류를 전송해서 접수만 진행하고,
   고객센터/상담사 실제 확인 후 M전산에서 후속 처리
```

#### 3.1 양도 고객 (내국인, 외국인)

> *(2026.03.10) 양도고객 신분증 인증 필요 여부 — 유통영업팀 확인 후 피드백*

**신분증 스캔 유형별 추가 항목**

| 신분증 유형 | 추가 입력 항목 |
|------------|---------------|
| 주민등록증 | 발급 일자 (YYYY.MM.DD) |
| 운전면허증 | 면허번호, 발급 일자, 면허지역 |
| 국내여권 | 발급 일자 (YYYY.MM.DD) |
| 외국인등록증 | 발급 일자 (YYYY.MM.DD) |
| 영주증 | 발급 일자 (YYYY.MM.DD) |
| 국내거소신고증 | 발급 일자 (YYYY.MM.DD) |
| 국가보훈증 | 발급 일자 (YYYY.MM.DD) |

**양도 고객 정보 입력 항목**

| 항목 | 필수 | 설명 |
|------|------|------|
| 고객유형 | ✅ | 내국인/미성년자/외국인/외국인미성년자/법인/공공기관 |
| 신분증 스캔 | ✅ | 신분증 유형 선택 후 스캔하기 (B3) |
| 이름 | ✅ | — |
| 생년월일 | ✅ | 8자리(YYYYMMDD) |
| 성별 | — | 남/여 (기본: 남) |
| 명의변경 휴대폰번호 | ✅ | 010 - 가운데 4자리 - 뒤 4자리 |

**B1 버튼 (인증)**

- 활성화: 이름, 생년월일, 휴대폰번호 모두 입력 시
- 완료 후: `[인증 완료]`로 변경, 비활성화, 입력 항목 비활성화
- 인증 완료 이후: M플랫폼 API 호출하여 기존 가입정보 조회 → 가입정보, 연락처 정보 등 기본 세팅
- **개발**: M플랫폼 API — 계약정보 유효성체크(Y04), 가입정보조회(X01), 가입중인 부가서비스 조회(X97), 납부방법조회NEW(Y08)

**B3 버튼 (스캔하기)**: 스캔 팝업 표시. 스캔 후 텍스트 화면에 자동 세팅.

**신분증 스캔 세팅 규칙**
- 미성년자가 아닌 경우: 고객명, 주민등록번호/외국인등록번호, 생년월일을 스캔 연동값으로 고정 노출 (수정 불가)
- 미성년자인 경우: 법정대리인 정보에 스캔 연동값으로 고정 노출 (수정 불가)
- 스캔 이미지 → 구비서류로 제출 (신청서 이미징 처리 시 이미지와 병합 처리)

**메시지**

| 상황 | 메시지 |
|------|--------|
| 휴대폰번호 유효성 오류 | `고객 정보와 휴대폰번호가 일치하지 않습니다. 휴대폰번호를 다시 확인해 주세요.` |

**경로**: 서식지 App > 명의변경 신청서 > 동의

#### 화면 구성

| 번호 | 설명 |
|------|------|
| 1 | 좌측 메뉴: 작성단계에 맞춰 활성화되어 표시 |
| 2 | 동의 정보 입력/선택 영역 화면에 세팅 |
| A1 | **[작성 완료] 버튼** 활성화 조건: 화면 내 입력/선택 항목이 모두 입력/선택되면 활성화<br>클릭 시 유효성 체크: 필수값 입력 여부, 입력값 유효성, 서명 후 등록 가능, **안내 녹취 후 등록 가능**<br>클릭 시 처리: 신청서 작성 내용 등록 → M전산 전송 → 리포트 이미징 → 이미징 시스템 전송 |
| B1 | 신청서 확인 팝업 표시 |

**개발**
- M포털과 동일 사용
- M전산 테이블로 등록 → 신분증, 구비서류, 신청서 이미지 전송 (M포털과 동일하게 처리)
- M전산에서 결과 받은 데이터를 스마트신청서 테이블 데이터 현행화 처리

#### 고객 안내 사항 (필수 체크 항목)

```
※ 개통정보 녹음거부 동의
  개통 정보에 대해 모두 확인하였고, 신청내용에 이의가 없으며
  더 이상의 설명을 거부합니다.

※ 명의 대여 위험 안내
  요금 상품 및 부가 서비스를 고객의사와 상관없이 무단 가입 및 의무기간을 설정하지 않으며
  공시지원금 및 요금할인(선택약정)은 개통시점 공시된 기준으로 지급되며, 이에 동의합니다.
  신청자 본인은 대출업자 등 제3자에게 고객명의의 휴대폰을 개통해 주거나,
  개통에 필요한 신청서류를 제공하는 경우 휴대폰 대출 사기 등에 악용되어
  심각한 경제적 피해를 입을 수 있음을 안내 받았으며, ...이에 동의합니다.
  
      위 안내에 대해 확인하고 이해합니다.

※ 통신범죄예방 안내
  타인으로부터 전화나 문자 요청을 받고 그에 따라 휴대폰 또는 USIM 개통을 하지 않았음을 확인합니다.
  (전기통신사업법 등 관련 법률에 따라 형사처벌을 받을 수 있습니다.)
  
      위 안내에 대해 확인하고 이해합니다.

※ 명의도용방지 서비스(M-Safer) 안내
  명의도용방지서비스(M-Safer)는 본인명의 통신서비스 가입 현황을 실시간 조회 가능하며,
  본인동의 없이 통신서비스에 가입하지 못하도록 차단서비스를 제공합니다.
  (이용 및 신청방법: 홈페이지 또는 PASS앱 제공)
  
      위 안내에 대해 확인하고 이해합니다.

※ 판매자 확인 안내
  [판매자] 고객 보호를 위해서 통신상품 가입 시 본인확인 및 가입의사,
  추가이용에 대해서 성실히 안내 하였습니다.

※ 가입자 확인 안내
  [가입자] 본인 명의의 통신상품을 타인에게 제공하거나 매개하는 경우
  법률에 따라 처벌 받을 수 있습니다.
```

**메시지**

| 상황 | 메시지 |
|------|--------|
| 신청서 등록 확인 | `신청서를 등록하시겠습니까?` + [취소] [확인] |
| 신청서 등록 실패 | `신청서 등록이 실패하였습니다. 다시 시도해 주세요.` |
| 신청서 등록 완료 | `신청서 등록이 완료되었습니다.` |
| 필수값 미입력 시 | `XXX 입력/선택해 주세요.` |
