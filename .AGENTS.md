# AGENTS.md

Guidance for Codex agents working in this workspace.

## Project Overview

**MSF (스마트서식지)** is a mobile customer service application system for agencies. It replaces the existing M포탈(MCP) system while preserving equivalent business behavior and M플랫폼/KT(KOS) integration specs.

| Folder | Role |
| --- | --- |
| `msf-be-form-api/` | New backend: Java 25, Spring Boot, MyBatis, PostgreSQL/Oracle |
| `msf-form-web/` | New frontend: Vue 3, Vite, Tailwind CSS 4 |
| `msf-be-admin-api/` | Admin backend |
| `msf-admin-web/` | Admin frontend |
| `msf-be-batch/` | Batch processing |
| `msf-form-android/` | Android app |
| `msf-form-ios/` | iOS app |

## Core Working Rules

- When porting ASIS logic, do not delete unrelated legacy behavior. Comment it out if it is intentionally excluded.
- ASIS exclusion comment format: `// [ASIS] {기능 설명} - {제외 이유}`.
- During early migration work, prefer minimal changes that compile over creating unnecessary new DTO/service layers.
- Implement behavior to match M포탈 source and integration API request/response specs.
- If table layouts or DB mappings are unclear, verify against DB metadata or `.doc/reference/` before changing SQL.
- Backend APIs use only `GET` and `POST`. Do not add `PUT`, `PATCH`, or `DELETE`.
- On Windows, test Korean JSON APIs with Node.js `http.request()` instead of `curl` to avoid encoding issues.

## Architecture

```
msf-form-web (port 7080) -> API proxy -> msf-be-form-api (port 8080)
    -> smartform DB (PostgreSQL)
    -> msp DB (Oracle, DB link DL_MSP)
    -> M플랫폼 HTTP -> KT(KOS)
```

## Backend Commands

Run from `msf-be-form-api/`.

```bash
./gradlew :app-boot:bootJar
./gradlew :app-boot:bootRun
java -jar app-boot/build/libs/app-boot-*.jar
./gradlew test
./gradlew test --tests "com.ktmmobile.msf.domains.form.SomeTest"
```

Local DB settings:

- Main local config: `app-boot/src/main/resources/application-private.yaml`
- `SERVER_NAME: LOCALSERVER` returns M플랫폼 mock responses.
- Existing table layouts should be treated as authoritative. Confirm before changing schemas or SQL mappings.

## Backend Structure

`msf-be-form-api/` Gradle modules:

```text
app-boot/           Spring Boot entry point (FormApiApplication.java)
commons/
  common/           Common utils, constants, cache, common enums
domains/
  form/             Business domain code
external/
  mybatis/          Two DataSources, MyBatis interceptors
  websecurity/      Spring Security config
```

Business code belongs in `domains:form`.

Main packages under `com.ktmmobile.msf.domains.form.*`:

| Package | Role | Main paths |
| --- | --- | --- |
| `common` | Common adapters, mspservice, commCode, cache, exceptions, constants | - |
| `config` | `FormMapperConfig` / `@MapperScan` | - |
| `form.common` | Order, phone, USIM query controllers | `/order/*`, `/usim/*` |
| `form.newchange` | New subscription/change application forms | `/appForm/*`, `/appform/*` |
| `form.ownerchange` | Owner change | `/mypage/myNameChg*` |
| `form.servicechange` | Service change, mypage, add-ons, plan, combination, data sharing | `/mypage/*` |
| `form.termination` | Service termination | `/mypage/cancelConsult*` |
| `login` | Login | - |
| `mobileapp` | Mobile app integration | - |
| `system.cert` | Identity verification | `/cert/*` |
| `system.faceauth` | Face authentication | `/fath/*` |

Typical domain layers: `controller / dao / dto / service`. Some domains also have `mapper/` and `constant/`.

## MyBatis Rules

DataSources:

- `smartform`: PostgreSQL MSF DB
- `msp`: Oracle M전산 DB

Default pattern:

- Existing code often uses `SqlSession` in `XxxDaoImpl`.
- New `@Mapper` interfaces are allowed, but their packages must be registered in `FormMapperConfig.java` `@MapperScan`.
- Missing mapper scan registration causes `NoSuchBeanDefinitionException` at startup.

Example:

```java
@MapperScan(
    basePackages = {
        "com.ktmmobile.msf.domains.form.form.common.mapper"
        // Add new mapper packages here.
    },
    sqlSessionTemplateRef = "sqlSessionTemplate"
)
```

## Naming Rules

- Controller: `XXXController` for internal use, `MsfXXXController` for service-change areas.
- Service: `XxxSvc.java` / `XxxSvcImpl.java` or `XxxService.java` / `XxxServiceImpl.java`.
- DAO: `XxxDao.java` / `XxxDaoImpl.java`.
- DTO: `XXXReqDto` for requests, `XXXResVO` or `XXXDto` for responses.
- Mapper XML IDs: `insert/save/update/delete/select/selectList` + service name.

## Windows API Test Example

Use Node.js instead of `curl` for Korean JSON payloads.

```js
const http = require('http');
const body = JSON.stringify({ name: "홍길동", ctn: "01012345678" });
const req = http.request(
  {
    hostname: 'localhost',
    port: 8080,
    path: '/api/v1/cancel/eligible',
    method: 'POST',
    headers: {
      'Content-Type': 'application/json; charset=utf-8',
      'Content-Length': Buffer.byteLength(body)
    }
  },
  res => {
    let d = '';
    res.on('data', c => d += c);
    res.on('end', () => console.log(d));
  }
);
req.write(body);
req.end();
```

## ASIS to TOBE DB Table Conversion

When porting ASIS mapper SQL, table handling depends on source.

| ASIS table source | Prefix | TOBE handling |
| --- | --- | --- |
| M포탈 DB | `MCP_*`, `NMCP_*` | Replace prefix with `MSF_*` when the same structure exists in MSF DB |
| M플랫폼 DB | `MSP_*` | Keep as `MSP_*@DL_MSP` DB link |

Examples:

- `NMCP_CUST_REQUEST_MST` -> `MSF_CUST_REQUEST_MST`
- `MCP_REQUEST` -> `MSF_REQUEST`
- `MSP_INTM_INSR_MST` -> `MSP_INTM_INSR_MST@DL_MSP`

Full table mapping: `.doc/reference/D2.테이블매핑_20260319.md`.
