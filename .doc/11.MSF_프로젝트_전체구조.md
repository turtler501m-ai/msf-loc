## 1. 워크스페이스 전체 구성

```
C:/MSF-LOC/workspace/
├── CLAUDE.md                   # 프로젝트 가이드 (AI 개발 지침)
├── .claudeignore               # Claude 컨텍스트 제외 설정
├── .claude/                    # Claude Code 설정 (skills, commands)
├── .doc/                       # 개발 문서 (마크다운)
├── mcp/                        # 기존 M포탈 (레거시, ASIS 참조용)
├── msp/                        # M플랫폼 연계처리 (레거시, 참조용)
└── msf/                        # 신규 스마트서식지 (TOBE, 개발 대상)
```

### 역할 구분

| 폴더 | 역할 | 사용 목적 |
|------|------|----------|
| `mcp/` | 기존 M포탈 전체 소스 | ASIS 참조 전용 (수정 금지) |
| `msp/` | M플랫폼 연계처리 레거시 | 참조용 (연동규격은 .doc에 정리됨) |
| `msf/` | 신규 스마트서식지 | 개발 대상 (msf-api + msf-web) |
| `.doc/` | 개발 분석·설계 문서 | 기능 명세, 연동 스펙 |
| `.claude/` | Claude Code 설정 | 스킬 가이드, 슬래시 커맨드 |

## 3. msf/ — 신규 스마트서식지

### 3.1 msf-api (백엔드)

**기술 스택**

| 항목 | 내용 |
|------|------|
| 언어 | Java 11 |
| 프레임워크 | Spring Boot 2.7.18 |
| ORM | MyBatis 2.3.2 (XML Mapper) |
| DB (로컬) | PostgreSQL (`localhost:5432/msf`) |
| DB (운영) | Oracle (`@DL_MSP` M전산) |
| 빌드 | Maven |
| 포트 | 8081 |

**도메인 패키지 역할**

| 패키지 | 역할 | Controller | API 경로 |
|--------|------|-----------|---------|
| `formComm` | 공통: 가입자정보조회, 동시처리체크 | SvcChgRestController | `/api/v1/{domain}/join-info` |
| `formSvcChg` | 서비스변경: 부가서비스·요금상품 | SvcChgBaseController, SvcChgRegController | `/api/v1/addition/*`, `/api/v1/service-change/*` |
| `formOwnChg` | 명의변경 | OwnChgController | `/api/v1/ident/*` |
| `formSvcCncl` | 서비스해지 | SvcCnclController | `/api/v1/cancel/*` |
| `common.mplatform` | M플랫폼 HTTP 연동 어댑터 | (서비스 계층에서 직접 호출) | - |

| 도메인 | 메서드 | 경로 | 설명 |
|--------|--------|------|------|

| 부가서비스 | POST | `/api/v1/addition/cancel` | 부가서비스 해지 (X38) |
| 서비스변경 | GET | `/api/v1/service-change/base/targets` | 변경 대상 조회 |
| 서비스변경 | POST | `/api/v1/service-change/base/check` | 유효성 체크 |
| 명의변경 | POST | `/api/v1/ident/eligible` | 명의변경 가능 여부 |
| 명의변경 | POST | `/api/v1/ident/apply` | 명의변경 신청 |
| 서비스해지 | POST | `/api/v1/cancel/eligible` | 해지 가능 여부 |
| 서비스해지 | POST | `/api/v1/cancel/remain-charge` | 잔금 조회 |
| 서비스해지 | POST | `/api/v1/cancel/apply` | 해지 신청 |

**M플랫폼 연동 코드**

| 이벤트코드 | 메서드 | 기능 |
|-----------|--------|------|
| Y04 | `contractValdChk()` | 계약정보 유효성체크 (휴대폰 인증) |
| X01 | `perMyktfInfo()` | 가입정보조회 |
| X20 | `getAddSvcInfoDto()` | 가입중 부가서비스 조회 |
| X21 | `regSvcChg()` | 부가서비스 신청 |
| X38 | `moscRegSvcCanChg()` | 부가서비스 해지 |

## 8. 개발 환경 설정 요약

### 백엔드 실행

```bash
# PostgreSQL DB 생성 (최초 1회)
createdb mform
psql -d mform -f msf/msf-api/src/main/resources/init.sql

# 서버 실행
cd msf/msf-api
mvn spring-boot:run
# → http://localhost:8081
```

### 프론트엔드 실행

```bash
cd msf/msf-web
npm install
npm run dev
# → http://localhost:9480 (/api → localhost:8081 자동 프록시)
```

### M플랫폼 환경 전환

```properties
# application.properties
SERVER_NAME=LOCAL   # Mock 응답 (기본값)
# SERVER_NAME=DEV   # 실서버 전환 시
# juice.url=http://[msc-prx]:7006/mPlatform/serviceCall.do
```

### Windows API 테스트

```js
// curl 대신 Node.js 사용 (한글 UTF-8 인코딩 문제 회피)
const http = require('http');
const body = JSON.stringify({ name: "홍길동", ctn: "01012345678" });
const req = http.request({
  hostname: 'localhost', port: 8081,
  path: '/api/v1/cancel/eligible', method: 'POST',
  headers: { 'Content-Type': 'application/json; charset=utf-8',
             'Content-Length': Buffer.byteLength(body) }
}, res => { let d = ''; res.on('data', c => d += c); res.on('end', () => console.log(d)); });
req.write(body); req.end();
```
