## 1. 공통 미적용 항목 (전 도메인)

E1 전체에 걸쳐 반복 등장하는 공통 처리 로직으로, 현재 MSF에 미적용 상태.

### 1-1. SCAN 서버 서식지 전송 (`custRequestScanService.prodSendScan`)

| 항목 | 내용 |
|------|------|
| E1 분석번호 | 2001-02(NC), 3001-02(CC), 5001-01(IS) |
| ASIS 클래스 | `CustRequestScanServiceImpl.prodSendScan` |
| 처리 | 신청서 XML 생성 → SCAN 서버 전송 |
| reqType | 명의변경=`NC` / 서비스해지=`CC` / 안심보험=`IS` |
| 상태 | ❌ 미구현 |

**XML 생성 흐름 (신규 포시에스 리포팅 시스템 적용 필요)**:
```
settingSsnData()            ← 주민번호 분석값 설정
settingData()               ← 각종 설정값 설정
createXml()
  └─ getGroupCode()
  └─ CustRequestMapper.getAppFormGroupPointList()
       → NMCP_APP_GROUP_MST / NMCP_APP_GROUP_INFO / NMCP_APP_FORM_INFO
saveXML()                   ← XML 파일 저장
xmlFileSend()               ← SCAN 서버 전송 + 삭제
```

> ⚠️ 포시에스 리포팅 시스템(신규) 기준으로 로직 변경 필요 — ASIS 로직 그대로 사용 불가

#### ③ N7001 — 청구계정 유효성 체크 SQL

| 항목 | 내용 |
|------|------|
| E1 분석번호 | N7001 |
| 처리 | 동일 고객 내 청구계정(BAN) 유효성 확인 |
| SQL | `MSP_JUO_BAN_INFO@DL_MSP` + `MSP_JUO_CUS_INFO@DL_MSP` JOIN |
| 조건 | BL_BILLING_METHOD IN ('C','D'), COL_DELINQ_STATUS = 'N' |
| 현재 | OwnChgController 청구계정 검증 내 적용 여부 확인 필요 |
| 상태 | ⚠️ 확인필요 |

```sql
-- N7001 적용 SQL (MSP DB링크 사용)
SELECT BAN
  FROM (SELECT BAN, CUSTOMER_ID
          FROM MSP_JUO_BAN_INFO@DL_MSP
         WHERE BL_BILLING_METHOD IN ('C', 'D')   -- 신용카드, 자동이체
           AND COL_DELINQ_STATUS = 'N'            -- 미납 없음
       ) A
 INNER JOIN (SELECT CUSTOMER_ID
               FROM MSP_JUO_CUS_INFO@DL_MSP
              WHERE CUSTOMER_SSN = #{customerSsn}
            ) B ON A.CUSTOMER_ID = B.CUSTOMER_ID
```
