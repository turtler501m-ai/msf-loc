## 2. 서비스해지 신청서 작성 흐름

```
고객 → 상품 → 동의 → 완료
```

### 작성 단계별 입력 항목 요약

| 단계 | 주요 입력 항목 |
|------|----------------|
| **고객** | 고객유형, 방문고객, 신분증 확인, 가입자 정보(이름/생년월일/법인등록번호), 연락처, 해지 휴대폰번호, 대리인 위임 정보, 법정대리인 정보(미성년자), 약관 동의 |
| **상품** | 해지 후 사용 여부(통신사 선택), 해지 정산(사용요금/위약금/단말기 할부금), 메모 |
| **동의** | 고객 안내 사항, 신청서 확인(가입자/대리인 서명) |
| **완료** | 신청서 열람, 발송(SMS/카톡), kt M모바일 앱 안내 발송 |

### 조건별 노출 규칙

| 조건 | 처리 |
|------|------|
| 법인/공공기관만 | 방문고객(본인/대리인) 선택 항목 노출 |
| 성인 | 방문고객 '본인' 고정 (미노출) |
| 미성년자 | 방문고객 '대리인' 고정 (미노출), 법정대리인 정보 노출 |
| 외국인 | 구비서류 스캔 항목 노출 |

## 3. 신청서 처리 흐름

```
신청서 작성 완료
    ↓
신청서 등록
    ↓
리포트 이미징 처리 (리포트 서식)
    ↓
리포트 이미징 전송 → 이미징 시스템 → 신청서
    ↓              
M전산 → 서비스해지 요청
    ↓
M플랫폼 → 서비스해지
    ↓
K-NOTE → 신청서 전송 + 첨부서류
```

#### 4-2-1. 고객 - 내국인 / 외국인

**화면명:** 서비스해지 신청서 – 고객(내국인, 외국인)

##### 화면 항목 구성

| 영역 번호 | 영역명 | 항목 |
|-----------|--------|------|
| 1 | 고객유형 | 고객유형* (내국인 / 미성년자(19세 미만) / 외국인(Foreigner) / 외국인(Foreigner) 미성년자 / 법인 / 공공기관) |
| 2 | 신분증 확인 | 신분증 스캔* (선택 드롭다운 + [스캔하기] B3 버튼) |
| 3 | 가입자 정보 | 이름*, 생년월일*(YYYYMMDD), 해지 휴대폰번호* (010-XXXX-XXXX + [인증] B1 버튼) |
| 8 | 구비서류 | 구비서류* [스캔하기 B6] — **외국인인 경우만 노출** |
| - | 해지 후 연락처 | → 18페이지 영역 참조 |
| - | 가입유형 선택 | → 18페이지 영역 참조 |
| - | 약관 동의 | → 18페이지 영역 참조 |

##### 항목 Description

**[1] 고객유형**
- 방문고객 노출 규칙:
  - 내국인, 외국인(Foreigner): 미노출 → `'본인'` 고정
  - 미성년자(19세 미만), 외국인(Foreigner) 미성년자: 미노출 → `'대리인'` 고정
  - 법인, 공공기관: 선택하도록 노출 (디폴트: 본인)

**[2] 신분증 확인 ([스캔하기] B3 버튼)**
- **활성화 조건:** 신분증 유형 선택시 버튼 활성화
- **인증 완료 이후:**
  - 버튼명 변경: `[신분증 유형]`
  - 버튼 비활성화
- **스캔 연동:**
  - 미성년자 아닌 경우: 고객명, 주민등록번호/외국인등록번호 생년월일 → 화면에 고정 노출 (수정 불가)
  - 미성년자인 경우: 법정대리인 정보에 고객명, 주민등록번호/외국인등록번호 생년월일 → 화면에 고정 노출 (수정 불가)
- **스캔 이미지:** 신분증 이미지 → 구비서류로 제출 (신청서 이미징 처리시 신청서 이미지와 병합)

**[3] 가입자 정보**
- 가입 정보 입력
- 해지할 휴대폰 번호 입력

**[B1] [인증] 버튼**
- **활성화 조건:** 이름, 주민등록번호, 휴대폰번호 모두 입력/선택시 활성화
- **인증 완료 이후:**
  - 버튼명 변경: `[인증 완료]`
  - 버튼 비활성화
  - 입력/선택 항목 비활성화(수정 불가): 이름, 주민등록번호, 휴대폰번호
- **유효성 체크:** 휴대폰번호 유효성 인증

**[8] 구비서류 ([스캔하기] B6)**
- **외국인인 경우만 노출**

##### B1 인증 상태별 화면 변화

| 상태 | 화면 표시 |
|------|-----------|
| 미인증 (모두 입력 전) | `[인증]` 버튼 비활성화 |
| 미인증 (모두 입력 후) | `[인증]` 버튼 활성화 |
| 인증 성공 | `[인증 완료]` 버튼 비활성화, 입력 항목 수정 불가 |

##### B1 메시지

| 상황 | 메시지 |
|------|--------|
| 휴대폰번호 유효성 오류 | `고객 정보와 휴대폰번호가 일치하지 않습니다. 휴대폰번호를 다시 확인해 주세요.` |
| 유효성 체크 완료 | `휴대폰번호 인증이 완료되었습니다.` |

##### B4 SMS 인증 흐름

```
[인증번호 발송] 버튼 클릭
    ↓
연락처(휴대폰)* 010-1111-2222  [인증번호 재발송 B4-1]
인증번호 입력 [ __________ ]  남은시간 02:33  [인증번호 확인 B4-2]
    ↓
인증 성공 → [인증 완료] 버튼 비활성화, 입력항목 수정 불가
```

**[B4-1] [인증번호 재발송] 버튼**
- **활성화 조건:** 이름, 신청인과의 관계, 주민등록번호, 연락처(휴대폰) 모두 입력/선택시
- 재발송 이후 남은 시간 다시 카운팅 (3분 기준, 처음부터)

**[B4-2] [인증번호 확인] 버튼**
- **활성화 조건:** 인증번호 입력시
- **인증 완료 이후:** 해당 영역 비노출
- 남은 시간 카운팅: 3분 기준으로 카운팅 시작
- **유효성 체크:** 휴대폰번호 SMS 인증번호 확인
- **개발 참고:** SMS 인증 — M전산 모듈 이용

##### B4 인증 메시지

| 상황 | 메시지 |
|------|--------|
| 인증번호 발송 처리 후 | `인증번호가 발송되었습니다.` |
| 인증번호 미발송된 경우 | `[인증번호 발송] 버튼을 클릭하시면, 인증번호가 등록된 휴대폰으로 발송됩니다.` |

**[1] 완료 메시지**

**[B1] [열람] 버튼**
- 클릭시 신청서 열람 팝업 화면 표시

**[B2] [발송] 버튼 - 신청서 교부**
- 발송 번호 자동 세팅: 교부 휴대폰번호 자동 세팅 → 해지 후 연락처 번호
- [발송] 버튼 클릭시 가입자 연락처 휴대폰 번호로도 강제 발송 처리
- **활성화 조건:** 휴대폰번호 입력시
- 클릭시 입력된 휴대폰번호 유효성 체크
- 입력된 휴대폰번호로 톡 발송 처리
  - SMS/카톡 내에 열람 URL 클릭시 솔루션사 화면 표시 후 해당 화면에서 패스워드 입력 후 이미징 열람
  - 솔루션 제공 API로 발송 처리
- **개발 참고:** 포시에스 신청서 교부 API

**[B3] [발송] 버튼 - kt M모바일 앱 안내**
- **활성화 조건:** 휴대폰번호 입력시

  > https://play.google.com/store/apps/details?id=kt.co.ktmmobile&hl=ko
- **개발 참고:** M모바일 SMS 발송 모듈 호출
