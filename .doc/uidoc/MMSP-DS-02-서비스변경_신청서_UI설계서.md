## 2. 신청서 작성 흐름

### 2.1 서비스변경 신청서 작성 (1/2)

**[고객 단계]**

- **고객유형**: 내국인 / 미성년자(19세 미만) / 외국인(Foreigner) / 외국인(Foreigner) 미성년자 / 법인 / 공공기관
- **방문고객**: 본인 / 대리인
  - 법인/공공기관만 선택
  - 성인 → `본인` 고정
  - 미성년자 → `대리인` 고정
- **가입자 정보**: 이름, 생년월일/법인등록번호
- **서비스변경 휴대폰번호** [인증]
- **법정대리인 정보** *(고객유형 '미성년자'인 경우만)*: 이름, 신청인과 관계, 주민등록번호, 휴대폰번호, 안내 및 동의사항
- **대리인 위임 정보** *(방문고객 '대리인'인 경우만)*: 이름, 생년월일, 신청인과 관계, 연락처
- **연락처 정보**: 연락처, 이메일주소, 주소
- **서비스 변경 선택**: 무선데이터차단 서비스 / 정보료 상한금액 설정·변경 / 부가서비스 신청·변경 / 요금제 변경 / 번호변경 / 분실복구·일시정지해제 신청 / 단말보험 가입 / 핸드폰 기변·일반기변 / USIM 변경 / 데이터쉐어링 가입·해지 / 아무나 SOLO 결합

**[상품 단계]**

- 무선데이터차단 서비스 [체크]
- 정보료 상한금액 설정/변경 [체크]
- 부가서비스 신청/변경 [체크] → 카테고리에서 선택 → 부가정보 설정
- 요금제 변경: 예약(익월1일)/즉시변경 [체크] → 카테고리에서 선택
- 번호변경 [체크]
- 분실복구/일시정지해제 신청 [체크]
- 단말보험 가입: 안심보험, 약관 동의 [체크] → 카테고리에서 선택
- 핸드폰 기변/일반 기변 [체크]

### 2.2 서비스변경 신청서 작성 (2/2)

**[상품 단계 계속]**

- USIM 변경 [체크]
- 데이터쉐어링 가입/해지: 가입 및 약관 동의, 해지 [체크]
- 아무나 SOLO 결합: 약관 동의 [체크]
- 메모

**[동의 단계]**

- 신청서 확인: 가입정보 확인, 서명(가입자(대리인) 서명)
- 팝업 화면으로 최종 정보 확인
- 신청서 처리: 신청서 등록, 리포트 이미징 처리, 이미징 시스템 리포트 전송, 서비스변경 M플랫폼 요청 *(별도 작성)*

**[완료 단계]**

- 신청서 교부: 열람, 발송 *(신규/변경 신청서와 동일)*

### 2.3 서비스변경 신청서 처리 흐름

```
스마트 서식지
  → 신청서 작성 완료
  → 신청서 등록 (신청서)
  → 이미징 시스템: 리포트 이미징 처리 (리포트 서식)
  → 리포트 이미징 전송 → KOS
  → M플랫폼: 서비스변경 요청

※ K-NOTE 에 이미징 자료 전송 불필요
```

**B4 버튼 (SMS 인증) 흐름**

```
1. 이름/관계/주민등록번호/연락처 입력 → [인증번호 발송] 활성화

4. 인증 성공 → [인증 완료]로 버튼명 변경, 비활성화
5. 재발송 필요 시 → [인증번호 재발송] 클릭 → 3분 재시작
```

> ※ 신규/변경 신청서와 동일

**발송 번호 자동 세팅 규칙**

| 고객유형 | 교부 휴대폰번호 |
|----------|----------------|
| 성인/법인/공공기관 | 개통한 휴대폰 번호 |
| 미성년자 | 법정대리인 휴대폰 번호 |

**B2 발송 처리**
- [발송] 버튼 클릭 시: 가입자 연락처 휴대폰번호로도 강제 발송 처리

- SMS/카톡 내 열람 URL 클릭 시: 솔루션사 화면 표시 → 패스워드 입력 → 이미징 열람
- **개발**: 포시에스 신청서 교부 API

**B3 앱 다운로드 안내 발송**
- 활성화 조건: 휴대폰번호 입력 시
- **개발**: M모바일 SMS 발송 모듈 호출

### 6.3 동시 변경 불가 업무 체크 로직

아래 3개 업무 중 1개를 선택하면 나머지 2개는 **비활성화(disabled)** 처리한다.

| 업무 | 항목 키 | 비고 |
|------|---------|------|
| 번호변경 | `numberChange` | B1 영역 |
| 분실복구·일시정지해제 | `lostRecover` | B3 영역 |
| 요금제 변경 | `planChange` | B5 영역 |

프론트엔드 규칙:
- 3개 중 하나라도 `true`이면 나머지 두 항목의 checkbox·영역을 `disabled`로 설정
- 취소(체크 해제) 시 나머지 영역 활성화 복원

백엔드 동시처리체크: `POST /api/v1/service-change/concurrent-check` — 이미 처리 중인 신청건 존재 여부 조회
