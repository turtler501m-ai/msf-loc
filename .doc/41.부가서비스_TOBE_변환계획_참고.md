## 1. ASIS 현황 진단

현재 `MsfRegSvcController` / `MsfRegSvcService` / `MsfRegSvcServiceImpl`은
**ASIS 소스를 이름만 변경한 상태**로 아래 문제가 있다.

| 문제 | 파일 | 내용 |
|------|------|------|
| MVC 방식 잔존 | Controller | `@Controller`, `@RequestMapping("*.do")`, `Model`, `HttpSession` |
| 세션 의존 | Controller/Service | `SessionUtils.getUserCookieBean()`, `MyPageSearchDto`로 ncn/ctn/custId 수신 |

| Map 반환 | Controller | `Map<String, Object>` 반환으로 타입 불명확 |

## 3. 변환 방식 — 기존 파일 직접 변환

신규 파일 생성 없이 기존 3개 파일을 직접 TOBE로 변환한다.

```
[변환 대상 — 기존 파일 직접 수정]
  MsfRegSvcController.java     ← @Controller → @RestController, *.do → /api/v1/addition/*
  MsfRegSvcService.java        ← 메서드 시그니처 TOBE 방식으로 교체
  MsfRegSvcServiceImpl.java    ← X20→X97, X21→Y25, Map→VO, 세션→ReqDto

[신규 생성 — DTO만]
  dto/AdditionReqDto.java          ← 공통 요청 (ncn/ctn/custId)
  dto/AdditionApplyReqDto.java     ← 신청/해지 요청 (+ soc, ftrNewParam, prodHstSeq)
  dto/AdditionMyListResVO.java     ← 이용중 목록 응답
  dto/AdditionAvailableResVO.java  ← 가입가능 목록 응답
  dto/AdditionApplyResVO.java      ← 신청/해지 응답

[수정 — MplatFormService]
  MplatFormService.java            ← Y25 메서드 신규 추가
```

## 6. MsfRegSvcServiceImpl 변환 상세

### 6-1. getMyAddSvcList (이용중 목록, X97)

ASIS `selectmyAddSvcList()` 로직을 그대로 이식, 파라미터만 `AdditionReqDto`로 교체.

```
[1] X97 호출: getAddSvcInfoParamDto(req.getNcn(), req.getCtn(), req.getCustId())
[2] "PL249Q800" 더미 SOC 제거
[3] 각 SOC → getMspRateMst() → canCmnt, onlineCanYn 세팅
    → Constants.REG_SVC_CD_4 → 강제 onlineCanYn = "N"
[4] getMpSocListByDiv() → 일반(G)/로밍(R) 분류 (기존 private 메서드 활용)
[5] AdditionMyListResVO 반환
```

### 6-2. getAvailableAddSvcList (가입가능 목록, X97)

ASIS `selectAddSvcInfoDto()` X20 → X97 교체.

```
[1] X97 호출: getAddSvcInfoParamDto() → 이용중 SOC 목록 추출 (useSocList)
[2] RegSvcDao.selectRegService() → DB 부가서비스 관리 목록
[3] useSocList 기준 useYn 매핑
[4] "PL249Q800" 더미 SOC 필터링
[5] getMcpRegServiceListByDiv() → 일반(G)/로밍(R) 분류
[6] AdditionAvailableResVO 반환
```

### 6-3. cancelAddSvc (해지, X38)

ASIS `moscRegSvcCanChg()` 로직 그대로 이식, 파라미터 교체.

```
[1] getMspRateMst(req.getSoc()) → onlineCanYn 확인
    → "Y" 아닌 경우: AdditionApplyResVO(success=false, NO_ONLINE_CAN_CHANGE_ADD)
    → null인 경우: AdditionApplyResVO(success=false, NO_EXSIST_RATE)
[2] prodHstSeq 있으면 → moscRegSvcCanChgSeq(ncn, ctn, custId, soc, prodHstSeq)
    없으면 → moscRegSvcCanChg(ncn, ctn, custId, soc)
[3] AdditionApplyResVO(success=true) 반환
```

### 6-4. regAddSvc (신청, Y25)

ASIS `regSvcChgAjax` 로직에서 X21 → Y25 교체, 세션 제거.

## 7. MplatFormService — Y25 추가

```java
// Y25 - 상품변경처리(multi), X21 대체
public MpRegSvcChgVO regSvcChgY25(String ncn, String ctn, String custId,
                                   String soc, String ftrNewParam) throws SocketTimeoutException {
    MpRegSvcChgVO vo = new MpRegSvcChgVO();
    HashMap<String, String> param = getParamMap(ncn, ctn, custId, "Y25");
    param.put("soc", StringUtil.NVL(soc, ""));
    param.put("ftrNewParam", StringUtil.NVL(ftrNewParam, ""));
    if ("LOCAL".equals(serverLocation)) {
        getVo(25, vo);
    } else {
        mplatFormServerAdapter.callService(param, vo, 30000);
    }
    return vo;
}
```
