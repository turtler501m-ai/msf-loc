## 1. 개요

대리점 담당자가 고객의 서비스 해지 신청서를 작성·접수하는 화면.
3단계 스텝(고객 → 상품 → 동의)으로 구성되며, 최종 제출 시 백엔드를 통해
M포탈 테이블(`MSF_CANCEL_REQUEST`)에 적재된다.

## 3. 공통 스텝 프레임워크 (`MsfFormView.vue`)

### 3.1 레이아웃

```
┌──────────────────────────────────────────────┐
│ MsfStepIndicator (좌측 스텝 표시)              │
│ ┌────────────────────────────────────────────┐│
│ │ MsfTitleBar (서비스 해지)                   ││
│ │                                            ││
│ │ [Step 0] TerminationCustomer (항상 노출)    ││
│ │ [Step 1] TerminationProduct  (다음 클릭 후) ││
│ │ [Step 2] TerminationAgreement (마지막)      ││
│ │                                            ││
│ │ ┌──────────────────────────────────────┐   ││
│ │ │  [작성완료] or [다음]                │   ││
│ │ └──────────────────────────────────────┘   ││
│ └────────────────────────────────────────────┘│
└──────────────────────────────────────────────┘
```

### 3.2 스텝 진행 로직

| 이벤트 | 동작 |
|--------|------|
| 자식 `@complete(true)` 발생 | 현재 스텝 `isStepComplete = true` → [다음] 버튼 활성화 |
| [다음] 버튼 클릭 | `stepRefs[i].save()` 호출 → 성공 시 `currentStepIndex++` |
| 마지막 스텝에서 [작성완료] | `stepRefs[i].save()` → 성공 시 신청 완료 처리 |
| 스텝 전환 시 | 해당 스텝 상단으로 자동 스크롤 |

### 3.3 임시저장 복원

`TerminationCustomer`만 `data(code)` 메서드 노출 (1단계가 임시저장 복원 진입점):

```js
// 반환값: null/0=임시저장없음, 1=고객, 2=상품, 3=동의
const data = async (code) => {
  if (code) { return '1' }  // 임시저장 데이터 조회 후 단계 반환 (구현 필요)
  return '0'
}
```

## 8. 백엔드 연계 구조 (ASIS 기준)

### 8.1 현재 백엔드 API 목록

| API URL | 메서드 | 설명 |
|---------|--------|------|
| `/mypage/cancelConsult.do` | GET | 해지상담 페이지 (ASIS 레거시, MVC) |
| `/mypage/cancelConsultAjax.do` | POST | 해지상담 신청 처리 (ASIS 레거시) |

### 8.2 ASIS 신청 처리 흐름 (`cancelConsultAjax.do`)

```
1. 미성년자 확인 (나이 < 19 → 거부)
2. 계약번호 검증 (cntrList 조회 후 일치 확인)
3. 중복 접수 확인 (countCancelConsult)
4. NICE 본인인증 검증 (현재 주석 처리)
5. cancelConsultRequest() @Transactional
   ├ insertNmcpCustReqMst() → MSF_CUST_REQUEST_MST (CUST_REQ_SEQ: YYYYMMDD+5자리)
   └ insertCancelConsult()  → MSF_CANCEL_REQUEST (PROC_CD='RC')
6. prodSendScan() → SCAN 서버 전송 + SCAN_ID 동기화
```

### 8.3 CancelConsultDto 주요 필드 매핑

| DTO 필드 | DB 컬럼 | 화면 입력 소스 |
|----------|---------|--------------|
| `custReqSeq` | `CUST_REQ_SEQ` | 시퀀스 자동 생성 |
| `cstmrName` | `CSTMR_NAME` | 세션(userName) |
| `contractNum` | `CONTRACT_NUM` | 계약번호 선택 또는 직접입력 |
| `cancelMobileNo` | `CANCEL_MOBILE_NO` | TerminationCustomer.cancelPhone |
| `receiveMobileNo` | `RECEIVE_MOBILE_NO` | TerminationCustomer.afterTel |
| `onlineAuthType` | `ONLINE_AUTH_TYPE` | NICE 본인인증 방법 |
| `onlineAuthInfo` | `ONLINE_AUTH_INFO` | ReqNo + ResNo |
| `cstmrNativeRrn` | `CSTMR_NATIVE_RRN` | ACE256 암호화 |
| `survey1Cd/Text` | `SURVEY1_CD/TEXT` | TerminationProduct.isActive (해지 후 이동) |
| `memo` | `MEMO` | TerminationProduct.memo |
| `procCd` | `PROC_CD` | 초기값: 'RC' (접수) |
| `regstId` | `REGST_ID` | 세션(userId) |
| `rip` | `RIP` | 클라이언트 IP |

### 8.4 M플랫폼 연동 (X18)

해지 정산 정보 조회:
- M플랫폼 코드: **X18** (잔여요금·위약금 조회)
- 현재 TerminationProduct에서 수동 입력 → 실제 구현 시 X18 조회 결과 자동 바인딩 필요

### 11.10 J — 해지 신청 최종 접수

- **시점**: STEP 3 [작성완료] 클릭 → `TerminationAgreement.save()`
- **URL**: `POST /api/v1/cancel/apply`
- **요청**: 전체 3스텝 수집 데이터 통합 전송

```json
{
  "tempCode": "xxxxxxxx",
  "agreeCheck1": true,

  "idCardScan": "idCardScan1",
  "userName": "홍길동",
  "userBirthDate": "19900101",
  "userGender": "userGender1",
  "contractNum": "123456789",
  "cancelMobileNo": "01012345678",
  "afterTel": "01087654321",
  "postMethod": "postMethod2",
  "agencyCode": "agencyName1",
  "openingDate": "20230105",

  // 상품 정보
  "isActive": "isActive1",
  "usageFee": 15000,
  "penaltyFee": 50000,
  "finalAmount": 65000,
  "remainPeriod": 12,
  "remainAmount": 360000,
  "memo": "메모내용"
}
```

- **응답**: `{ "requestKey": "20260410XXXXX", "resultCode": "00000" }`
- **백엔드 처리 순서**:

```
1. 중복 접수 확인 (MSF_CANCEL_REQUEST WHERE PROC_CD != 'BK')
2. MSF_CUST_REQUEST_MST INSERT (시퀀스 채번)
3. MSF_CANCEL_REQUEST INSERT (PROC_CD='RC')  ← @Transactional
4. M포탈 msf-register API 호출 (비동기 처리 권장)
   → 성공: MSF_CANCEL_REQUEST.RES_NO UPDATE
5. SCAN 서버 전송 (FormSend reqType=CC)
   → 성공: MSF_CANCEL_REQUEST.SCAN_ID UPDATE
         M포탈 update-scan API 호출
6. requestKey 반환
```

- **백엔드 현황**: `MsfCancelConsultController.cancelConsultAjax()` 포팅 필요
  - Mapper XML `insertNmcpCustReqMst`, `insertCancelConsult` 주석 해제 + 테이블명 MSF_* 교체
