## [5007-01] 서비스변경 화면 진입

**ASIS:** `RegSvcController.regServiceView` — `/m/mypage/regServiceView.do`
**TOBE API:** 별도 API 불필요 (가입자조회는 `POST /api/v1/{domain}/join-info` 기구현)

```
[1] 가입자 조회 (비회원)
    - mypageService.selectCntrList (비회원 버전)
      [TOBE] JoinInfoSvc.joinInfo() 기구현 → 동일 역할 수행
    ※ 제외: selectCntrList (회원 로직 불필요)

[2] 유저타입 체크
    - mypageService.checkUserType
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용
      → 가입자 조회 결과(contractNum, custId)로 판별 (법인/개인 등)

[3] 나이 계산
    - NmcpServiceUtils.getBirthDateToAmericanAge
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용
      → 생년월일 → 만 나이 변환 로직 (미성년자 여부 판별용)

[4] 마스킹 해제 로그 적재
    - maskingSvc.insertMaskingReleaseHist
      [TOBE] SQL Copy → MCP_MASKING_RELEASE_HIST → MSF_MASK_RELES_RQT_HST
    ※ 공통 미구현 항목 — 31번 문서 §1-4 참조
```

```

      [TOBE] JoinInfoSvc.joinInfo() 기구현
```

## [5007-06] 부가서비스 신청 처리

> ⚠️ 처리 순서: 기존 부가서비스 해지(X38) → 신규 부가서비스 신청(Y25)

```
      [TOBE] 본인인증 불필요
             → 스마트신청서 인증마다 로그 적재로 개발 (공통모듈 — 미구현)

    - fCommonSvc.getMspRateMst
        -> [M포탈-M전산 API] /common/mspRateMst

      [TOBE] X38 순차 호출 (기구현 X38과 동일, 순서 파라미터 차이)

[7] 부가서비스 신청 ★ X21 → Y25 대체

        -> mPlatFormService.regSvcChg
```
