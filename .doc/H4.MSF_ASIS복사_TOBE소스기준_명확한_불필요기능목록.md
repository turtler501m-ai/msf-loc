# H4. MSF ASIS 복사 TOBE 소스 기준 명확한 불필요 기능목록

> **참조 문서:**

> - `.doc/H1.MSF_ASIS_TOBE_컨트롤러_서비스_기능목록.md`
>
> **작성일:** 2026-04-13
> **목적:** ASIS에서 복사해온 TOBE **Controller/Service** 소스 중 현재 MSF 전환 범위에서 **명확하게 유지 불필요한 기능**을 분리 정리

---

## 1. 판단 기준

- H1에서 `🚫 제외` 또는 H2에서 `ASIS 전용/TOBE 범위 외`로 확인된 기능만 본 문서의 “명확한 불필요” 대상으로 본다
- 팝업 전용 함수: JSP 팝업 렌더링 용도이며 TOBE에서 독립 API 필요 없음
- 배치/탭목록/ASIS 전용 XML 응답: 현재 MSF 전환 범위 밖이거나 프론트/공통 API로 대체
- KT 결합 계열: 현재 TOBE 범위는 `아무나SOLO`만 포함, KT 결합 흐름은 제외
- ASIS 고유 부가/운영 함수: 현행 TOBE 설계에서 직접 대응 API가 없고 유지가치가 없음
- 반대로 H1에서 `✅ 완료` 또는 `⚠️ 확인필요`로 사용 중인 화면/컨트롤러 메서드는 본 문서에서 제외한다

> 기준 근거: H2의 “TOBE 불필요 판단 기준”, 컨트롤러별 실제 public 메서드 분류, H1의 `🚫 제외` 항목.

## 3. 삭제 우선순위 정리

### 3-1. 즉시 삭제 가능

| 분류 | 함수 |
|------|------|
| 팝업 | `addSvcViewPop`, `doRegServicePop`, `suspenPosHisPop` |
| ASIS 전용 Controller | `getContRateAdditionAjax`, `getRateAdsvcGdncDtlXml`, `combineSoloType`, `getCombiSvcList`, `domyCntrListAjax`, `custRequestInsrSaveApi` |
| ASIS 탭목록 | `farPriceCtgTab1Ajax`, `getFarPricePlanListAjax2`, `getFarPricePlanListAjax3`, `farPriceContent` |
| KT 결합 제외 | `combineWireless`, `combineWirelessComplete`, `checkCombine`, `childCertify`, `childCertifyKt`, `childCertifyMore`, `childVerifyAjax`, `regCombine`, `regCombineKt`, `eSimWatchInfo` |
| 범위 외 화면/서비스 | `reqCallList`, `reqJoinForm`, `reqUsimPuk`, `insertCustRequestCallList`, `insertCustRequestJoinForm`, `getUsimPukByMP` |

### 3-2. 전환 제외 보관 가능

| 분류 | 함수 | 보관 이유 |
|------|------|----------|
| 배치 | `resChangeAddPrd` | 운영 이관 여부가 확정되지 않음 |
| 기타 제외 | `reqFormDownloadCountUpdateAjax` | 업무 중요도는 낮지만 운영 참고 가능 |

### 3-3. 본 문서에서 제외한 항목

- `regServiceView`, `numberView`, `reSpnsrPlcyDc`, `suspendCnl`, `lostCnlView`, `doFarPricePlanView`, `combineSelf`, `doMySharingView`, `dataSharingStep1~4`, `reqInsr`, `myNameChg`, `cancelConsult`
- 위 함수들은 JSP 화면 진입 성격이 있으나 H1에서 `✅ 완료` 또는 `⚠️ 확인필요`로 관리되고 있어 “명확한 불필요”로 분류하지 않았다.
- 향후 프론트 완전 이관 후 정리 후보가 될 수는 있으나, 그 판단은 H4 범위를 벗어나므로 별도 문서에서 다룬다.

### 3-4. 소스는 남겨도 되지만 ASIS 복사 재사용은 금지

- `custRequestScanService.prodSendScan` 계열 원형 로직
- 이유: 기능 자체는 필요하지만 ASIS 구현체를 그대로 쓰는 것은 불가
- 조치: 포시에스 기준 새 구현으로 대체 후 ASIS 복사 로직 폐기

## 4. H2/H1 근거 요약

| 근거 문서 | 핵심 내용 |
|----------|----------|
| `H2` | 화면 진입 함수 일반론은 있으나, H4에는 H1에서 실제 사용 중인 항목을 포함하지 않도록 보정 |
| `H2` | 팝업/배치/탭목록/KT 결합 계열은 TOBE 불필요로 명시 |
| `H1` | `addSvcViewPop`, `doRegServicePop`, `combineSoloType`, `reqFormDownloadCountUpdateAjax` 등은 `🚫 제외` 처리 |
| `H1` | `regServiceView`, `numberView`, `reSpnsrPlcyDc`, `suspendCnl`, `lostCnlView`, `doFarPricePlanView`, `combineSelf`, `doMySharingView`, `reqInsr`, `myNameChg`, `cancelConsult` 등은 사용 중 항목 |
| `H2` | `MyNameChgController.userInfoChk`는 ASIS 실소스에 없는 TOBE 보완항목으로 확인 |

## 5. 권고사항

1. TOBE 프로젝트에서 ASIS 복사 소스 유지 시, H1에서 `🚫 제외`로 확정된 `팝업/탭목록/ASIS 전용 함수`부터 우선 제거 후보로 분리한다.
2. KT 결합 계열은 현재 범위 외이므로 패키지 단위로 비활성 또는 삭제 검토한다.
3. `CustRequestService`의 고객요청/가입신청서/USIM PUK 계열은 실제 요구사항 재등장 전까지 전환 대상에서 제외한다.
4. SCAN/XML 계열은 “불필요 기능”이 아니라 “ASIS 복사본 재사용 불가 기능”으로 별도 관리한다.
