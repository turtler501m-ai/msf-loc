## 8001-01

**JSP/JS**: `/rcp/statsMgmt/statsReqNameChgInfo.jsp`

### 주요내용

명의변경 신청 목록 조회·상세·상태처리 및 **안면인증(FATH) 연동** 화면.
스마트서식지 App 데이터를 M포탈 명의변경 테이블(`NMCP_CUST_REQUEST_*`)에 동기화하고,
M전산 OSST를 통한 명의변경 사전체크(MC0) 및 안면인증(FATH FS9/FT1) 후처리를 수행한다.

| 구분 | 내용 |
|------|------|
| 주요 Controller | `StatsMgmtController`, `NstepCallController`, `FathController` |
| Mapper XML | `StatsMgmt_SQL.xml`, `Fath_SQL.xml` |
| 명의변경 테이블 | `NMCP_CUST_REQUEST_MST`, `NMCP_CUST_REQUEST_NAME_CHG`, `NMCP_CUST_REQ_NAME_CHG_AGENT` |
| 안면인증 테이블 | `NMCP_FATH_RESLT_PUSH`, `NMCP_FATH_SELF_URL`, `NMCP_CD_DTL` |
| EMV 스캔 테이블 | `EMV_SCAN_FILE` |
| TO-BE 영향도 | 명의변경·안면인증 테이블 → MSF 테이블 현행화 처리 필요 (기준 미확정) |

### 처리순서

```
1. 명의변경 신청 목록 조회
   getNameChgList.json
     statsMgmtMapper.getNameChgList  →  NMCP_CUST_REQUEST_MST + NMCP_CUST_REQUEST_NAME_CHG
                                         + MSP_JUO_SUB_INFO + MSP_REQUEST_DTL

2. 명의변경 신청 상세 조회
   getNameChgDtl.json
     statsMgmtMapper.getNameChgDtl   →  + NMCP_CUST_REQ_NAME_CHG_AGENT (법정대리인)

3. 엑셀 다운로드
   getNameChgListExcel.json
     statsMgmtMapper.getNameListExcel  →  동일 테이블 조회

4. 명의변경 상태 처리 + 서식지 합본
   updateName.json
     statsMgmtMapper.getOldScanId       →  MSP_JUO_SUB_INFO (영향도 없음)
     statsMgmtMapper.getFileNum         →  EMV_SCAN_FILE@DL_MCP  SELECT
     statsMgmtMapper.getNewScanIdNm     →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  SELECT
     statsMgmtMapper.insertEmvFile      →  EMV_SCAN_FILE@DL_MCP  INSERT
     statsMgmtMapper.updateProcCdNm     →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  UPDATE

5. 타 신청건 복사
   copyMcnRequest.json
     statsMgmtMapper.getNameChgState    →  MSP_JUO_SUB_INFO + MSP_REQUEST_DTL (영향도 없음)
     statsMgmtMapper.copyNameChgMst     →  NMCP_CUST_REQUEST_MST@DL_MCP  INSERT
     statsMgmtMapper.copyNameChgDtl     →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  INSERT
     statsMgmtMapper.updateProcCdNm     →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  UPDATE

6. OSST 명의변경 사전체크 (MC0)
   osstServiceCall.json
     fathMapper.getNameChgInfo          →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  SELECT
     fathMapper.updateNameChgInfoFathSkip → NMCP_CUST_REQUEST_NAME_CHG@DL_MCP UPDATE

7. 안면인증 결과 수신 (FATH FS9)
   reqFathTxnRetv.json
     fathMapper.selectFathResltPush         →  NMCP_FATH_RESLT_PUSH@DL_MCP  SELECT
     fathMapper.selectFathInfo              →  MCP_REQUEST + MCP_REQUEST_CSTMR + MCP_REQUEST_AGENT
     fathMapper.selectFathInfoMcn           →  NMCP_CUST_REQUEST_NAME_CHG + NMCP_CUST_REQ_NAME_CHG_AGENT
     fathMapper.updateNameChgSelfCertInfoMinor → NMCP_CUST_REQ_NAME_CHG_AGENT@DL_MCP  UPDATE
     fathMapper.updateNameChgSelfCertInfo      → NMCP_CUST_REQUEST_NAME_CHG@DL_MCP    UPDATE

8. 셀프 안면인증 URL 발급
   insertFathSelfUrl.json
     fathMapper.selectFathSelfUrl       →  NMCP_FATH_SELF_URL@DL_MCP  SELECT
     fathMapper.insertFathSelfUrl       →  NMCP_FATH_SELF_URL@DL_MCP  INSERT
     fathMapper.updateNameChgFathMobile →  NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  UPDATE

9. 안면인증 SKIP (FT1)
   requestCustFathTxnSkip.json
     fathMapper.isEnabledFT1            →  NMCP_CD_DTL@DL_MCP  SELECT
     fathMapper.updateNameChgInfoFathSkip → NMCP_CUST_REQUEST_NAME_CHG@DL_MCP  UPDATE
```

### 처리요약

1) 명의변경 신청 목록·상세를 NMCP_CUST_REQUEST_MST + NMCP_CUST_REQUEST_NAME_CHG 에서 조회하며, 법정대리인은 NMCP_CUST_REQ_NAME_CHG_AGENT 추가 조인으로 처리.
2) 상태 처리(updateName)는 EMV 스캔 파일을 합본하고 NMCP_CUST_REQUEST_NAME_CHG 상태코드를 UPDATE.
3) 타 신청건 복사(copyMcnRequest)는 MST·NAME_CHG 행을 INSERT 후 상태코드를 갱신하며, 기준 조회 테이블(MSP_JUO_SUB_INFO)은 영향도 없음.
4) OSST MC0 사전체크 후 안면인증 결과(FS9)는 NMCP_FATH_RESLT_PUSH 조회 → NAME_CHG + NAME_CHG_AGENT UPDATE 순으로 처리.
5) 셀프 안면인증은 NMCP_FATH_SELF_URL INSERT 후 NAME_CHG를 UPDATE하고, SKIP(FT1)은 NMCP_CD_DTL 조회로 가능 여부 확인 후 NAME_CHG를 UPDATE.

ㅇ 명의변경 신청 목록 조회
- NMCP_CUST_REQUEST_MST + NMCP_CUST_REQUEST_NAME_CHG
  ㅇ 명의변경 신청 상세 조회
- NMCP_CUST_REQ_NAME_CHG_AGENT (법정대리인)
  ㅇ 명의변경 상태 처리 + 서식지 합본
- NMCP_CUST_REQUEST_NAME_CHG 상태코드 UPDATE
  ㅇ 타 신청건 복사
  ㅇ OSST 명의변경 사전체크 (MC0)
  ㅇ 안면인증 결과 수신 (FATH FS9)
  ㅇ 셀프 안면인증 URL 발급
  ㅇ 안면인증 SKIP (FT1)

## 8001-03

**JSP/JS**: `/rcp/canCustMgmt/msp_rcp_bs_0003.jsp`

### 주요내용

해지상담 신청 고객 목록 조회·상태 처리 및 **서식지 합본(EMV 스캔)** 화면.
스마트서식지 App 해지 데이터를 M포탈 해지상담 테이블(`MCP_CANCEL_REQUEST@DL_MCP`)에 동기화하고,
EMV 스캔 파일 합본 후 상태 코드를 갱신한다.

| 구분 | 내용 |
|------|------|
| 주요 Controller | `CanCustController` |
| Mapper XML | `CanCustMgmt_SQL.xml` |
| 핵심 테이블 | `MCP_CANCEL_REQUEST@DL_MCP` |
| EMV 스캔 테이블 | `EMV_SCAN_FILE@DL_MCP`, `EMV_SCAN_MST@DL_MCP` |
| TO-BE 영향도 | 스마트서식지 해지 데이터 → `MCP_CANCEL_REQUEST` 현행화 처리 필요 (기준 미확정) |

### 처리순서

```
1. 해지상담 신청 고객 목록 조회
   selectCanCslList.do
     canCustMapper.selectCanCslList  →  MCP_CANCEL_REQUEST@DL_MCP + MSP_JUO_SUB_INFO + MSP_REQUEST_DTL

2. 목록 엑셀 다운로드
   selectCanCslListByExcel.json
     canCustMapper.selectCanCslListByExcel  →  MCP_CANCEL_REQUEST@DL_MCP + MSP_JUO_SUB_INFO + MSP_REQUEST_DTL

3. 해지상담 상태값 조회
   getCustStatus.json
     canCustMapper.getCustStatus  →  MCP_CANCEL_REQUEST@DL_MCP  SELECT

4. 해지상담 상태 처리 + 서식지 합본
   updateCanCsl.json
     canCustMapper.getCanCslFileNum       →  EMV_SCAN_FILE@DL_MCP  SELECT (MAX FILE_NUMBER)
     canCustMapper.insertCanCslEmvFile    →  EMV_SCAN_FILE@DL_MCP  INSERT (합본)
     canCustMapper.updateCanCslEmvFileMst →  EMV_SCAN_MST@DL_MCP   UPDATE
     canCustMapper.updateCanCslProcCd     →  MCP_CANCEL_REQUEST@DL_MCP  UPDATE (상태코드)
```

### 처리요약

1) 해지상담 신청 목록을 MCP_CANCEL_REQUEST@DL_MCP 기준으로 조회하며, 스마트서식지 해지 데이터를 해당 테이블에 동기화.
2) 상태 처리(updateCanCsl)는 EMV_SCAN_FILE 합본 INSERT → EMV_SCAN_MST UPDATE → MCP_CANCEL_REQUEST 상태코드 UPDATE 순으로 진행.


### 관련테이블

| 테이블명 (TOBE) | ASIS 원본 | DML | 용도 |
|----------------|-----------|-----|------|
| `MSF_REQUEST_CANCEL` | MCP_CANCEL_REQUEST@DL_MCP | SELECT / UPDATE | 해지상담 신청 목록·상태코드 갱신 |
| `MSF_EMV_SCAN_FILE` | EMV_SCAN_FILE@DL_MCP | SELECT / INSERT | EMV 스캔 파일 합본 |
| `MSF_EMV_SCAN_MST` | EMV_SCAN_MST@DL_MCP | UPDATE | EMV 스캔 마스터 갱신 |
