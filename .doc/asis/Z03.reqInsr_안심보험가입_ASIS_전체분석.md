# 22. ASIS 안심보험 가입신청 전체 분석 — reqInsr.do

> 작성일: 2026-03-19 / 최종수정: 2026-03-20
> 목적: ktmmobile.com/mypage/reqInsr.do 화면의 안심보험 가입신청 ASIS 백엔드 전체 분석 → MSF TOBE 구현 참조

---

## 1. 전체 처리 흐름

```
[1] 사용자 접근 GET /mypage/reqInsr.do
        ↓
[2] CustRequestController.reqInsr()
    ├─ 세션 · 계약정보 확인
    ├─ getInsrInfo() API 호출 → 현재 보험가입상태 조회
    └─ insrViewType 결정 (A/B/C/D/E/F/ING)
        ↓
[3] reqInsr.jsp 렌더링 (타입별 화면 분기)
        ↓
[4] 사용자: 보험상품 선택 + 본인인증(NICE) 완료
        ↓
[5] AJAX POST /mypage/custRequestAjax.do  (reqType=IS)
        ↓
[6] CustRequestController.custRequestAjax()
    ├─ NICE 인증 세션 검증 (certStep=4 필수)
    ├─ CUST_REQUEST_SEQ 채번 (yyyymmdd + 5자리 시퀀스)
    ├─ NMCP_CUST_REQUEST_MST INSERT
    ├─ NMCP_CUST_REQUEST_INSR INSERT
    ├─ RegSvcService.regSvcChg() → MplatFormService X21 (부가서비스 신청)
    ├─ SCAN 서버 서식지 생성 (WORK_ID=S0008, DOC_ID=E0008)
    └─ SMS 발송 (D/E/F 고객 대상, 템플릿 239/240)
        ↓
[7] JSON 응답: { result: "SUCCESS" / "FAIL4" }
```

#### insrViewType 결정 로직 (6가지 분기)

```
getInsrInfo() 응답값
  INSR_STAT_CD  : 현재 보험 상태 (0=미가입, 1=가입, 2=해지...)
  ORDER_EXIST   : 보험가입 진행중 여부 (Y/N)
  REQUEST_EXIST : 신청상태 여부 (Y/N)
  REQ_BUY_TYPE  : 구매유형 (MM=단말, UU=유심)

분기 결과:
  A   → 보험가입중 안내 (INSR_STAT_CD = 0 or 1)
  B   → 고객센터 안내 (미성년자 / 법인 / 신청상이 고객)
  C   → 신규단말 45일 이내 (REQ_BUY_TYPE=MM, 개통후 45일 이하)
  D   → 신규단말 45일 이후 (REQ_BUY_TYPE=MM, 개통후 45일 초과)
  E   → 유심 45일 이내    (REQ_BUY_TYPE=UU, 개통후 45일 이하)
  F   → 유심 45일 이후    (REQ_BUY_TYPE=UU, 개통후 45일 초과)
  ING → 보험가입 신청 진행중 (ORDER_EXIST=Y or REQUEST_EXIST=Y)
```

```java
// MplatFormService.regSvcChg() 라인 514~527
public MpRegSvcChgVO regSvcChg(String ncn, String ctn, String custId,
                               String soc, String ftrNewParam) throws SocketTimeoutException {
    MpRegSvcChgVO vo = new MpRegSvcChgVO();
    HashMap<String, String> param = getParamMap(ncn, ctn, custId, "X21");
    param.put("soc", StringUtil.NVL(soc, ""));              // 보험상품코드
    param.put("ftrNewParam", StringUtil.NVL(ftrNewParam, ""));
    if ("LOCAL".equals(serverLocation)) {
        getVo(21, vo);   // Mock 응답
    } else {
        mplatFormServerAdapter.callService(param, vo, 30000);
    }
    return vo;
}
```

### 5-5. 보험신청 데이터 조회 (서식지 생성용)

```sql
SELECT
    A.REQ_TYPE,
    A.CONTRACT_NUM,
    A.CSTMR_NAME,
    A.CSTMR_NATIVE_RRN,
    A.ONLINE_AUTH_INFO   AS INSR_AUTH_INFO,
    TO_CHAR(A.SYS_RDATE, 'YYYY') AS REQ_IN_DAY_YYYY,
    TO_CHAR(A.SYS_RDATE, 'MM')   AS REQ_IN_DAY_MM,
    TO_CHAR(A.SYS_RDATE, 'DD')   AS REQ_IN_DAY_DD,
    B.INSR_CD            AS INSR_PROD_CD
FROM NMCP_CUST_REQUEST_MST A
JOIN NMCP_CUST_REQUEST_INSR B ON A.CUST_REQ_SEQ = B.CUST_REQ_SEQ
WHERE A.CUST_REQ_SEQ = #{custReqSeq}
  AND A.REQ_TYPE = 'IS'
```

## 11. MSF TOBE 구현 현황 (2026-03-20 기준)

### 11-1. ASIS vs TOBE 대응표

| 항목 | ASIS | TOBE 구현 상태 |
|------|------|--------------|
| 보험상태 조회 | `custRequestService.getInsrInfo()` → mcp-api REST 호출 | `SvcChgInsrSvcImpl.getInsrInfo()` — DB링크(MSP DB) 직접 조회 ✅ |
| 화면 타입 분기 | Controller에서 insrViewType 결정 후 JSP 분기 | API 응답에 `insrViewType` 포함 → 프론트 분기 ✅ |
| Y24 사전체크 | ASIS에 없음 (reqInsr.do 플로우에 미포함) | 없음 — ASIS 동일 ✅ |
