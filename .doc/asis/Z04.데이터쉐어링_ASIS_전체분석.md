# 데이터쉐어링 ASIS 전체 분석

> 분석 대상: `mcp/mcp-portal-was/` 레거시 M포탈 + E1 ITO소스분석 문서
> 참조 문서: `.doc/reference/E1.스마트서식지-DS-08-ITO소스분석_v1_0_20260324.md` (§4001-03, §4001-07, §4001-08)
> 분석 일자: 2026-03-26

---

## 1. 개요

데이터쉐어링은 **2가지 기능** × **2가지 신청 경로**로 구성된다.

| 기능 | 설명 |
|------|------|
| **데이터쉐어링 (MyShareData)** | 개인 회선 간 데이터 공유 신청/해지 |
| **함께쓰기 (ReShareData)** | 주기회선 ↔ 받기회선 방식 결합 신청 |

데이터쉐어링 신청 경로는 다시 두 갈래다:
- **간편신청** (`saveDataSharingSimple`, 4001-03): 비회원(세션 기반) 1-Step 방식
- **단계별신청** (`saveDataSharing` + `saveDataSharingStep1`, 4001-07, 4001-08): 포탈/상담사 다단계 방식

MSP 폴더에는 해당 기능 없음 — MCP 전용.

## 5. [4001-03] saveDataSharingSimple — 간편신청 상세

**URL**: `/m/appform/saveDataSharingSimpleAjax.do` (모바일 전용)
**JSP**: `reqSharingView.jsp` / `reqSharingView.js`
**방식**: 비회원 1-Step 간편신청 (OSST 비동기 폴링 방식)

### 처리 흐름

```
1. 입력값 검증
   - contractNum, reqUsimSn 필수
   └─ McpCommonJsonException("0001") if 누락

2. 세션 기반 사용자 확인
   - 회원: SessionUtils.getUserCookieBean() → mypageService.selectCntrList(userId)
   - 비회원: SessionUtils.getNonmemberSharingInfo() → mypageService.selectCntrListNoLogin(out)
   └─ McpCommonJsonException("0003") if 비회원 세션 없음

3. 계약번호 본인 소유 검증
   - cntrList에서 contractNum 매칭 → isOwn=true, cntrMobileNo, certContractNum, customerId 추출
   └─ McpCommonJsonException("0004") if 불일치

4. 본인인증 세션 확인
   - SessionUtils.getNiceResCookieBean() != null
   └─ McpCommonJsonException("0005") if NICE 세션 없음

5. 청구계정번호 조회
   - mypageService.selectBanSel(contractNum)

6. 신청서 저장 (중복 방지: 세션 확인 후 최초 1회만)
   6-1. appformSvc.getCopyMcpRequest(appformReqDto)
        └─ SQL: AppformMapper.getCopyMcpRequest
           테이블: MCP_REQUEST, MCP_REQUEST_CSTMR, MCP_REQUEST_REQ,
                   MCP_REQUEST_SALEINFO, MCP_REQUEST_AGENT
           → 스마트신청서에 없으면 M포탈 테이블에서 조회
   6-2. 인증정보 주민등록번호 앞 6자리 생년월일 비교
        └─ McpCommonJsonException("0007") if 불일치
   6-3. AppformReqDto 필드 설정
        - operType=NAC3, serviceType=PO, socCode=KISOPMDSB
        - onOffType: 모바일=7, PC=5
        - prdtSctnCd=LTE, joinPrice=0
        - usimPrice=0, usimPayMthdCd=1
        - 유심비/가입비: NmcpServiceUtils.getCodeNmDto(GROUP_CODE_USIM_PRICE_INFO)
          → usimService.selectJoinUsimPriceNew([M포탈-M전산 API] /storeUsim/joinUsimPriceNew)
        - agentCode: appformSvc.getAgentCode → [M포탈-M전산 API] /appform/agentCode
        - prmtId: appformSvc.getChrgPrmtId → [M포탈-M전산 API] /appform/getChrgPrmtId
        - additionKeyList: ["12", "29", "30", "31"]
   6-4. STEP 인증 검증
        - certService.getModuTypeStepCnt("nicePin") → NMCP_CRT_VLD_DTL
        - certService.vdlCertInfo("D", [urlType, contractNum, reqUsimSn, connInfo])
   6-5. appformSvc.saveAppform(appformReqResult)
        └─ appformDao.generateRequestKey → SQ_RCP_REQUEST_KEY_01
        └─ appformDao.generateResNo → SQ_RCP_RES_NO_01 (실제 신청서번호로 통합)
        └─ fathService.validateFathSession / updateFathMcpRequestOsst → MCP_REQUEST_OSST
        └─ certMenuType="sharing"
   6-6. SessionUtils.saveAppformSession(rtnAppformReqDto)

7. PC0 사전체크 및 고객생성 (OSST 연동)
   - Thread.sleep(3000) 후 실행
   - appformSvc.sendOsstService(osstParam, EVENT_CODE_PRE_CHECK)
     osstParam: resNo, prntsContractNo(모회선 계약번호), custNo(고객ID)
   - 실패 시: RESULT_CODE 1001~1005 반환

8. PC2 콜백 폴링 (5초 × 50회 = 최대 250초)
   - appformSvc.getRequestOsst(mcpRequestOsstDto)
     조건: mvnoOrdNo=resNo, prgrStatCd=PC2(EVENT_CODE_PC_RESULT)
   - 성공(rsltCd=OSST_SUCCESS): 다음 단계 진행
   - 실패: RESULT_CODE 2001 반환
   - 타임아웃: RESULT_CODE 2002 반환

9. 사전체크 완료 상태 확인
   - appformSvc.chkRealPc2Result(resNo, contractNum)
   └─ 미완료 시: RESULT_CODE + EVENT_CODE_PRE_SCH 반환

10. Y39 아이핀 CI 조회
    - mPlatFormService.MoscSvcContService(osstOrdNo)
    - LOCAL/DEV/STG: NICE 세션 CI로 대체
    └─ 실패 시: McpCommonJsonException("7001")

11. STEP 검증 (CI 비교)
    - certService.vdlCertInfo("D", [urlType=chkPreOpenCi, stepEndYn=Y, connInfo=CI])
    └─ 실패 시: McpCommonJsonException("STEP03")

12. 응답
    - RESULT_CODE: "SUCCESS"
    - REQUEST_KET: requestKey
    - RES_NO: resNo
```

### 주요 입력 파라미터 (AppformReqDto)

```

reqUsimSn       — 유심번호
selfCertType    — 신분증 종류 (02/04이면 selfIssuNum도 필요)
selfIssuExprDt  — 신분증 만료일
selfInqryAgrmYn — 조회동의 (Y 고정)
fathTrgYn       — 안면인증 대상여부
clauseFathFlag  — 안면인증 약관
fathTransacId   — 안면인증 트랜잭션ID
fathCmpltNtfyDt — 안면인증 완료일시
operTypeSmall   — 개통유형 (유심비 조회용)
prdtSctnCd      — 상품구분 (LTE 고정)
cntpntShopId    — 접점 대리점 ID

## 6. [4001-07] saveDataSharing — 포탈/상담사 신청 상세

**URL**: `/appform/saveDataSharingAjax.do`
**JSP**: `dataSharingStep3.jsp` / `dataSharingStep3.js`
**방식**: 포탈(상담사 포함) 신청서 저장 → SMS 발송 후 완료

### 처리 흐름

```
1. 입력값 검증 (contractNum, reqUsimSn 필수)

2. fnSetDataOfdataSharing(appformReqDto) 호출
   → AppformReqDto 완성 + DB 저장 (상세는 §8 참조)
   → dataInfoMap.get("APP_FORM_REQ") = rtnAppformReqDto

3. SMS 발송 (onOffType=0 또는 3일 때)
   - NmcpServiceUtils.getCodeList(SEND_LMS_CNTPNT_GROP_CODE) — SMS 발송 접점코드 목록
   - cntpntShopId 매칭 시:
     fCommonSvc.getMspSmsTemplateMst(SMS_TEMPLATE_ID_NEW)
     → smsSvc.sendKakaoNoti(subject, mobile, text, callback, kTemplateCode, kakaoSenderKey, templateId)
   - 청소년(CSTMR_TYPE_NM)이면 법정대리인 번호에도 추가 발송

4. 응답
   - RESULT_CODE: "SUCCESS"
   - REQUEST_KET, RES_NO
```

> **참고**: 이 API는 신청서 저장 + SMS 발송만 처리. 사전체크(PC0)는 4001-08(`saveDataSharingStep1`)에서 수행.

## 7. [4001-08] saveDataSharingStep1 — 단계별 신청 Step1 상세

**URL**: `/appform/saveDataSharingStep1Ajax.do`
**JSP**: `dataSharingStep3.jsp` / `dataSharingStep3.js`
**방식**: PC0 사전체크만 담당 (4001-07에서 신청서 저장 완료 후 호출)

### 처리 흐름

```
1. fnSetDataOfdataSharing(appformReqDto) 호출
   → rtnAppformReqDto, certContractNum, customerId 획득

2. 중복 PC0 방어로직
   - PC2 이미 성공했으면 바로 SUCCESS 반환:
     appformSvc.requestOsstCount({mvnoOrdNo=resNo, prgrStatCd=PC2}) > 0
   - PC0에 ITL_SST_E0002 결과 이미 있으면 SUCCESS 반환:
     appformSvc.requestOsstCount({prgrStatCd=PC0, rsltCd=ITL_SST_E0002}) > 0

3. PC0 사전체크 및 고객생성 (OSST 연동)
   - Thread.sleep(3000) 후 실행
   - appformSvc.sendOsstService(
       {resNo, prntsContractNo=certContractNum, custNo=customerId},
       EVENT_CODE_PRE_CHECK
     )
   - 성공 → RESULT_CODE=SUCCESS, REQUEST_KET, RES_NO
   - 실패 → RESULT_CODE 1001~1005

4. Step2(/appform/saveDataSharingStep2Ajax.do)에서 PC2 폴링 + NU1(번호조회) + NU2(번호예약)
```

## 8. fnSetDataOfdataSharing — 공통 데이터 설정 함수

4001-07과 4001-08 양쪽에서 호출하는 내부 공통 함수.

```
입력: AppformReqDto (contractNum, reqUsimSn 등)
출력: Map<String, Object>
  - APP_FORM_REQ: AppformReqDto (완성된 신청서 데이터)
  - CONTRANT_NUM: certContractNum (모회선 계약번호)
  - CUSTOMER_ID:  customerId

처리 순서:
  1. 회선정보 조회 (회원/비회원 분기)
     - 회원:   mypageService.selectCntrList → [M포탈-M전산 API] /mypage/cntrList    [제외-회원불필요]
     - 비회원: mypageService.selectCntrListNoLogin → [M포탈-M전산 API] /mypage/cntrListNoLogin

  2. 기존 신청서 조회 (appformSvc.getCopyMcpRequest)
     - AppformMapper.getCopyMcpRequest
     - 테이블: MCP_REQUEST, MCP_REQUEST_CSTMR, MCP_REQUEST_REQ,
               MCP_REQUEST_SALEINFO, MCP_REQUEST_AGENT
     - MSF에 없으면 M포탈 테이블에서 fallback 조회

  3. 공통코드 조회
