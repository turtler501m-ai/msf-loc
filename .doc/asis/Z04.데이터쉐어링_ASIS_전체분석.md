## 1. 개요

데이터쉐어링은 **2가지 기능** × **2가지 신청 경로**로 구성된다.

| 기능 | 설명 | TOBE |
|------|------|------|
| **데이터쉐어링 (MyShareData)** | 개인 회선 간 데이터 공유 신청/해지 | ✅ 개발 대상 |
| **함께쓰기 (ReShareData)** | 주기회선 ↔ 받기회선 방식 결합 신청 | ❌ **TOBE 제외** |

> 함께쓰기(ReShareData)는 TOBE 개발 대상이 아니다. 이 문서의 분석 범위는 **데이터쉐어링(MyShareData)** 에 한정한다.

데이터쉐어링 신청 경로는 두 갈래:
- **간편신청 A** (`saveDataSharingSimple`, 4001-03): 비회원(세션 기반) 1-Step 모바일 전용
- **단계별신청 B** (`saveDataSharing`+`saveDataSharingStep1`+`saveDataSharingStep2`+`saveDataSharingStep3`): 포탈/상담사 다단계 — OSST 신규개통 처리 포함

MSP 폴더에는 해당 기능 없음 — MCP 전용.

## 6. [4001-01] dataSharingStep2 — 인증후 화면 상세

**URL**: `/m/content/dataSharingStep2.do`
**메서드**: `MyShareDataController.dataSharingStep2()`
**JSP**: `dataSharingStep2.jsp`
**역할**: Step1 인증 완료 후 쉐어링 가능 여부 체크 + 결합 목록 조회

### 처리 흐름

```
1. X71 결합 중인 대상 조회
   - myShareDataSvc.mosharingList(custId, ncn, ctn)
   - 단, customerType(법인/사업자) 또는 userType이 있을 경우 X71 호출 스킵

2. SOC 체크 (socChkYn)
   - appFormService.selectSocDesc(contractNum, "SHARINGRATE")
   - SHARINGRATE 코드에 해당하는 부가서비스가 없으면 쉐어링 불가
   - model.addAttribute("socChkYn", socChkYn)

3. 고객유형 체크 (customerType)
   - M전산 조회 → 법인(G) / 사업자(B) 고객은 가입 불가
   - model.addAttribute("customerType", customerType)

4. 청구계약 변경 가능 여부 체크 (changeYn)
   - farChangewayInfo → 지로 청구 방식인 경우 신규개통 불가
   - model.addAttribute("changeYn", changeYn)

5. 신규 개통 가능 시간 체크 (isMacTime)
   - appFormService.isMacTime() — 현재 시간이 개통 가능 시간대인지 확인
   - model.addAttribute("isMacTime", isMacTime)

6. 결합 목록 모델 전달
   - moscDataSharingResDto.getSharingList() → model
```

### TOBE GAP 분석

| ASIS 처리 | TOBE 구현 여부 | 비고 |
|-----------|--------------|------|
| X71 결합목록 조회 | ✅ 구현됨 | `getDataSharingList` API |
| socChkYn SOC 체크 | ❌ 미구현 | SHARINGRATE 코드 비교 |
| customerType 법인/사업자 체크 | ❌ 미구현 | G/B 고객 차단 |
| changeYn 지로청구 체크 | ❌ 미구현 | 청구계약 방식 확인 |
| isMacTime 개통가능시간 체크 | ❌ 미구현 | 신규개통 경로(B)에 한정 |

> **TOBE 제외 확정**: selectCntrList(회원), getNicePinCi(본인인증), 배송, 결제/포인트/추천인

## 8. [4001-03] saveDataSharingSimple — 간편신청 (경로 A)

**URL**: `/m/appform/saveDataSharingSimpleAjax.do` (모바일 전용)
**JSP**: `reqSharingView.jsp` / `reqSharingView.js`
**방식**: 비회원 1-Step 간편신청 (OSST 비동기 폴링 방식)

### 처리 흐름

```
1. 입력값 검증
   - contractNum, reqUsimSn 필수

2. [TOBE 제외] 세션 기반 사용자 확인
   - 회원: SessionUtils.getUserCookieBean() → mypageService.selectCntrList(userId)
   - 비회원: SessionUtils.getNonmemberSharingInfo() → mypageService.selectCntrListNoLogin()
   - TOBE 대체: ncn, ctn, custId 직접 전달 (stateless)

3. 계약번호 본인 소유 검증 (isOwn)
   - cntrList에서 contractNum 매칭 → cntrMobileNo, certContractNum, customerId 추출

4. [TOBE 제외] NICE 본인인증 세션 확인
   - SessionUtils.getNiceResCookieBean()
   - TOBE: 대리점 시스템 — 본인인증 불필요

5. [TOBE 제외] 청구계정번호 조회
   - mypageService.selectBanSel(contractNum)

6. 신청서 저장 (중복 방지: 세션 확인 후 최초 1회만)
   6-1. appformSvc.getCopyMcpRequest(appformReqDto)
        └─ MCP_REQUEST, MCP_REQUEST_CSTMR 등에서 기존 정보 복사
   6-2. 주민등록번호 앞 6자리 생년월일 비교 (NICE 세션과 매칭)
   6-3. AppformReqDto 고정값 설정
        - operType=NAC3(OPER_TYPE_NEW), serviceType=PO
        - socCode=KISOPMDSB (데이터쉐어링 신규개통 SOC)
        - onOffType: 모바일=7, PC=5
        - prdtSctnCd=LTE, joinPrice=0, usimPrice=0, usimPayMthdCd=1
        - additionKeyList: ["12", "29", "30", "31"]
   6-4. [TOBE 제외] STEP 인증 검증 (certService.vdlCertInfo)
   6-5. appformSvc.saveAppform(appformReqResult, certMenuType="sharing")
        └─ MSF_REQUEST (TOBE) / MCP_REQUEST (ASIS) INSERT
        └─ resNo 생성 (SQ_RCP_RES_NO_01)
   6-6. [ASIS 세션] SessionUtils.saveAppformSession(rtnAppformReqDto)

7. PC0 사전체크 및 고객생성 (OSST 연동)
   - Thread.sleep(3000) 대기
   - appformSvc.sendOsstService({resNo, prntsContractNo, custNo}, EVENT_CODE_PRE_CHECK)
   - 실패 시: RESULT_CODE 1001~1005

8. PC2 콜백 폴링 (5초 × 50회 = 최대 250초)
   - appformSvc.getRequestOsst(mvnoOrdNo=resNo, prgrStatCd=PC2)
   - 성공(rsltCd=OSST_SUCCESS) → 다음 단계

9. 사전체크 완료 상태 확인
   - appformSvc.chkRealPc2Result(resNo, contractNum) — ST1 연동

10. Y39 아이핀 CI 조회
    - mPlatFormService.MoscSvcContService(osstOrdNo)
    - LOCAL/DEV/STG: NICE 세션 CI로 대체

11. STEP CI 검증
    - certService.vdlCertInfo("D", [urlType=chkPreOpenCi, connInfo=CI])

12. 응답
    - RESULT_CODE: "SUCCESS", REQUEST_KET, RES_NO
```

### TOBE 제외 항목 (경로 A)

| 항목 | 이유 |
|------|------|
