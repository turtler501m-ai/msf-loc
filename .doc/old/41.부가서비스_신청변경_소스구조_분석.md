# 부가서비스 신청/변경 소스 구조 분석

**대상**: M포털(MCP) 내 "부가서비스 신청/해지" 화면 및 연동 처리  
**비고**: 화면 테이블 캡션의 "부가세"는 이용요금(VAT포함) 의미이며, 업무는 **부가서비스(Add-on Service)** 신청·변경·해지임.

---

## 1. 개요

| 구분 | 설명 |
|------|------|
| 화면명 | 부가서비스 신청/해지 |
| URL | `/mypage/regServiceView.do`, `/m/mypage/regServiceView.do` |
| 탭 | **신청**(가입 가능 목록·신청) / **변경·해지**(이용중 목록·변경·해지) |
| M플랫폼 연동 | **X20** 부가서비스 조회, **X21** 부가서비스 신청, **X38** 부가서비스 해지 |

---

## 2. 레이어별 소스 구조

### 2.1 프론트엔드 (View · JS)

| 구분 | 경로 | 역할 |
|------|------|------|
| 화면(PC) | mcp-portal-was/src/main/webapp/WEB-INF/views/**portal/mypage/regServiceView.jsp** | 부가서비스 신청/해지 메인 JSP |
| 화면(모바일) | mcp-portal-was/.../views/**mobile/mypage/regServiceView.jsp** | 동일 기능 모바일 뷰 |
| 스크립트 | mcp-portal-web/src/resources/js/portal/mypage/**regServiceView.js** | 탭·조회·신청·해지·팝업 호출 |

**regServiceView.jsp 요약**  
- 탭: `#tab1` 신청, `#tab2` 변경/해지  
- 신청 탭: 회선 선택(`#tab_ncn1`) → 조회 → 무료 부가서비스(`.freeDataList`) / 유료 부가서비스(`.noFreeDataList`) 아코디언 → [부가서비스 신청]  
- 변경/해지 탭: 회선 선택(`#tab_ncn2`) → 조회 → 이용중 부가서비스 테이블(`#addSvcList`), 변경/해지 버튼  
- 테이블 캡션: 부가서비스명, 이용요금(VAT포함), 해지

**regServiceView.js 주요 함수**  
- `regSvcSearch()`: 신청 탭 조회 → `/mypage/addSvcListAjax.do` (신청 가능 목록 + 이용중 SOC 반영)  
- `rateCdDtl(type, indexType)`: 부가서비스 상세 설명 → `/mypage/getContRateAdditionAjax.do`  
- `addRegSvcSearch()`: 변경/해지 탭 조회 → `/mypage/myAddSvcListAjax.do` (이용중 부가서비스 = X20)  
- `btn_regSvc()`: 신청 버튼 → 선택 검증 후 `/mypage/addSvcViewPop.do` (팝업, 파라미터: ncn, rateAdsvcCd, rateAdsvcNm, baseVatAmt)  
- `btn_regChg()`: 변경 버튼 → `/mypage/addSvcViewPop.do` (동일 팝업, flag 등)  
- `btn_regCancel()`: 해지 확인 후 `/mypage/moscRegSvcCanChgAjax.do` (X38 해지)

### 2.5 DB · API (mcp-api · 마이페이지)

| 구분 | 경로 | 역할 |
|------|------|------|
| API 컨트롤러 | mcp-api/.../mypage/controller/**MypageController.java** | POST `/mypage/regService` → selectRegService(ncn) |
| Mapper | mcp-api/.../mypage/mapper/**MypageMapper.java** | selectRegService(ncn) → List\<McpRegServiceDto\> |
| DTO | mcp-api/.../mypage/dto/**McpRegServiceDto** | 부가서비스 목록용 (rateCd, rateNm, baseAmt, svcRelTp 등) |

**역할 정리**  
- **selectRegService(ncn)**: 계약번호(ncn) 기준으로 **신청 가능한 부가서비스 목록**(마스터성) 조회.  
- **X20**: 해당 회선의 **현재 가입중인 부가서비스(SOC)** 실시간 조회.  
- addSvcListAjax에서는 **X20으로 이용중 SOC 리스트**를 구한 뒤, **selectRegService로 가져온 목록**에 useYn 등 표시해 신청 탭에 뿌림.

## 3. 데이터 흐름 요약

### 3.1 신청 탭

```
1. 회선 선택(ncn) → addSvcListAjax.do
2. RegSvcController.addSvcListAjax
   - regSvcService.selectAddSvcInfoDto(ncn, ctn, custId)  → X20 이용중 SOC 리스트
   - mypageService.selectRegService(contractNum)          → DB 신청 가능 목록
   - 목록별 useYn = 목록.rateCd가 useSocList에 있으면 'Y'
   - 무료(listC) / 유료(listA) 구분 → listC, listA, list 반환
3. 화면: 무료·유료 아코디언 렌더, [부가서비스 신청] 클릭 시 선택한 rateAdsvcCd로 addSvcViewPop.do
4. 팝업에서 신청 확정 시 regSvcChgAjax.do (soc, ftrNewParam) → regSvcChg(X21)
```

### 3.2 변경/해지 탭

```
1. 회선 선택(ncn) → myAddSvcListAjax.do
2. RegSvcController.myAddSvcListAjax
   - regSvcService.selectmyAddSvcList(ncn, ctn, custId) → X20 + 해지가능/변경가능 보강
3. 화면: 이용중 부가서비스 테이블, [변경] [해지] 버튼
4. [해지] → moscRegSvcCanChgAjax.do(rateAdsvcCd=soc) → moscRegSvcCanChg(X38)
5. [변경] → addSvcViewPop.do(flag 등) → 팝업에서 변경 시 regSvcChgAjax.do(flag='Y') → X38 해지 후 X21 신청
```

### 3.3 M플랫폼 이벤트 정리

| 이벤트 | 용도 | 호출 경로 |
|--------|------|-----------|
| **X20** | 가입중 부가서비스 조회 | getAddSvcInfoDto → addSvcListAjax(이용중 SOC 마킹), myAddSvcListAjax(이용중 목록) |
| **X21** | 부가서비스 신청 | regSvcChg → regSvcChgAjax(신청 또는 변경 시 신청 단계) |
| **X38** | 부가서비스 해지 | moscRegSvcCanChg → moscRegSvcCanChgAjax(해지), regSvcChgAjax(변경 시 해지 후 신청) |

## 7. 요약

- **부가세**는 테이블 컬럼 "이용요금 (VAT포함)"에서 쓰인 VAT 의미이며, **업무는 부가서비스(Add-on Service) 신청·변경·해지**이다.  
- **AS-IS 메인 진입점**: RegSvcController + regServiceView.jsp / regServiceView.js. **TO-BE**: 동일하게 RegSvcController 에서 REST API (`/api/v1/addition/current` 등)로 제공.  
- **실제 조회/처리**: X20(이용중 조회), X21(신청), X38(해지)는 MplatFormService 경유, 신청 가능 목록은 mcp-api selectRegService(DB)와 X20 결과를 조합해 사용.  
- **변경**은 “해지(X38) 후 신청(X21)” 한 번에 처리하는 플로우로, regSvcChgAjax에서 flag='Y'일 때 moscRegSvcCanChg 후 regSvcChg 호출.

이 구조를 기준으로 스마트서식지(mform-web)에서 부가서비스 신청/변경 기능을 이식하거나 API 스펙을 맞출 수 있다.
