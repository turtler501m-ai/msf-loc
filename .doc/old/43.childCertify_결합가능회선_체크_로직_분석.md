# childCertify 결합가능 회선 체크 로직 분석

**대상**: MCP 포탈 결합(아무나/가족 결합) 시 **자회선(결합 대상 회선)** 이 결합 가능한지 검증하는 흐름.  
**관련 컨트롤러**: `mcp-portal-was` `CombineController.java`  
**API**: `childCertifyAjax.do`(MVNO-MVNO), `childCertifyKtAjax.do`(MVNO-KT), `childCertifyMoreAjax.do`(인증 시간 연장)

---

## 1. API 구분

| API | 용도 | 모회선 | 자회선(결합 대상) |
|-----|------|--------|-------------------|
| **childCertifyAjax.do** | 결합 대상 회선 가능 여부 | MVNO(로그인 회선) | MVNO(입력 휴대폰번호) |
| **childCertifyKtAjax.do** | 결합 대상 회선 가능 여부 | KT(로그인 회선) | MVNO 또는 KT(이름·생년월일·성별·회선) |
| **childCertifyMoreAjax.do** | 인증 유효 시간 연장 | - | 세션만 갱신 |

- **childCertify**: 모회선·자회선 모두 **KT M모바일(MVNO)**. 자회선은 **휴대폰번호**(`cntrMobileNo`)로 조회.
- **childCertifyKt**: 모회선 **KT**(인터넷/모바일), 자회선 **KT 또는 MVNO**. 본인명의/가족명의 분기, 가족명의 시 **X77 → X78 사전체크** 후 SMS 인증으로 이어짐.

---

## 2. childCertify (MVNO-MVNO) 결합가능 회선 체크 순서

**진입**: `CombineController.childCertify(authSmsDto, userCntrMngDto, prcsMdlInd)`  
**입력**: 세션(모회선), `userCntrMngDto.cntrMobileNo`(자회선 휴대폰번호)

### 2.1 세션 검증

- `SessionUtils.getAuthSmsBean(authSmsDto)` → 모회선 정보.
- `rtnDto == null` 또는 `svcCntrNo` 없음 → **0001** "세션 없음" 예외.

### 2.2 자회선 M전산 조회 (KT M모바일 가입 여부)

- `mypageService.selectCntrListNoLogin(userCntrMngDto)` (휴대폰번호로 계약 조회).
- **cntrObj == null** → **00001** "kt M모바일 고객만 결합이 가능합니다."

### 2.3 청소년/외국인·주민번호 형식

- `cntrObj.getUnUserSSn()`: 숫자 7자리 미만이면 **10008** "주민번호 형식이 일치하지 않습니다."

### 2.4 동일 회선·사용중·개인고객

- 모회선 `svcCntrNo` == 자회선 `svcCntrNo` → **00002** "동일한 회선으로 요청 하였습니다."
- 자회선 `subStatus != "A"`(사용중 아님) → **00003** "현재 회선을 사용 중인 고객만 결합이 가능합니다."
- 자회선 `customerType != "I"`(개인 아님) → **00004** "개인고객만 가입이 가능합니다."

### 2.5 요금제 결합 가능 여부 (M전산·msp)

- `mypageService.selectSocDesc(cntrObj.getContractNum())` → 자회선 요금제(SOC).
- `myCombinationSvc.selectMspCombRateMapp(tempRateCd)` → 결합 매핑 조회.
- **myCombination == null** → **0005** "해당 상품은 결합이 불가합니다."
- 성공 시 `RESULT_SERVICE_NM` = 요금제명.

### 2.6 결합 승인 대기 회선 제외

- **모회선** 기준: `selectMcpReqCombine(mSvcCntrNo=모회선, rsltCd="R")` 존재 → **1001** "결합 승인 대기 중인 회선입니다.…"
- **자회선** 기준: `selectMcpReqCombine(mSvcCntrNo=자회선, rsltCd="R")` 존재 → **1002** 동일 메시지.

### 2.7 X77 개별 서비스 조회 (결합 여부)

- **MoscCombReqDto**: 모회선 custId, ncn, ctn, `combSvcNoCd="M"`, `combSrchId=자회선 휴대폰번호`.
- `myCombinationSvc.selectCombiMgmtListLog(moscCombReqDto, …)` → **X77** 호출.
- **실패** (비 LOCAL222): **1001** "개별 서비스 조회(X77) 실패…"
- **성공**:
  - `moscSrchCombInfoList`에서 자회선 번호와 일치하고 `combYn=="Y"`인 항목 있으면 **childCobun = "Y"**.
  - **모회선·자회선 모두 결합(Y)** 이면 → **0006** "이미 결합된 회선입니다."

### 2.8 X78 결합 사전체크

- 모회선이 이미 결합(Y)이면: 자회선을 **회선추가(A)** 로, 모회선/자회선 파라미터 reverse.
- 모회선 미결합이면: 모회선 기준으로, 자회선이 이미 결합(Y)이면 **A(회선추가)**, 아니면 **N(신규결합신청)**.
- **MoscCombChkReqDto** 세팅: custId, ncn, ctn, jobGubun(N/A), svcNoCd="M", svcNoTypeCd="MB", cmbStndSvcNo(상대 회선), aplyMethCd="04", aplyRelatnCd="01" 등.
- `myCombinationSvc.insertCombChkInfoLog(moscCombChkReqDto, …)` → **X78** 호출.
- **SelfServiceException**:
  - ITL_SFC_E110 → **0007** "결합중인 회선입니다. 결합신청이 불가합니다."
  - ITL_SYS_E00001 → **0007** "결합 중인 회선이 많아 추가 결합이 되지 않습니다."
- **기타 예외** → **0008** M플랫폼 응답 빈값 메시지.

### 2.9 X78 응답 판단

- **성공·가능**: `moscCombChkRes.isSuccess() == true` 이고, `moscCombChkList` 안에 `sbscYn == "N"` 인 항목이 없고, 상위 `sbscYn == "Y"`.
- **불가**: 리스트 중 **한 건이라도 sbscYn == "N"** → **0009** + 해당 `resltMsg` (또는 상위 메시지).
- **실패**: success 아니거나 null → **0010** M플랫폼 응답 빈값.

이후 정상이면 SMS 인증 번호 발송·세션 저장 등으로 진행.

## 3. childCertifyKt (KT–MVNO/KT) 결합가능 회선 체크 순서

**진입**: `CombineController.childCertifyKt(authSmsDto, prcsMdlInd)`  
**입력**: 세션(모회선), `authSmsDto`(자회선 정보: 동일명의 여부, 이름·생년월일·성별·회선 등).

### 3.1 세션·X77 조회

- 모회선 세션 없으면 **0001** 예외.
- `myCombinationSvc.selectCombMgmtList(prntDto, authSmsDto, prcsMdlInd)` 호출.
  - 내부에서 **X77** 호출 (`selectCombiMgmtListLog`), `combSvcNoCd="K"`(KT 모회선), `combSrchId=이름`, 생년월일·성별·cmbStndSvcNo(자회선 번호/ID) 등.

### 3.2 selectCombMgmtList 내부 로직 (서비스)

- **X77 실패** → RESULT_CODE **1001**, "개별 서비스 조회(X77) 실패…"
- **모회선이 이미 결합(parentObj.combYn == "Y")** → **0006** "kt m 모바일 회선이 이미 결합된 회선입니다."
- **본인명의(sameCustYn == "Y")**:
  - KT 정보제공동의(indvInfoAgree) 미동의 시 **1002** "KT고객센터로 전화하여…"
  - X77 결과에서 모바일/인터넷 구분·가입일(인터넷은 전월/당월 등) 필터 후 **combList** 세션 저장, **AJAX_SUCCESS** + combList 반환.
- **가족명의**:
  - 자회선 번호와 일치하는 항목 중 `combYn=="Y"` 있으면 **childCobun = "Y"**.
  - **childCobun**, **AJAX_SUCCESS** 반환.

### 3.3 본인명의인 경우

- `combListMap`만 반환하고 **사전체크·SMS 없이** 리스트만 내려줌.

### 3.4 가족명의인 경우 (사전체크 + SMS)

- `myCombinationSvc.getCombChkInfo(prntDto, authSmsDto, prcsMdlInd, childCobun)` 호출.
  - **X78** 요청 구성: jobGubun = childCobun=="Y" ? **"A"(회선추가)** : **"N"(신규결합신청)**, svcNoCd="K", 자회선 정보(이름·생년월일·성별·회선), aplyRelatnCd="11"(대리인) 등.
  - 본인명의로 잘못 들어온 경우(이름·생년월일·성별이 모회선과 동일) → **0100** "본인명의의 경우 '본인명의'를 선택하여 진행 바랍니다."
  - **insertCombChkInfoLog**(X78) 호출.
  - X78 결과에서 **moscCombChkList** 중 **sbscYn == "N"** 한 건이라도 있으면 **0009** + 메시지.
  - 전부 **Y**이면 **AJAX_SUCCESS**.
- **getCombChkInfo**가 성공이면 SMS 발송·세션 저장 후 **AJAX_SUCCESS** 반환.
- 실패 시 **getCombChkInfo** 결과 그대로 반환.

## 4. 공통 판단 요약 (결합 “가능” 회선 조건)

| 순서 | 체크 항목 | 불가 시 코드/메시지 |
|------|-----------|----------------------|
| 1 | 모회선 세션 존재 | 0001 |
| 2 | 자회선이 KT M모바일 가입 회선(또는 KT 회선) | 00001 "kt M모바일 고객만 결합이 가능합니다." |
| 3 | 주민번호 형식(숫자 7자리 이상) | 10008 |
| 4 | 모·자 동일 회선 아님 | 00002 "동일한 회선으로 요청 하였습니다." |
| 5 | 자회선 사용중(subStatus=A) | 00003 "현재 회선을 사용 중인 고객만…" |
| 6 | 자회선 개인고객(customerType=I) | 00004 "개인고객만 가입이 가능합니다." |
| 7 | 자회선 요금제가 결합 매핑에 존재(msp) | 0005 "해당 상품은 결합이 불가합니다." |
| 8 | 모/자 회선이 결합 승인대기(rsltCd=R) 아님 | 1001/1002 "결합 승인 대기 중인 회선입니다." |
| 9 | X77: 모·자 모두 결합(Y)이면 불가 | 0006 "이미 결합된 회선입니다." |
| 10 | X78: 사전체크 성공, 리스트 전부 sbscYn=Y | 0009/0010 또는 SelfServiceException 0007 |

## 5. 연동 정리

| 구분 | API/DB | 역할 |
|------|--------|------|
| M전산 | selectCntrListNoLogin | 휴대폰번호로 자회선 계약·고객유형·상태·요금제 조회 |
| M전산 | selectSocDesc | 계약번호로 SOC(요금제) 조회 |
| msp | selectMspCombRateMapp | 요금제 코드로 결합 가능 여부 조회 |
| DB | selectMcpReqCombine(rsltCd=R) | 결합 승인 대기 회선 조회 |
| M플랫폼 | **X77** (moscCombSvcInfoLog) | 개별 서비스(결합) 조회 → 모/자 회선 결합 여부(combYn) |
| M플랫폼 | **X78** (moscCombChkInfoLog) | 결합 사전체크 → sbscYn 리스트로 결합 가능 여부 판단 |

---

## 6. 프론트 호출 (참고)

- **combineWireless.js**: 자회선 휴대폰번호 입력 후 `childCertifyAjax.do` 호출.
- **combineKt.js**: 본인/가족명의 선택, 자회선 정보 입력 후 `childCertifyKtAjax.do` 호출; 가족명의이면 이후 `childCertifyMoreAjax.do`(시간 연장), SMS 인증 후 `childVerifyAjax.do` 등으로 이어짐.

이 문서는 **결합가능 회선 체크**가 세션 검증 → M전산 자회선 조회 → 요금제 결합 매핑 → 승인대기 제외 → **X77**(결합 여부) → **X78**(사전체크) 순으로 이루어지며, 각 단계에서 불가 시 위 코드/메시지로 즉시 return 하는 구조임을 정리한 것이다.
