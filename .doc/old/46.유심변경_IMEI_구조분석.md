# 유심변경(IMEI) 관련 구조 분석

## 1. 개요

본 문서의 **유심변경**은 다음 두 맥락을 다룹니다.

1. **셀프개통 시 유심(ICC_ID) 입력·검증 및 “이미 사용 이력 USIM” 오류(2302) 대응**  
   - 사용자가 입력한 **유심번호(19자리 ICC_ID)** 의 유효성 조회 후, 개통 결과(OP2)에서 **이미 사용된 USIM**으로 판정될 때 **유심 변경(재입력/유심구매)** 유도.
2. **기기변경**  
   - 기존 가입 회선에 대해 **휴대폰만 변경**하는 서비스(OPER_TYPE_CHANGE/EXCHANGE). 단말 식별 시 **IMEI** 등이 약관/개인정보 처리에 언급되나, 포털에서 별도 “IMEI 변경” 전용 메뉴는 없음.

**용어 정리**

- **ICC_ID / reqUsimSn**: 유심 일련번호(19자리, ‘-’ 제외 입력).
- **IMEI**: 단말기 국제 식별번호. 약관·개인정보 처리 목적 등에서 “단말기 정보(모델명, IMEI번호, USIM정보 등)”로 수집·보유 대상에만 등장.

---

## 2. 셀프개통 흐름에서의 유심(ICC_ID) 처리

### 2.1. 유심번호 입력 및 저장

- **화면**: 서식지(appForm/appOutForm) — 유심 단독/셀프개통 시 **유심번호 입력** 필드 노출.
- **입력 필드**
  - **ID**: `#reqUsimSn`
  - **placeholder**: `직접입력('-'없이 입력)`, **maxlength**: 19.
  - **저장/전송**: `reqUsimSn` → 서버에서는 **ICC_ID**와 동일 값으로 사용.
- **검증**: `validator.config['reqUsimSn'] = 'isNumFix19'` (19자리 숫자).

### 2.2. 유심번호 유효성 조회 (사용 가능 여부)

- **URL**: `POST /msp/moscIntmMgmtAjax.do`
- **Controller**: `AppformController.moscIntmMgmt(JuoSubInfoDto juoSubInfo)`
- **파라미터**: `JuoSubInfoDto.iccId` = `$("#reqUsimSn").val()` (trim).
- **처리**
  - `mPlatFormService.moscIntmMgmtSO(juoSubInfo)` 호출 → M-Platform 유심 사용 가능 여부 조회.
  - 응답 `psblYn == "Y"`: 사용 가능 → `RESULT_CODE` 성공, `USIM_ORGN_ID` 반환(접점코드), `#isUsimNumberCheck` = "1", `reqUsimSn` disabled.
  - `psblYn != "Y"`: 사용 불가 → `RESULT_CODE` "0001", `RESULT_MSG` = `rsltMsg`.
- **프론트**: 유효성 통과 시 “유심번호 유효성 체크 완료” 표시, 실패 시 메시지 표시.

### 2.3. DB 저장

- **테이블**: `MCP_REQUEST`
- **컬럼**: `REQ_USIM_SN` (유심번호/ICC_ID)
- **매퍼**: `AppFormMapper.xml` — insert/update 시 `REQ_USIM_SN` 포함.
- **API/OSST**: `MPlatformMapper.xml` 등에서 `A.REQ_USIM_SN AS iccId` 로 매핑되어 개통 요청/OSST 연동 시 **iccId** 로 전달.

## 3. 개통 결과(OP2)와 “이미 사용 이력 USIM”(2302)

### 3.1. OP2 조회

- **이벤트**: `EVENT_CODE_REQ_OPEN_RESULT = "OP2"` (개통 및 수납 결과).
- **저장**: OSST 콜백 등으로 `MCP_REQUEST_OSST` 에 `PRGR_STAT_CD='OP2'`, `RSLT_CD`, `RSLT_MSG` 등 적재.
- **조회**: `AppformController` 내 개통 결과 확인 로직에서  
  `appformSvc.getRequestOsst(mcpRequestOsstDto)` 호출  
  → `AppformMapper.getRequestOsst` (동일 `MVNO_ORD_NO`, `PRGR_STAT_CD='OP2'`, 최신 1건).

### 3.2. 결과 코드 전달

- **DAO**: `McpRequestOsstDto` 에 `rsltCd`, `rsltMsg` 등 매핑.
- **Controller**:  
  - `rsltCd` 가 성공이 아니면 `OSST_RESULT_CODE = "RESULT_" + rsltCd` 로 세팅 (예: `RESULT_2302`).  
  - 일부 `rsltCd`/`rsltMsg` 조합에 따라 세부 코드 변환(예: 2202_01, 2412_01~04, 2028_01, 9000_01).

### 3.3. 2302 (이미 사용한 USIM)

- **의미**: 입력한 USIM(ICC_ID)이 **이미 사용 이력이 있어 재등록 불가**.
- **메시지** (simpleErrMsg.js):  
  `SIMPLE_OP2_MSG.RESULT_2302`  
  = "입력하신 USIM은 이미 사용한 이력이 있는 USIM으로 재 등록이 불가합니다. … 새 유심을 구매하여 재 시도 바랍니다. (구매처: 당사 홈페이지 또는 제휴사이트 …)"
- **버튼 영역**: `SIMPLE_OP2_TYPE.RESULT_2302 = "divChangeUsimButton"`  
  → 개통 실패 다이얼로그에서 **divChangeUsimButton** 표시.

### 3.4. “유심 변경” UI (divChangeUsimButton)

- **위치**: `#simpleDialog` 내부, `._simpleDialogButton` 중 `#divChangeUsimButton`.
- **구성**
  - **유심구매** 버튼: `location.href='/m/appForm/reqSelfDlvry.do'` (모바일), portal은 동일 경로에서 `/appForm/reqSelfDlvry.do` 사용.
  - **확인** 버튼: 다이얼로그 닫기.
- **역할**: 2302 발생 시 사용자에게 **새 유심 구매 페이지(유심구매)** 로 이동하거나 확인 후 닫도록 유도 → 사실상 “유심 변경”은 **다른 USIM(새 유심)으로 재시도**하는 플로우.

---

## 4. 프론트엔드 요약 (appForm.js)

- **유심번호 검증**  
  - `#reqUsimSn` 입력 후 유효성 조회 시 `iccId` 로 `/msp/moscIntmMgmtAjax.do` 호출.
- **개통 결과(OP2) 수신**  
  - 응답 `OSST_RESULT_CODE`(예: `RESULT_2302`)에 따라 `SIMPLE_OP2_MSG`/`SIMPLE_OP2_TYPE` 로 문구·버튼 영역 결정.  
  - `layerType === "divChangeUsimButton"` 이면 `#divChangeUsimButton` 표시 → “유심구매” / “확인”.
- **기타**  
  - `reqUsimSn` 은 서식지 제출/임시저장 시 `paramObj.reqUsimSn` 또는 `pageObj.applDataForm` 등으로 함께 전송.

---

## 5. 기기변경과 IMEI

- **기기변경**: 기존 고객이 **휴대폰만 교체**, 번호·요금제 등은 유지.  
  - `OPER_TYPE_CHANGE` / `OPER_TYPE_EXCHANGE`, 상수 주석 등에서 “기기변경”으로 통일.
- **화면**: appForm/appOutForm 등에서 “기기변경” 선택 시  
  - 기기변경 고객인증 레이어, `#inputPhoneNum`(기기변경 휴대폰 번호) 등 노출.  
  - 인증·세션: `SessionUtils` 의 “기기변경 고객인증” 관련 저장/조회.
- **IMEI**:  
  - 약관/개인정보 처리 안내(term_03, ET1001010200 등)에서 **단말기 정보(모델명, IMEI번호, USIM정보 등)** 수집·보유 목적으로만 기술.  
  - 포털 내 “유심변경(IMEI)” 또는 “IMEI 변경” 전용 메뉴/API는 없으며, 유심 재입력/유심구매 유도는 위 **2302 + divChangeUsimButton** 플로우로 처리.

## 7. 정리

- **유심변경**은, 셀프개통 맥락에서는 **입력한 유심(ICC_ID)이 이미 사용 이력(2302)** 일 때 **새 유심 구매(유심구매)** 또는 재입력을 유도하는 플로우와, DB/OSST 상의 **reqUsimSn(ICC_ID)** 저장·조회 구조를 의미한다.
- **유심번호**는 **19자리 ICC_ID**, **moscIntmMgmtAjax**로 사용 가능 여부를 먼저 검증하고, 개통 결과 **OP2**의 **RSLT_CD=2302** 시 **divChangeUsimButton**(유심구매/확인)으로 “유심 변경” 대응을 한다.
- **IMEI**는 단말 식별자로 약관/개인정보 처리 항목에만 등장하며, 포털에 “유심변경(IMEI)” 또는 IMEI 전용 변경 API/화면은 없다.
