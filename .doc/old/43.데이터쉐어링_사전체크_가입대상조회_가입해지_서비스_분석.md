# 데이터쉐어링 사전체크·가입대상 조회·가입/해지 처리 서비스 분석

**대상**: MCP 포탈 데이터쉐어링 관련 **사전체크(X69)**, **가입대상 조회(X71)**, **가입/해지 처리(X70)** 서비스.  
**관련**: `MyShareDataController`, `MyShareDataSvcImpl`, `MplatFormService` (X69, X70, X71), `MoscDataSharingResDto`, `OutDataSharingDto`.

---

## 1. API·M플랫폼 구분

| 구분 | API 이벤트 | 용도 | 호출 위치 |
|------|-------------|------|------------|
| **X69** | moscDataSharingChk | 데이터쉐어링 **사전체크** 및 **가입 가능 대상 조회** | 셀프개통 가입 시 사전체크(insertOpenRequestAjax) |
| **X70** | moscDataSharingSave | 데이터쉐어링 **가입(결합)** / **해지** | insertOpenRequestAjax(가입), 해지 시 opmdWorkDivCd=C |
| **X71** | mosharingList | 데이터쉐어링 **결합 중인 대상 조회** | mySharingView, dataSharingStep2 (가입대상·결합 여부 표시) |

---

## 2. X69 — 사전체크 및 가입 가능 대상 조회

### 2.1 역할

- **사전체크**: 대표회선(ncn, ctn, custId) 기준으로 데이터쉐어링 **가입 가능 여부** 및 **가입 가능 대상(회선) 목록** 조회.
- **가입 플로우**: 셀프개통 가입 시 X69 호출 후, 응답 목록 중 **rsltInd = "Y"** 인 항목이 있으면 가입 가능으로 판단 → 이어서 X70(결합) 호출.

### 2.2 M플랫폼 연동

- **진입**: `MplatFormService.moscDataSharingChk(custId, ncn, ctn, crprCtn)`
- **파라미터**: `getParamMap(ncn, ctn, custId, "X69")`. (crprCtn은 주석 처리되어 실제 전달 안 함)
- **응답 VO**: `MoscDataSharingResDto` (CommonXmlNoSelfServiceException 상속)
  - **body 파싱**: `outDataSharingDto` 목록 → `List<OutDataSharingDto> sharingList`

### 2.3 OutDataSharingDto (X69·X71 공통)

| 항목 | 설명 |
|------|------|
| svcNo | 전화번호(데이터쉐어링 대상 회선) |
| efctStDt | 결합 시작일 (YYYYMMDD) |
| rsltInd | **결합가능여부** (X69 사전체크 시: "Y" 이면 가입 가능) |
| rsltMsg | 결과 메시지 |

### 2.4 호출 흐름 (Controller)

- **insertOpenRequestAjax.do** (데이터쉐어링 가입 요청)
  1. 세션/비회원 정보로 `MyShareDataReqDto` 세팅 (ncn, ctn, custId, opmdSvcNo, **selfShareYn="Y"**).
  2. **selfShareYn == "Y"** 일 때만 **X69** 호출: `myShareDataSvc.moscDataSharingChk(myShareDataReqDto)`.
  3. `moscDataSharingChk.getSharingList()`에서 **rsltInd == "Y"** 인 항목이 하나라도 있으면 `shareDataYn = "Y"`.
  4. **shareDataYn == "Y"** 이면 **X70 가입** 호출: `myShareDataSvc.moscDataSharingSave(myShareDataReqDto)` → RESULT_CODE="S".
  5. 그 외면 RESULT_CODE="E".

### 2.5 에러 케이스 (M플랫폼 응답 예)

- **ITL_SFC_E106**: "데이터쉐어링 결합 가능한 개통된 고객이 없습니다."
- X69 실패 또는 sharingList 비어 있음 → 가입 불가(E).

## 3. X71 — 가입대상(결합 중인 대상) 조회

### 3.1 역할

- **결합 중인 데이터쉐어링 회선 목록** 조회.
- **mySharingView**, **dataSharingStep2** 등에서 “이미 결합된 회선이 있는지”, “결합 가능한 회선인지” 판단 및 테이블 표시에 사용.

### 3.2 M플랫폼 연동

- **진입**: `MplatFormService.mosharingList(custId, ncn, ctn)`
- **파라미터**: `getParamMap(ncn, ctn, custId, "X71")` (추가 파라미터 없음)
- **응답 VO**: `MoscDataSharingResDto`
  - **body**: `outDataSharingDto` 목록 → `sharingList` (svcNo, efctStDt, rsltInd, rsltMsg)

### 3.3 호출 흐름 (Controller)

- **dataSharingStep2.do**
  - 고객구분(customerType), 외국인(userType)이 아니면 **X71** 호출: `myShareDataSvc.mosharingList(myShareDataReqDto)`.
  - 성공 시 `resultCode = "00"`, `moscDataSharingResDto.sharingList`를 뷰에 전달.
  - **이미 결합 회선 있음**: sharingList 비어 있지 않으면 “이미 결합되어 있는 회선이 있습니다” 등 표시.
  - **결합 가능**: 요금제(SHARINGRATE 코드), 청구계약(지로 등), 일시정지 여부 등과 함께 판단 후 “데이터쉐어링 결합 가능” 또는 “가입하기” 버튼 노출.

- **mySharingView.do**
  - 동일하게 **X71**로 결합 중인 대상 조회 후, 결합 유무·납부방법·요금제 등에 따라 Case1~7 분기 표시.

### 3.4 에러 케이스 (M플랫폼 응답 예)

- **ITL_SFC_E105**: "데이터쉐어링 결합 중인 대상이 없습니다."  
  → 조회 실패 또는 결합 회선 0건으로 처리.

## 4. X70 — 가입(결합) / 해지 처리

### 4.1 역할

- **가입(결합)**: 데이터쉐어링 대상 회선(opmdSvcNo)을 대표회선에 **결합**.
- **해지**: 이미 결합된 데이터쉐어링 회선(opmdSvcNo)을 **해지**.

### 4.2 M플랫폼 연동

- **진입**: `MplatFormService.moscDataSharingSave(custId, ncn, ctn, opmdSvcNo, opmdWorkDivCd)`
- **파라미터**:
  - `getParamMap(ncn, ctn, custId, "X70")`
  - **opmdSvcNo**: 데이터쉐어링 대상 전화번호(가입 시 개통된 쉐어링 번호, 해지 시 해지할 쉐어링 번호)
  - **opmdWorkDivCd**: 처리구분코드  
    - **A** = 결합(가입)  
    - **C** = 해지

- **응답 VO**: `MpCommonXmlVO` (commHeader 성공/실패만 사용)

### 4.3 가입 처리 흐름 (현행 MCP)

- **MyShareDataSvcImpl.moscDataSharingSave**
  - **opmdWorkDivCd를 "A"로 고정**하여 호출 → 즉, **가입(결합) 전용**으로만 사용 중.
  - 입력: `MyShareDataReqDto` (custId, ncn, ctn, **opmdSvcNo**).
  - opmdSvcNo는 셀프개통 완료 후 받은 전화번호(tlphNo) 또는 기존 결합 회선 번호.

- **insertOpenRequestAjax.do**
  - X69 사전체크 성공(shareDataYn="Y") 후 **X70 결합** 1회 호출.
  - 세션에 `opmdSvcNo` 저장 후 완료 뷰(reqSharingCompleteView)로 이동.

### 4.4 해지 처리 (설계·확장)

- **M플랫폼**: X70에 **opmdWorkDivCd = "C"** 로 호출하면 해지.
- **현행 MCP**: `MyShareDataSvcImpl`은 opmdWorkDivCd를 DTO에서 받지 않고 "A" 고정.  
  → **해지**를 넣으려면:
  - `MyShareDataReqDto.getOpmdWorkDivCd()`를 사용하거나,
  - 해지 전용 메서드/API에서 `moscDataSharingSave(..., opmdSvcNo, "C")` 호출하도록 수정 필요.
- **입력**: 해지할 데이터쉐어링 회선 번호 = **opmdSvcNo**, **opmdWorkDivCd = "C"**.

## 5. 데이터 구조 정리

### 5.1 요청 DTO (MyShareDataReqDto)

| 항목 | 설명 |
|------|------|
| custId | 고객번호 |
| ncn | 서비스 계약번호 (대표회선) |
| ctn | 대표회선 전화번호 |
| opmdSvcNo | 데이터쉐어링 대상 전화번호 (가입 시 개통된 번호, 해지 시 해지할 번호) |
| opmdWorkDivCd | 처리구분코드 A:결합, C:해지 (현행 서비스는 가입만 사용, "A" 고정) |
| selfShareYn | 셀프개통 사전체크 여부 "Y" 시 X69 호출 |
| iccId | 유심 일련번호 등 (가입 플로우) |
| name, birthday, contractNum 등 | 인증·세션용 |

### 5.2 응답 (MoscDataSharingResDto)

- **isSuccess()**: commHeader responseType 기준 성공 여부.
- **getSharingList()**: `List<OutDataSharingDto>`
  - **X69**: 사전체크 결과 목록. **rsltInd="Y"** 인 항목이 있으면 가입 가능.
  - **X71**: 현재 결합 중인 데이터쉐어링 회선 목록 (svcNo, efctStDt 등).

## 7. TOBE 개발 시 참고 사항

1. **사전체크**: X69 한 번 호출로 “가입 가능 여부” + “가입 가능 대상 목록” 확인. 목록 중 **rsltInd="Y"** 이 있으면 가입 가능.
2. **가입대상 조회**: X71로 “현재 결합 중인 데이터쉐어링 회선 목록” 조회 → 이미 결합 여부, 테이블 표시.
3. **가입**: X69 성공 후 X70 호출, **opmdSvcNo** = 개통된 쉐어링 번호, **opmdWorkDivCd = "A"**.
4. **해지**: X70 호출 시 **opmdWorkDivCd = "C"**, **opmdSvcNo** = 해지할 쉐어링 회선 번호. (현행 MCP는 미구현, DTO·서비스 확장 필요)
5. **에러 메시지**: ITL_SFC_E105(결합 중인 대상 없음), ITL_SFC_E106(가입 가능한 개통 고객 없음) 등 M플랫폼 응답 코드별 메시지 매핑 권장.

이 문서는 **데이터쉐어링 사전체크(X69)·가입대상 조회(X71)·가입/해지 처리(X70)** 서비스를 분석한 내용을 정리한 것이다.
