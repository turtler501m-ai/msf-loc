# AS-IS 서식지·폼 관련 분석

스마트서식지 TO-BE 개발 참고를 위해 MCP(AS-IS)의 **서식지 폼** 관련 기능(약관/서식 문구 조회, 신청서 저장, 이미징 연동)을 정리한 문서이다.

---

## 1. 개요

| 구분 | 설명 |
|------|------|
| **서식지 폼** | 약관·안내문 등 **문서 내용(HTML)** 을 코드로 관리하고, 화면/팝업에서 조회·표시하는 기능 |
| **서식지 저장** | 신청서 입력 데이터를 DB에 저장하고, 필요 시 SCAN 서버로 전송해 **이미징(서식지 이미지 생성)** 하는 흐름 |
| **관련 테이블** | **NMCP_FORM_DTL**(서식 문구), **NMCP_REQUEST_APD** 등(신청서 본문), **NMCP_REQUEST_*_TEMP**(임시저장) |

---

## 2. 서식 문구 조회 (약관/폼 내용)

### 2.1 API·역할 정리

| 구분 | URL | Controller | 용도 |
|------|-----|------------|------|
| **AJAX 조회** | `GET/POST /getFormDescAjax.do` | AppformController.getFormDesc | cdGroupId1, cdGroupId2로 **최신 버전 서식 1건** JSON 반환 (문자열만 필요할 때) |
| **팝업용 조회** | `GET/POST /termsPop.do` | FCommonController.termsPop | cdGroupId1, cdGroupId2, docType, expnsnStrVal 등으로 **서식 1건** 조회 후 **팝업에 표시용** (제목·이벤트 기간 반영) |
| **기준일 조회** | `GET/POST /termsPopByDate.do` | FCommonController.termsPopByDate | adminEventDate 기준으로 **해당 시점 적용 서식** 조회 (이벤트 기간 이중화 시 사용) |

### 2.2 getFormDesc (서식 문구 1건 조회)

**흐름**

1. **AppformController.getFormDesc(FormDtlDTO)**
   - `appformSvc.getFormDesc(formDtlDTO)` 호출
   - 반환값이 있으면 `docContent`에 대해 `StringEscapeUtils.unescapeHtml` 적용 후 반환

2. **AppformSvcImpl.getFormDesc**
   - `formDtlDao.getFormList(formDtlDTO)` 호출
   - `getFormList`: **FormDtlMapper.getFormList** 사용  
     - 조건: `USE_YN = 'Y'`, `DOC_TYPE = '01'`, (선택) cdGroupId1, cdGroupId2  
     - 정렬: CD_GROUP_ID2, DOC_VER  
   - 리스트가 있으면 **마지막 요소(최신 DOC_VER) 1건** 반환

3. **FormDtlMapper.getFormList (FormDtlMapper.xml)**

```xml
<select id="getFormList" resultType="com.ktmmobile.mcp.appform.dto.FormDtlDTO" parameterType="...">
    SELECT T1.CD_GROUP_ID1, T1.CD_GROUP_ID2, T1.DOC_VER, T1.DOC_CONTENT, T1.USE_YN, ...
    FROM NMCP_FORM_DTL T1, NMCP_CD_DTL T2
    WHERE T1.USE_YN = 'Y'
      AND T1.CD_GROUP_ID1 = T2.CD_GROUP_ID AND T1.CD_GROUP_ID2 = T2.DTL_CD
      AND T1.DOC_TYPE = '01'
      [AND T1.CD_GROUP_ID1 = #{cdGroupId1}] [AND T1.CD_GROUP_ID2 = #{cdGroupId2}]
    ORDER BY T1.CD_GROUP_ID2, T1.DOC_VER
</select>
```

- **NMCP_FORM_DTL**: 서식 상세 (문서 버전·내용)
- **NMCP_CD_DTL**: 코드명 등 (FORM_NM 등 조인)

### 2.3 termsPop / FormDtlView (팝업·기간 반영)

**FCommonController.termsPop**

- 파라미터: `FormDtlDTO` (cdGroupId1, cdGroupId2, docType, expnsnStrVal 등)
- docType 미입력 시 `"01"` 설정
- `formDtlSvc.FormDtlView(oFormDtlDTO)` 호출 → **FormDtlMapper.FormDtlView** 사용

**FormDtlView 쿼리 요약**

- **NMCP_FORM_DTL**에서 `CD_GROUP_ID1`, `CD_GROUP_ID2`, (선택) `DOC_VER`, `DOC_TYPE`, `EXPNSN_STR_VAL` 조건
- **이벤트 기간 이중화**:  
  - `EVENT_START_DT`~`EVENT_END_DT` + `START_HOUR`~`END_HOUR` 구간이면 `DOC_CONTENT`  
  - 그 외에는 `EVENT_START_DT_SEC`~`EVENT_END_DT_SEC` 구간의 `DOC_CONTENT_SEC`  
  - `ORDER BY DOC_VER DESC` 후 `ROWNUM = 1`로 **1건**만 반환
- 반환: CD_GROUP_ID1, CD_GROUP_ID2, DOC_VER, DOC_CONTENT, EVENT_START_DT, EVENT_END_DT 등

**termsPopByDate**

- `formDtlSvc.FormDtlViewByDate(oFormDtlDTO)` 호출
- FormDtlViewByDate: 동일 구조이나, **기준 시점**을 `#{adminEventDate}`로 넣어 해당 시점에 적용되는 DOC_CONTENT/DOC_CONTENT_SEC 선택

### 2.4 FormDtlDTO 주요 필드

| 필드 | 설명 |
|------|------|
| cdGroupId1 | 서식 그룹1 (예: FORMREQUIRED, FORMSELECT, FORMINFO, FormNotice, FormEtcCla) |
| cdGroupId2 | 서식 그룹2 (약관/문서별 코드) |
| docVer | 문서 버전 |
| docContent | 문서 본문 (HTML) |
| docType | 문서 유형 (01: 기본, 02/03: 제휴/위탁 등) |
| expnsnStrVal | 확장 조건 (요금제·제휴사 등 필터) |
| useYn | 사용 여부 |
| eventStartDt / eventEndDt | 이벤트 적용 기간 (기본) |
| docContentSec | 이벤트 기간 외 적용 본문 (이중화) |

### 2.5 프론트 사용 패턴

- **getFormDesc(cdGroupId1, cdGroupId2)**  
  - `appForm.js`, `appOutForm.js`, `moscSdsSvcRegView.js`, `reSpnsrPlcyDc.form.js`, `cvsSrcService.js` 등  
  - AJAX: `/getFormDescAjax.do` 또는 `/termsPop.do` (동기 호출로 문구 받아서 영역에 삽입)
- **약관 팝업**  
  - `openPage('termsPop', '/termsPop.do', param)` 형태로 많이 사용  
  - param: cdGroupId1, cdGroupId2, docType, expnsnStrVal 등
- **코드 그룹 예**  
  - FORMREQUIRED, FORMSELECT, FORMINFO (가입/신청 필수·선택·안내)  
  - FormNotice, FormEtcCla (공지·기타 약관)  
  - FormEtcCla + spnsrPlcyDcInfo, rePlcyClause 등 (정책/약관 문구)

## 3. 서식지 저장 (신청서 DB 저장)

### 3.1 저장 API

| 구분 | URL | Controller 메서드 | 설명 |
|------|-----|-------------------|------|

### 3.2 saveAppformDb 흐름 요약

1. **입력**: AppformReqDto (requestKey, 고객/상품/배송/대리점 등), (선택) resUniqId
2. **검증**: NICE 인증·세션·중복 등 (업무별 분기)
3. **저장**: AppformSvc 내부에서 **AppformDao** 호출  
   - **NMCP_REQUEST_APD** (신청서 마스터)  
   - **NMCP_REQUEST_APD_STATE** (진행 상태)  
   - **NMCP_REQUEST_APD_SALEINFO** (판매/요금 정보)  
   - **NMCP_REQUEST_APD_DLVRY** (배송 정보)  
   - 기타 NMCP_REQUEST_* 테이블
4. **응답**: RESULT_CODE, requestKey 등 Map 반환

### 3.3 임시 저장

- **NMCP_REQUEST_TEMP**, **NMCP_REQUEST_APD_TEMP**, **NMCP_REQUEST_*_TEMP** 등
- AppFormMapper: `insertAppFormTempSave`, `insertAppFormApdTempSave`, MERGE/UPDATE 등으로 임시 저장·복구

### 3.4 신청서 조회

- **AppformSvc.getMcpRequest(long requestKey)**  
  - AppformDao.getMcpRequest → **AppformMapper.getMcpRequest**  
  - NMCP_REQUEST_APD 등에서 requestKey 기준 1건 조회 (McpRequestDto)

## 4. 서식지 이미징 (SCAN 연동)

### 4.1 역할

- 저장된 **신청서(requestKey)** 데이터를 이용해 **서식지 XML**을 만들고, **SCAN 서버**로 보내 이미지(PDF 등)를 생성·보관하는 흐름

### 4.2 호출 경로

| 구분 | URL | Controller | 설명 |
|------|-----|------------|------|
| **이미징 요청** | `POST /appform/sendScanAjax.do` | AppformController.sendScan | requestKey 기준 서식지 SCAN 전송 |

### 4.3 ScanSvcImpl.prodSendScan 요약

1. **appformDao.getAppFormData(requestKey)**  
   - 서식지에 필요한 항목들을 Map으로 조회
2. **복호화**  
   - 주민번호·계좌·카드 등 암호화 컬럼 복호화
3. **settingSsnData / settingData**  
   - 주민번호 분석값·기타 설정값 세팅
4. **createXml(requestKey, cretId, requestData)**  
   - 서식지 **XML 파일 생성**  
   - 그룹코드·서식 위치 정보: `appformDao.getAppFormPointList(groupCode)`  
   - NMCP_FORM_DTL 기반 그룹/레이아웃 참고
5. **xmlFileSend(requestKey)**  
   - 생성된 XML을 **SCAN URL**로 전송
6. **전송 결과 UPDATE**  
   - 해당 requestKey 서식지 전송 상태 갱신

- 설정: `scan.app.path`, `scan.url`, `scan.V25.url` 등

## 7. TO-BE 참고 사항

- **서식 문구 조회**  
  - cdGroupId1/cdGroupId2(및 docType, expnsnStrVal) 기준 **최신 버전 1건** 조회와 동일한 규칙 유지 시, 프론트 연동·약관 팝업 치환 시 유리
- **이벤트 기간 이중화**  
  - FormDtlView/FormDtlViewByDate의 **기간별 DOC_CONTENT / DOC_CONTENT_SEC** 로직을 TO-BE에서도 필요하면 동일하게 설계
- **신청서 저장**  
  - 스마트서식지는 신청서 종류가 서비스변경/명의변경/서비스해지 등으로 한정되므로, NMCP_REQUEST_APD 계열 테이블 구조를 참고해 **요청 단위(requestKey)·상태·판매/배송** 구조를 단순화해 설계 가능
- **이미징(SCAN)**  
  - 서식지 XML 생성 규격·SCAN URL 연동은 AS-IS ScanSvcImpl·getAppFormData·getAppFormPointList 흐름을 참고해, TO-BE에서 동일 연동 또는 규격 매핑 설계

## 8. appForm.js 중심 서식지·폼 처리

포털 **appForm.js** (경로: `mcp-portal-web/src/resources/js/portal/appForm/appForm.js`) 기준으로, 서식지·폼이 어떻게 쓰이는지 정리한다. 모바일용 `mobile/appForm/appForm.js`는 구조가 유사하다.

### 8.1 전역·스텝 구조

| 구분 | 설명 |
|------|------|
| **pageObj** | 전역 객체. `requestKey`, `requestKeyTemp`, `resNo`, `step`, `applDataForm`, `niceResSmsObj` 등 폼/인증/진행 상태 보관 |
| **stepText** | `["본인확인·약관동의","유심정보·신분증 확인","가입신청 정보","부가서비스 정보","납부정보·가입정보 확인"]` — 5단계 스텝 제목 |
| **setStep(setStep)** | `pageObj.step` 갱신, 스테퍼 UI 갱신, 해당 step 영역만 표시 (`#step1`~`#step5`) |
| **APPL_DATA_FORM()** | `appForm/dataFormConfig.js`에 정의. 폼 제출용 **필드 집합 객체** 반환 (operType, requestKey, 고객/배송/부가/약관동의 플래그 등) |

- **operType**: NEW(신규), CHANGE(기변), EXCHANGE(기변), MOVE_NUM(번호이동) 등  
- **onOffType**: "0"(일반), "5"(셀프개통), "6" 등 — 서식 유형·버튼 문구 분기

### 8.2 서식 문구 조회 (getFormDesc / termsPop)

**getFormDesc(cdGroupId1, cdGroupId2)**

- **위치**: appForm.js 내부 함수 (약 5433행 근처)
- **동작**: `ajaxCommon.getSerializedData({ cdGroupId1, cdGroupId2 })` → **동기 AJAX** (`async:false`)로 `/termsPop.do` 호출 → 응답 JSON(FormDtlDTO) 반환
- **용도**: 약관/서식 문구를 **문자열로 받아** 특정 영역에 넣거나 조건 처리할 때 사용. appForm.js에서는 **getFormDesc를 호출하는 코드가 직접적으로 드러나지 않고**, 같은 파일에서 **termsPop 팝업**을 쓰는 패턴이 주로 사용됨

**약관 팝업 (termsPop)**

- **제휴/위탁 약관**  
  - `#clauseJehuButton` 클릭: `openPage('termsPop', '/termsPop.do', data, 0)`  
    - data: `cdGroupId1:'FORMREQUIRED', cdGroupId2:'clauseJehuFlag', docType:'02', expnsnStrVal:$("#jehuProdType").val(), name:$("#jehuProdName").val()`  
  - `#clausePartnerOfferButton` 클릭: `cdGroupId2:'clausePartnerOfferFlag', docType:'03', expnsnStrVal:$("#jehuPartnerType").val(), name:$("#jehuPartnerName").val()`
- **공통**: `fnSetEventId('clauseJehuFlag'/'clausePartnerOfferFlag')` 로 이벤트 ID 설정 후 팝업 오픈

**셀프개통 안내 (서식 없이 고정 문구)**

- `#btnClauseSimple` 클릭 시 **termsPop 호출 없이** 고정 HTML 배열을 `#termsModalContents`에 넣고 `#modalTerms` 다이얼로그 오픈 (개통 사기 경고 문구 등)

### 8.3 약관 동의·유효성

| 요소 | 설명 |
|------|------|
| **._agree** | Step1 필수 약관 동의 체크박스들. `#btnStplAllCheck`로 일괄 체크, `handleCommonAgreeClick()`에서 개별 체크 시 검증 갱신 |
| **#clauseSimpleOpen** | 셀프개통 필수 동의. `validator.config['clauseSimpleOpen'] = 'isNonEmpty'` 로 필수 체크 (onOffType "5"일 때) |
| **#clauseMoveCode** | 번호이동 시 필수 동의. `validator.config['clauseMoveCode'] = 'isNonEmpty'` |
| **._agreeInsr** | Step4 안심보험 등 부가 동의. `#btnInsrAllCheck`로 일괄 체크, `isValidateNonEmptyStep4()` 호출 |
| **isValidateNonEmptyStep1()** | Step1 약관·본인정보 등 필수 입력/동의 검사 후 다음 스텝 허용 여부 판단 |

- 서식 **내용**은 termsPop으로 보여주고, **동의 여부**는 체크박스와 validator로만 관리 (서식 문구를 JS에서 직접 getFormDesc로 받아 넣는 구간은 이 파일에서 드물게 사용).

### 8.4 임시 저장 (스텝별)

- **fnSaveTempStep1 ~ fnSaveTempStep5**: 각 스텝에서 “다음” 진행 전 호출
- **공통**: `paramObj = APPL_DATA_FORM()` 로 폼 객체 복사 후, 해당 스텝에서 입력한 값만 보강 (예: tmpStep="01"~"05", requestKey, operType, cstmrType, 고객/유심/배송/부가/결제 등)
- **전송**: `ajaxCommon.getSerializedData(paramObj)` → `POST /appForm/updateRequestTemp.do` (async:false)
- **결과**: RESULT_CODE가 STEP01/STEP02면 오류 메시지, 아니면 다음 스텝으로 진행

### 8.5 최종 저장 (fnSaveForm / fnSaveForm2)

**fnSaveForm()**

- **호출 시점**: Step5 “주문완료”/“개통요청” 등 최종 제출 시. 기변인 경우 Step4→5 전환 시 `fnSaveForm()`만 호출하고 setStep(5) 없이 완료 처리하는 분기 있음
- **역할**:  
  - `pageObj.applDataForm`에 주소·배송타입(dlvryType) 등 최종 반영  
  - 바로배송(dlvryType "02")이면 `reqDlvryForm` 직렬화 후 `/m/appForm/saveSelfDlvryAjax.do` 호출 → 성공 시 **fnSaveForm2()** 호출  
  - 그 외에는 바로 **fnSaveForm2()** 호출

**fnSaveForm2()**

- `pageObj.applDataForm.requestKey=0` 으로 초기화, `evntCdPrmt` 등 보강
- **varData = ajaxCommon.getSerializedData(pageObj.applDataForm)** 로 전체 폼 직렬화
- **POST `/appForm/saveAppformAjax.do`** 로 전송
- **성공 시**: `pageObj.requestKey = jsonObj.REQUEST_KET` 저장 후, 결제 유형에 따라 smartro 결제 호출 또는 **ajaxCommon.createForm + attachHiddenElement("requestKey", pageObj.requestKey) + formSubmit()** 으로 `/appForm/appFormComplete.do` 등 완료 페이지로 이동

### 8.6 APPL_DATA_FORM (dataFormConfig.js) 주요 필드

- **업무/상태**: operType, requestKey, onOffType, tmpStep, pstate, serviceType

### 8.7 appForm.js에서 쓰는 서식·폼 관련 URL 요약

| 용도 | URL | 비고 |
|------|-----|------|
