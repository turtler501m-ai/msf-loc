# 서비스변경 신청서 – M포털 소스 매핑

**기준**: `서비스변경신청서_UI설계서_분석.md`, `개발기능목록_담당.md`  
**목적**: 화면 ID별로 참고할 M포털(MCP) 소스 경로와 적용 포인트를 정리하여 개발에 반영

---

## 1. 공통 참조

| 구분 | M포털 소스 (workspace 기준) | 적용 포인트 |
|------|-----------------------------|-------------|
| 서비스변경·신청서 플로우 | mcp/mcp-portal-web/src/resources/js/portal/appForm/appForm.js | 단계 전환, 유효성, 메시지, conPreCheck 호출 |
| 백엔드 API | mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/appform/controller/AppformController.java | conPreCheckAjax, saveAppform, X01 등 연동 |
| M플랫폼 연동 | mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/common/mplatform/MplatFormService.java | X01, X20, X21, X38, X19, X89, X90, X32, X26, X28, X30 등 |

## 4. mform-web 적용 체크

- [x] ChangeType: 동시 변경 불가 에러 메시지 설계서 문구
- [x] ChangeCust: 인증 성공/실패 메시지 설계서 문구
- [x] ChangeAgree: 서명 안내 문구, 등록 확인·완료/실패 메시지
- [x] ServiceCompleteView: 서비스변경 완료 시 고객명 마스킹·설계서 문구·발송 UI
