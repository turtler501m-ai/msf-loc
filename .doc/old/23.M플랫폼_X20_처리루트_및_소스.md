# M플랫폼 X20(부가서비스 조회) 처리 루트 및 현재 소스

X20은 **가입중 부가서비스 조회** API이며, 현재 M포털(mcp)에서 호출되는 전체 루트와 관련 소스 위치를 정리한 문서입니다.

---

## 1. 처리 루트 요약

```
[화면/컨트롤러] → [서비스] → MplatFormService.getAddSvcInfoDto()
       → getParamMap(..., "X20") → MplatFormServerAdapter.callService()
       → HTTP POST (juice.url) → [msc-prx] /mPlatform/serviceCall.do
       → MplatformService.mPlatformServiceWebServiceCall()
       → SOAP 호출: mPlatForm.url + "/MoscRegSvcService" (실제 M플랫폼)
       → 응답 XML → MpAddSvcInfoDto.toResponseParse()
```

- **LOCAL** 환경: 실제 연동 없이 `getVo(20, vo)` 로 Mock XML 반환.
- **운영/DEV/STG** 환경: `juice.url`(msc-prx) 경유 후 M플랫폼 SOAP 서비스 호출.

---

## 3. mcp-portal-was (M포털 WAS) — X20 진입점

### 3.1 MplatFormService.getAddSvcInfoDto()

**파일:** `mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/common/mplatform/MplatFormService.java`

- **라인:** 526~539
- **역할:**
  - `getParamMap(ncn, ctn, custId, "X20")` 로 전송 파라미터 생성 (ncn, ctn, custId, userId, appEventCd=X20).
  - **LOCAL:** `getVo(20, vo)` 로 Mock XML 세팅 후 `vo` 반환 (실제 연동 없음).
  - **그 외:** `mplatFormServerAdapter.callService(param, vo)` 로 연동.
- **반환:** `MpAddSvcInfoDto` (응답 파싱 후 부가서비스 목록 등 보유).

### 3.2 getParamMap()

**파일:** 동일 `MplatFormService.java` (2118~2134 라인 근처)

- **X20 파라미터:** `ncn`, `ctn`, `custId`, `userId`(세션), `appEventCd` = `"X20"`.

### 3.3 LOCAL Mock — getVo(20, vo)

**파일:** 동일 `MplatFormService.java`

- **라인:** case 20 (2205~2209 근처) — 주석: `//가입중부가서비스조회`
- **내용:** `appEventCd` X20 형식의 Mock XML (`outDto` 내 `effectiveDate`, `prodHstSeq`, `soc`, `socDescription`, `socRateValue` 등)을 넣어 `vo.setResponseXml()` 후 `vo.toResponseParse()` 호출.

### 3.4 MplatFormServerAdapter (실제 HTTP 호출)

**파일:** `mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/common/mplatform/MplatFormServerAdapter.java`

- **설정:** `@Value("${juice.url}")` → 예: `http://{host}:7006/mPlatform/serviceCall.do`
- **동작:** `callService(param, vo)` 에서 param을 `getURL()`로 쿼리스트링 형태로 만든 뒤 **UTF-8 인코딩**하여 `getURL` 이라는 단일 파라미터로 **POST**.
- **응답:** 수신 XML을 `vo.setResponseXml(responseXml)` 후 `vo.toResponseParse()` 호출.

### 3.5 응답 파싱 — MpAddSvcInfoDto

**파일:** `mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/common/mplatform/vo/MpAddSvcInfoDto.java`

- **역할:** `CommonXmlVO` 상속, `parse()` 에서 `outDto` 리스트를 파싱.
- **파싱 필드:** `socDescription`, `socRateValue`, `effectiveDate`, `soc` 등 → `MpSocVO` 리스트로 저장.
- **추가:** 과세/비과세, 할인/유료 건수·금액 등 계산.

## 4. msc-prx (M플랫폼 프록시) — X20 연동

### 4.1 진입 URL

- **URL:** `{juice.url}` = `http://{host}:7006/mPlatform/serviceCall.do` (환경별 properties 참고)
- **메서드:** POST
- **파라미터:** `getURL` (ncn, ctn, custId, userId, appEventCd=X20, ip, url, mdlInd 등을 `&`로 연결 후 UTF-8 인코딩한 값)

### 4.2 MplatformController

**파일:** `mcp/msc-prx/src/main/java/com/ktis/mcpif/mPlatform/web/MplatformController.java`

- **라인:** 50~62
- **매핑:** `@RequestMapping("/mPlatform/serviceCall.do")`
- **동작:** `getURL` 디코딩 후 `mPlatformService.mPlatformServiceWebServiceCall(decodeURL)` 호출.

### 4.3 MplatformService (SOAP 호출)

**파일:** `mcp/msc-prx/src/main/java/com/ktis/mcpif/mPlatform/service/MplatformService.java`

- **X20 매핑 (345~352 라인):**
  - `getEventCodeInfo("X20", "service")` → **"MoscRegSvcService"** (SOAP 서비스 경로)
  - `getEventCodeInfo("X20", "info")` → **"moscRegSvcInfo"** (SOAP 오퍼레이션/요청 래퍼)
  - `getEventCodeInfo("X20", "vo")` → **"MoscRegSvcInfoInVO"** (요청 VO 요소명)

- **호출 URL:** `propertiesService.getString("mPlatForm.url")` + **"/MoscRegSvcService"** (실제 M플랫폼 SOAP 엔드포인트)

- **요청 XML 생성:** `xmlMessageCreate(reqMap)`  
  - X20은 **else** 분기(1442~1477 라인):  
    `initXmlMessageCreate("start")` + `initHeaderCreate(eventCd)` + `getInDto(reqMap)`  
  - SOAP Body: `<sel:moscRegSvcInfo><MoscRegSvcInfoInVO>...</MoscRegSvcInfoInVO></sel:moscRegSvcInfo>`  
  - X20용 `getInDto()` 전용 분기는 없어 **공통 헤더 + selfCareIn(ctn, custId, ncn, clntIp, clntUsrId)** 만 전송.

- **연동:** SOAP Connection으로 `requestUrl`(mPlatForm.url + "/MoscRegSvcService") 호출 후 응답 XML 반환 → portal-was 쪽에서 수신해 `MpAddSvcInfoDto.toResponseParse()` 로 파싱.

## 5. 설정 (juice.url / mPlatForm.url)

| 설정키 | 용도 | 예시 위치 |
|--------|------|-----------|
| **juice.url** | M포털 WAS → msc-prx 호출 주소 | `mcp-portal-was/.../configuration/properties/mplatform.properties` (resources-local, resources-dev, resources-stg, resources-prd 등) |
| **mPlatForm.url** | msc-prx → 실제 M플랫폼 SOAP Base URL | msc-prx 프로젝트 내 properties (해당 서버 URL) |

- **juice.url** 예: `http://14.49.35.116:7006/mPlatform/serviceCall.do`, `http://10.20.28.38:7006/mPlatform/serviceCall.do` 등

## 6. 소스 존재 여부 요약

| 구분 | 소스 존재 | 위치 |
|------|-----------|------|
| X20 호출 진입 (getAddSvcInfoDto) | 있음 | mcp-portal-was `MplatFormService.java` |
| X20 파라미터 생성 (getParamMap) | 있음 | 동일 |
| X20 LOCAL Mock (getVo case 20) | 있음 | 동일 |
| HTTP 클라이언트 (MplatFormServerAdapter) | 있음 | mcp-portal-was `MplatFormServerAdapter.java` |
| X20 응답 VO/파싱 (MpAddSvcInfoDto) | 있음 | mcp-portal-was `vo/MpAddSvcInfoDto.java` |
| msc-prx 수신 (serviceCall.do) | 있음 | msc-prx `MplatformController.java` |
| X20 → MoscRegSvcService / moscRegSvcInfo 매핑 | 있음 | msc-prx `MplatformService.java` (getEventCodeInfo, xmlMessageCreate else 분기) |
| SOAP 요청 생성 (getInDto 등) | 있음 | 동일 (X20은 공통 헤더+selfCareIn만 사용) |

**정리:** M플랫폼 X20으로 처리되는 루트와 그에 필요한 **현재 소스는 모두 존재**하며,  
portal-was → juice.url(msc-prx) → M플랫폼 MoscRegSvcService SOAP 호출 → MpAddSvcInfoDto 파싱까지 한 흐름으로 구현되어 있습니다.
