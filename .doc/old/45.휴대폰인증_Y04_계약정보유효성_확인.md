# 휴대폰 인증 및 M플랫폼 계약정보 유효성체크(Y04) 확인

## 1. 화면설계서 요구사항

기기변경, **서비스변경**, **명의변경**, **서비스해지**에서 **휴대폰번호 인증을 필수**로 진행한다.

## 2. M플랫폼 API – 계약정보 유효성체크 (Y04)

- **인터페이스**: IF_0179 / Y04  
- **설명**: 계약정보 유효성체크  
- **M플랫폼 서비스**: `MoscSvcContService.moscSvcContValdChk`  
- **규격**: 연동규격서_MVNO-Selfcare-API_v4.84.xlsx  

### Y04 Input (필수 항목)

| 항목 | 필수 | 설명 |
|------|------|------|
| Client IP | ○ | 요청 클라이언트 IP |
| cIntUsrId | ○ | 사용자 아이디 |
| ncn | ○ | 사용자 서비스 계약번호 (계약번호) |
| ctn | ○ | 사용자 전화번호 (휴대폰번호) |
| custId | ○ | 고객번호 |

- **휴대폰번호(ctn)**: 고객이 입력하는 값.
- **고객번호(custId), 계약번호(ncn)**: 개통 시 KT(M전산)에서 부여·관리하는 값으로, 고객이 직접 알지 못함.

## 3. M포탈(기기변경) 휴대폰 인증 흐름 (확인된 사항)

- 휴대폰 인증 요청 시 **M전산 DB**에서 **이름 + 휴대폰번호**로 **고객번호(custId)** 및 **계약번호(ncn)** 를 조회.
- 조회한 custId, ncn과 사용자 입력 ctn, 기타 필수값(Client IP, cIntUsrId)을 넣어 **M플랫폼 Y04(계약정보 유효성체크)** 를 호출.

MCP 소스 기준:

- `MypageMapper.xml` – `selectContractNum`:  
  `MSP_JUO_SUB_INFO@DL_MSP`(M전산)에서 `SUBSCRIBER_NO`(휴대폰번호), `SUB_LINK_NAME`(이름)으로 `CONTRACT_NUM`(계약번호) 조회.
- `MypageUserController` 등에서 이름·휴대폰번호로 `selectContractNum(name, phone)` 호출 후, 조회한 계약번호 등으로 이후 M플랫폼 연동 수행.

즉, **M전산 연동(이름+휴대폰 → ncn/custId 조회) → Y04 호출** 구조로 이해한 내용이 맞습니다.

## 4. M모바일 스마트서식지(mform) – 방안 A 적용 현황

### 4.1 휴대폰 인증 사용처

- **서비스변경**: `ChangeTypeCust.vue` – 인증 후 고객정보/서비스 선택  
- **명의변경**: `IdentCust.vue` – `/v1/ident/join-info`  
- **서비스해지**: `CancelCust.vue` – `/v1/cancel/join-info`  

위 모두에서 “이름·생년월일(또는 법인등록번호)·휴대폰번호” 입력 후 [인증] 시 **join-info** 계열 API를 호출합니다.

### 4.2 join-info API (mform-api) – 방안 A 구현 완료

- **컨트롤러**: `SvcChgInfoRestController`
- **경로**:  
  - 서비스변경: `POST /api/v1/join-info`  
  - 명의/해지: `POST /api/v1/ident/join-info`, `POST /api/v1/cancel/join-info`  
- **요청** (`JoinInfoRequest`): **name(필수)**, ctn(필수), ncn/custId(선택, 인증 전 빈값).
- **흐름 (M포탈과 동일)**  
  1. **M전산 조회**: 이름(name) + 휴대폰(ctn) → `SvcChgMypageMapper.selectContractInfo` → ncn, custId  
  2. **Y04 호출**: `MplatFormSvc.contractValdChk(clientIp, cIntUsrId, ncn, ctn, custId)`  
  3. **응답**: 성공 시 `JoinInfoResponse`에 success, ncn, custId, addr, email, homeTel 등 반환
- **구성**: 매퍼(`SvcChgMypageMapper`), 서비스(`SvcChgInfoService`), 컨트롤러(`SvcChgInfoRestController`) 동일하게 구현.

## 5. 스마트서식지에서 Y04를 쓰기 위한 선택지

화면설계서대로 **기기변경·서비스변경·명의변경·서비스해지**에서 휴대폰 인증 시 **Y04를 필수 호출**한다고 하면, 아래 두 가지 중 하나가 필요합니다.

### 방안 A: M전산 연동 (M포탈과 동일)

- 스마트서식지 백엔드(mform-api)가 **M전산**에 연동 가능한 경우:
  - 휴대폰 인증 요청 시 **이름 + 휴대폰번호(ctn)** 로 M전산에서 **custId, ncn** 조회.
  - 조회 결과 + ctn + Client IP, cIntUsrId 등으로 **M플랫폼 Y04** 호출.
- M포탈 기기변경과 동일한 패턴이므로, M전산 조회 조건(테이블/뷰, 권한 등)만 확보하면 재사용 가능.

### 방안 B: M플랫폼에서 ncn/custId 조회 API 제공

- 스마트서식지가 **M전산에 직접 연동되지 않는** 경우:
  - 고객이 입력 가능한 것은 **이름, 생년월일(또는 법인등록번호), 휴대폰번호(ctn)** 뿐이므로, **ncn·custId를 스마트서식지가 스스로 알 수 없음.**
  - 이 경우 **M플랫폼**에  
    “이름·생년월일(또는 법인등록번호)·휴대폰번호 등으로 **계약번호(ncn)·고객번호(custId) 조회**” API가 있으면:
    - 1) 해당 조회 API로 ncn, custId 취득  
    - 2) 그 결과로 Y04 호출  
  - 이런 **“ncn/custId 조회용 API”** 가 규격에 없다면, M플랫폼 측에 요청이 필요합니다.

## 6. 정리

- 화면설계서: 기기변경·서비스변경·명의변경·서비스해지에서 **휴대폰 인증 필수** → **Y04(계약정보 유효성체크) 호출**이 필요하다는 이해가 맞습니다.
- Y04 Input: **Client IP, cIntUsrId, ncn, ctn, custId** 모두 필수이며, **ctn은 고객 입력**, **ncn·custId는 KT(M전산) 관리값**이라는 정리도 맞습니다.
- M포탈 기기변경: **M전산에서 이름+휴대폰으로 ncn(·custId) 조회 후 Y04 호출**하는 흐름으로 확인했습니다.
- M모바일 스마트서식지: **방안 A 적용 완료.** M전산 조회(매퍼) → Y04 호출(서비스)로 M포탈과 동일 구조 구현. M전산 미연동 환경에서는 `mform.join-info.mock-when-no-db=true`(로컬 한정) 또는 M전산 Oracle URL 설정 필요.

이 문서는 위 내용을 개발/요구사항 확인용으로 정리한 것입니다.

### 7.1 스마트서식지(mform) – 백엔드 (mform-api) – 방안 A 반영

#### 7.1.1 join-info API 컨트롤러 (SvcChgInfoRestController)

**파일**: `mform-api/.../formSvcChg/controller/SvcChgInfoRestController.java`

- join-info 는 `SvcChgInfoService.joinInfo(req, clientIp)` 에 위임. `request.getRemoteAddr()` 로 Client IP 전달.

```java
@PostMapping("/join-info")
public ResponseEntity<JoinInfoResponse> joinInfo(@RequestBody JoinInfoRequest req, HttpServletRequest request) {
    String clientIp = request.getRemoteAddr();
    JoinInfoResponse res = svcChgInfoService.joinInfo(req, clientIp);
    if (!res.isSuccess() && res.getMessage() != null) {
        return ResponseEntity.badRequest().body(res);
    }
    return ResponseEntity.ok(res);
}
```

#### 7.1.1-2 M전산 조회 매퍼 (M포탈 MypageMapper와 동일 구조)

**파일**: `mform-api/.../formSvcChg/mapper/SvcChgMypageMapper.java`, `resources/mapper/SvcChgMypageMapper.xml`

- `selectContractInfo(Map)` : name, mobileNo → ContractInfoDto(ncn, custId)
- SQL: `MSP_JUO_SUB_INFO@DL_MSP` 에서 CONTRACT_NUM, CUSTOMER_ID 조회 (이름·휴대폰 조건).

#### 7.1.1-3 휴대폰 인증 서비스 (SvcChgInfoService)

**파일**: `mform-api/.../formSvcChg/service/SvcChgInfoService.java`, `SvcChgInfoServiceImpl.java`

- `joinInfo(JoinInfoRequest, clientIp)` : ctn/name 검증 → M전산 조회 → Y04 `MplatFormSvc.contractValdChk(...)` 호출 → 성공 시 ncn/custId 포함 응답.
- 로컬 M전산 미연동 시: `mform.join-info.mock-when-no-db=true` 이면 Mock ncn/custId 로 Y04까지 진행.

#### 7.1.2 JoinInfoRequest DTO

**파일**: `mform-api/src/main/java/com/ktmmobile/msf/formSvcChg/dto/JoinInfoRequest.java`

- **name(필수)**: 고객명, M전산 조회 키.
- **ctn(필수)**: 휴대폰번호.
- **ncn, custId**: 인증 전에는 생략 가능, M전산 조회 결과로 서버에서 채움.

#### 7.1.3 JoinInfoResponse DTO

**파일**: `mform-api/src/main/java/com/ktmmobile/msf/formSvcChg/dto/JoinInfoResponse.java`

- 인증 성공 시 **ncn**, **custId** 포함 (이후 부가서비스/요금제 등 API에서 사용). addr, email, homeTel, initActivationDate, message 동일.

#### 7.1.4 Y04 계약정보 유효성체크 (MplatFormSvc)

**파일**: `mform-api/.../common/mplatform/MplatFormSvc.java`

- `contractValdChk(clientIp, cIntUsrId, ncn, ctn, custId)` : Y04 호출. LOCAL 이면 Mock 성공, 아니면 adapter.callService(appEventCd=Y04). 응답 VO: `Y04ValdChkVo`.

#### 7.1.5 M플랫폼 파라미터 맵 (X20 등 기존 연동 참고)

**파일**: `mform-api/src/main/java/com/ktmmobile/msf/common/mplatform/MplatFormSvc.java`

- X20 등 기존 연동은 `getParamMap(ncn, ctn, custId, appEventCd)` 사용.  
- Y04 연동 시 **Client IP, cIntUsrId** 등을 추가한 전용 파라미터 맵/호출 메서드 추가 필요.

```java
private HashMap<String, String> getParamMap(String ncn, String ctn, String custId, String eventCd) {
    HashMap<String, String> param = new HashMap<>();
    String userId = mplatformUserId != null ? mplatformUserId : "";
    param.put("ncn", ncn != null ? ncn : "");
    param.put("ctn", ctn != null ? ctn : "");
    param.put("custId", custId != null ? custId : "");
    param.put("userId", userId);
    param.put("appEventCd", eventCd);
    return param;
}
```

  }
}

### 7.3 M포탈(MCP) – M전산 계약번호 조회

#### 7.3.1 M전산 DB 조회 (계약번호)

**파일**: `mcp/mcp-api/src/main/resources/mcp/mapper/MypageMapper.xml`

```xml
<select id="selectContractNum" parameterType="java.util.Map" resultType="java.lang.String">
    SELECT CONTRACT_NUM, SUBSCRIBER_NO AS MOBILE_NO
      FROM MSP_JUO_SUB_INFO@DL_MSP
     WHERE SUB_STATUS <![CDATA[<>]]> 'C'
       AND SUBSCRIBER_NO = TRIM(#{mobileNo})
       AND UPPER(REPLACE(SUB_LINK_NAME,' ','')) = UPPER(REPLACE(#{name},' ',''))
</select>
```

- **DL_MSP**: M전산 DB 링크.
- **mobileNo**: 휴대폰번호, **name**: 이름 → **CONTRACT_NUM**(ncn) 조회.  
- Java 반환 타입이 `String` 이므로 실제로는 첫 번째 컬럼(CONTRACT_NUM)만 반환. 고객번호(custId)가 필요하면 동일 뷰/테이블에서 컬럼 추가 조회 필요.

#### 7.3.2 계약번호 API 엔드포인트

**파일**: `mcp/mcp-api/.../mypage/controller/MypageController.java`

```java
@RequestMapping(value = "/mypage/contractNum", method = RequestMethod.POST)
public String contractNum(@RequestBody HashMap<String, String> map) {
    String contractNum = null;
    try {
        contractNum = mypageMapper.selectContractNum(map);
    } catch (Exception e) {
        throw new McpCommonJsonException(COMMON_EXCEPTION);
    }
    return contractNum;
}
```

- **map** 에 `name`, `mobileNo` (휴대폰번호) 전달.

#### 7.3.3 비회원 휴대폰 인증 시 계약번호 조회

**파일**: `mcp/mcp-portal-was/.../mypage/controller/MypageUserController.java` (약 806~859행)

- 이름(name), 휴대폰(phone)으로 **selectContractNum(name, phone)** 호출.
- 계약번호 없으면 `RESULT_CODE "0002"`, 사용자 정보 불일치 처리.
- 계약번호 있으면 선납 여부 등 추가 검사 후 **contractNum** 을 응답에 담아 반환 (`rtnMap.put("contractNum", contractNum)`).
- 이후 해당 세션/요청에서 **ncn(contractNum)** 을 보유한 상태로 M플랫폼(Y04 등) 연동 가능.

#### 7.3.4 selectContractNum 서비스 구현

**파일**: `mcp/mcp-portal-was/.../mypage/service/MypageUserServiceImpl.java`

```java
@Override
public String selectContractNum(String name, String phone) {
    RestTemplate restTemplate = new RestTemplate();
    Map<String, String> params = new HashMap<>();
    params.put("name", name);
    params.put("mobileNo", phone);
    String contractNum = restTemplate.postForObject(
        apiInterfaceServer + "/mypage/contractNum", params, String.class);
    return contractNum;
}
```

- WAS에서 MCP API(`/mypage/contractNum`)를 호출해 M전산 조회 결과(계약번호)를 받는 구조.
