## 개발 계획

### 1. 사업·구조 전제

| 구분 | 기존 (M포탈) | 신규 (스마트서식지) |
|------|--------------|---------------------|
| **사용처** | M포탈 웹 | 대리점용 스마트서식지 |
| **역할** | 웹에서 신청서·서비스 처리 | M포탈과 **동일 기능** 제공 |
| **개발 방식** | 기존 M포탈 소스 | **신규 개발** (프론트 + **백엔드 모두 신규**) |
| **연동 API** | M플랫폼 ↔ KT(KOS) 연계 | **동일하게 구현** (M포탈 API 스펙·연동 방식 따름) |
| **화면** | M포탈 웹 UI | 화면은 **조금 다르게** 구성 (mform-web) |

- **정리**: 스마트서식지는 **프론트(mform-web)·백엔드 모두 신규로 개발**한다. 다만 **연동 API는 M포탈과 동일하게 구현**하고(요청/응답·M플랫폼·KT(KOS) 연동 스펙 동일), 기존 M포탈 코드를 재사용하는 것이 아니다.

---

### 2. 연동 구조 (신규 개발, 연동 API 동일 구현)

```
[대리점] 스마트서식지 프론트 (mform-web, Vue3·JS)  ←  신규 개발
    │
    │  API 호출 (스펙은 M포탈과 동일)
    ▼
[스마트서식지 백엔드]  ←  신규 개발 (연동 API 동일하게 구현)
    │
    │  M플랫폼·KT(KOS) 연동 (M포탈과 동일 스펙/방식)
    ▼
[M플랫폼]
    │
    ▼
[KT(KOS)]
```

- **프론트·백엔드 모두 신규 개발.** M포탈 소스 재사용이 아님.
- **연동 API**: M포탈에서 쓰는 API 스펙·M플랫폼/KT(KOS) 연동 방식을 **동일하게 구현**하여 새 백엔드에서 제공.

---

### 3. 기술 스택 (mform-web 기준)

- **언어**: JavaScript (TypeScript 미사용)
- **프레임워크**: Vue 3, Vue Router 5, Pinia
- **빌드/스타일**: Vite 7, Tailwind CSS 4
- **기타**: reka-ui, @vueuse/core 등 (상세는 `doc/Mform-Web_폴더구조_분석.md` 참조)

---

### 4. 연동 API 동일 구현 원칙 (백엔드 신규 개발)

- 기능목록(`스마트서식지-AN-01-개발과제및기능목록_20260225.md`)의 **연동/API 컬럼**에 기재된 API를 기준으로 **스마트서식지 백엔드를 신규 개발**한다.
- **연동 API 스펙**은 M포탈과 **동일하게 구현** (요청/응답 형식, M플랫폼 호출, KT(KOS) 연동 방식 동일). 예: X01 가입정보조회, X20 부가서비스 조회, X21/X38 부가서비스 신청/해지, X19 요금상품 변경 등.
- 코드는 재사용이 아니라 **신규 작성**하며, M포탈 연동 명세·기능목록을 참고하여 **동일한 API·연동**을 새 백엔드에서 제공.
