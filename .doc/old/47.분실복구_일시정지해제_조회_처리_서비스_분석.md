# 분실복구/일시정지해제 신청 — 조회·처리 서비스 분석

**작성 목적**: 서비스변경 신청서 내 "분실복구/일시정지해제 신청"(LOST_RESTORE) 관련 MCP(AS-IS)의 조회·처리 서비스 흐름을 정리하여 TOBE(스마트서식지) 개발 시 참고.

**관련 문서**: `40.서비스변경신청서_UI설계서_분석.md` 3.3.6, `15.개발기능목록_ASIS_TOBE_기능분석명세서.md` (C27~C30, C140~143)

---

## 1. 개요

- **분실복구**: 분실신고 상태를 해제하여 서비스 복구 (M-Platform **X35** 분실신고 해제).
- **일시정지해제**: 일시정지 상태인 회선을 해제 (M-Platform **X28** 해제가능여부 조회 → **X30** 일시정지해제신청).

AS-IS에서는 **마이페이지**에서 별도 메뉴로 제공되며, 서비스변경 신청서(스마트서식지)에서는 **LOST_RESTORE** 옵션 선택 시 상품 단계에서 동일 기능을 제공하는 구조로 설계됨.

## 2. 일시정지 관련 서비스 (조회·해제)

### 2.1 API 매핑

| 구분 | M-Platform 이벤트 | 메서드(AS-IS) | 설명 |
|------|-------------------|----------------|------|
| 일시정지 이력 조회 | **X26** | `suspenPosHis` | 일시정지 이용/잔여 횟수·일수·만료예정일 등 |
| 일시정지 가능 여부 조회 | **X27** | `suspenPosInfo` | 일시정지 **신청** 가능 여부 (사유코드 CR01/CS01) |
| 일시정지 **해제** 가능 여부 조회 | **X28** | `suspenCnlPosInfo` | 해제 가능 여부, 정지일자, 발/착신구분 등 |
| 일시정지 신청 | **X29** | `suspenChg` | 일시정지 신청 (기간·비밀번호·발착신코드 등) |
| 일시정지 해제 신청 | **X30** | `suspenCnlChgIn` | 일시정지 해제 처리 |

### 2.2 X26 — 일시정지 이력 조회

- **Controller**: `PauseController.suspendView01()` (첫 화면 로드 시), `suspenPosHisPop()` (이력 조회 팝업).
- **호출**: `mPlatFormService.suspenPosHis(ncn, ctn, custId, termGubun)`  
  - `termGubun`: `"1"` = 최근 1년, `"A"` = 전체.
- **요청 파라미터**: `ncn`(서비스계약번호 9자리), `ctn`(전화번호 11자리), `custId`, `termGubun`.
- **응답 VO**: `MpSuspenPosHisVO`
  - `subStatus`: 현재상태 (A: 개통, S: 일시정지)
  - `susCnt`, `susDays`: 일시정지 누적 횟수/일수
  - `remainSusCnt`: 잔여 정지 횟수
  - `solSusDays`: 이용정지기간일수
  - `expectSpDate`: 일시정지만료예정일
  - `itemList`: `bkInfoDto` 리스트 (colSusDays, csaActivityRsnDesc, remainSusCnt, remainOgDays 등)

### 2.3 X27 — 일시정지 가능 여부 조회

- **Controller**: `PauseController.getSuspenPosInfoAjax()` — 일시정지 **신청** 전 인증 단계에서 호출.
- **호출**: `mPlatFormService.suspenPosInfo(ncn, ctn, custId, stopRsnCd)`  
  - `stopRsnCd`: 일시정지사유코드. CR01(고객요청), CS01(고객요청 M2M/MVNO) 등.
- **응답 VO**: `MpSuspenPosInfoVO` (map)
  - `ctnStatus`: 계약상태
  - `rsltInd`: 일시정지 가능 여부 (Y/N)
  - `rsltMsg`, `insurMsg`

동일 Ajax에서 **X33 분실신고가능여부**도 호출하여 `runMode` 등을 함께 반환. 분실신고 상태(runMode=U)이면 일시정지 신청/해제 불가 안내.

### 2.4 X28 — 일시정지 해제 가능 여부 조회

- **Controller**: `PauseController.suspendView03()` (일시정지 해제 화면 진입 시).
- **호출**: `mPlatFormService.suspenCnlPosInfo(ncn, ctn, custId)` — 추가 파라미터 없음.
- **응답 VO**: `MpSuspenCnlPosInfoInVO` (map)
  - `rsltInd`: 일시정지 **해제** 가능 여부 (Y)
  - `rsltMsg`: 메시지
  - `subStatusDate`: 일시정지일자 (yyyyMMddHHmmss)
  - `sndarvStatCd`: 발/착신구분코드 (01 발신정지 등)
  - `subStatusRsnCode`, `rsnDesc`, `ctnStatus`

해제 불가 시(`rsltInd != "Y"`) 예외 처리 후 "온라인으로 일시 정지 해제가 불가합니다. 고객센터(1899-5000)로 문의를 부탁 드립니다." 메시지.

### 2.5 X29 — 일시정지 신청

- **Controller**: `PauseController.suspenChgAjax()`.
- **호출**: `mPlatFormService.suspenChg(ncn, ctn, custId, reasonCode, userMemo, cpDateYn, cpEndDt, cpStartDt, cpPwdInsert)`  
  - `reasonCode`: 발착신 상태코드 (99:정지없음, 01:발신정지, 02:착신정지, 03:착발신정지)
  - `cpDateYn`: 일시정지 기간여부 (Y/N)
  - `cpStartDt`, `cpEndDt`: 시작/만료일자 (yyyymmdd)
  - `cpPwdInsert`: 일시정지 비밀번호 (4~8자리 숫자)
- **응답 VO**: `MpSuspenChgVO` (parse 비어 있음, commHeader 성공 여부로 판단).

전제 조건: 본인인증(NICE), step 검증, X27로 신청 가능 여부 확인 후 진행.

### 2.6 X30 — 일시정지 해제 신청

- **Controller**: `PauseController.suspenCnlChgInAjax()`.
- **호출**: `mPlatFormService.suspenCnlChgIn(ncn, ctn, custId, pwdType, cpPwdInsert)`  
  - `pwdType`: PP(일시정지), CP(개인정보 암호)
  - `strPwdNumInsert`: 일시정지 비밀번호 4~8자리 (미입력 시 AS-IS에서는 임의값 "12345678" 설정)
- **응답 VO**: `MpSuspenCnlChgInVO` (parse 비어 있음).

전제: X28로 해제 가능 여부 확인 후, 본인인증·step 검증 후 호출.

## 3. 분실신고/분실복구 관련 서비스

### 3.1 API 매핑

| 구분 | M-Platform 이벤트 | 메서드(AS-IS) | 설명 |
|------|-------------------|----------------|------|
| 분실신고 가능 여부 조회 | **X33** | `pcsLostInfo` | 분실신고 가능 여부, 현재 분실신고 상태 등 |
| 분실신고 신청 | **X34** | `pcsLostChg` | 분실신고 접수 (정지 유형·비밀번호 등) |
| 분실신고 해제 (분실복구) | **X35** | `pcsLostCnlChg` | 분실복구 처리 |

### 3.2 X33 — 분실신고 가능 여부 조회

- **Controller**: `PauseController.pcsLostInfoAjax()` (분실신고 화면), `getSuspenPosInfoAjax()`에서도 호출해 runMode 등 반환.
- **호출**: `mPlatFormService.pcsLostInfo(ncn, ctn, custId)` — 추가 파라미터 없음.
- **응답 VO**: `MpPcsLostInfoVO` (map)
  - `runMode`: I = 신청 가능, U = 이미 분실신고 상태
  - `rsltCd`: Y 만 정상
  - `rsltMsg`: 메시지
  - `subStatusLastAct`: SUS = 일시정지 상태, NAC 등
  - `asfdYn`, `coldLinqStatus`: 미납 등 (VO에는 `coldLinqStatus`로 파싱, XML/모킹은 `coldeLinqStatus` 사용 가능 — TOBE 시 통일 권장)

**검증 로직 (AS-IS)**  
- `runMode` null/공백 → "유심이 휴대폰과 분리되어있는 상태에서는 온라인으로 분실신고가 불가합니다."  
- `rsltCd != "Y"` → rsltMsg 반환  
- `runMode != "I"` → 이미 신청된 상태 등  
- `subStatusLastAct == "SUS"` → 일시정지 상태에서는 분실신고 불가  
- `coldeLinqStatus == "D"` → 미납 시 불가  

(참고: Controller는 `coldeLinqStatus` 키로 조회하지만 VO는 `coldLinqStatus`로 넣음 — 키 불일치 가능성 있음.)

### 3.3 X34 — 분실신고 신청

- **Controller**: `PauseController.pcsLostChgAjax()`.
- **호출**: `mPlatFormService.pcsLostChg(ncn, ctn, custId, loseType, guideYn, cntcTlphNo, loseCont, loseLocation, strPwdInsert)`  
  - `loseType`: H(핸드폰), U(USIM분실), A(모두분실)
  - `guideYn`: Y(발신정지/착신가능), N(발착신정지), C(정지없음)
  - `cntcTlphNo`: 연락받을 번호
  - `loseCont`: 분실내용
  - `loseLocation`: N(국내분실), Y(해외분실)
  - `strPwdInsert`: 일시정지 패스워드 (4~8자리)
- **응답 VO**: `MpPcsLostChgVO` (parse 비어 있음).

필수 파라미터 모두 입력 시에만 호출.

### 3.4 X35 — 분실신고 해제 (분실복구)

- **Controller**: `PauseController.pcsLostCnlChgAjax()`.
- **호출**: `mPlatFormService.pcsLostCnlChg(ncn, ctn, custId, strPwdNumInsert, pwdType)`  
  - `strPwdNumInsert`: 일시정지 패스워드 4~8자리
  - `pwdType`: PP(일시정지), CP(개인정보 암호)
- **응답 VO**: `MpPcsLostCnlChgVO` (parse 비어 있음).

AS-IS: 일일 X35 호출 건수 제한(예: 10건) `checkMpCallEventCount(userId, "X35")` 적용.

## 5. 서비스변경 신청서(LOST_RESTORE) 상품 단계 기대 동작

`40.서비스변경신청서_UI설계서_분석.md` 3.3.6 요약:

- **섹션**: "분실복구/일시정지해제 신청" (h4).
- **동작**: 일시정지 이력 조회 후 정지유형별 화면 세팅. 일시정지 상태가 아니면 불가 안내.
- **입력**: 해제 비밀번호 (필수).
- **버튼**: [확인] — 유효성 후 "확인 완료". 불가 시 "일시정지 해제가 불가능합니다."

즉, LOST_RESTORE 선택 시:

1. **일시정지 해제**:  
   - X28로 해제 가능 여부 조회 → 가능 시 해제 비밀번호 입력 후 X30 호출.
2. **분실복구**:  
   - X33으로 분실신고 상태/가능 여부 조회 → 분실신고 상태(runMode=U)이면 X35로 해제(분실복구) 진행, 비밀번호 입력 후 X35 호출.

동시에 “일시정지 이력”을 보여주려면 X26 호출로 이력·잔여 횟수·만료예정일 등을 표시할 수 있음.

## 7. TOBE(스마트서식지) 개발 시 참고 사항

1. **LOST_RESTORE 영역**  
   - 상품 단계(`ChangeProd.vue`)에서 LOST_RESTORE 선택 시, 일시정지 해제(X28→X30)와 분실복구(X33→X35)를 구분해 노출.  
   - 일시정지 이력/안내가 필요하면 X26 연동.

2. **API**  
   - `MplatFormSvc`에 X26, X27, X28, X29, X30, X33, X34, X35 메서드 추가.  
   - 요청/응답 구조는 위 VO·파라미터 표 참고. LOCAL/DEV 시 mock 응답(기존 getVo 26~30, 33~35 형식) 유지 가능.

3. **검증**  
   - 일시정지 해제: X28 rsltInd=Y 여부, 본인인증·step 검증.  
   - 분실신고: X33 runMode, rsltCd, subStatusLastAct, 미납 여부.  
   - 분실복구: X33으로 분실신고 상태 확인 후 X35 호출, 필요 시 일일 호출 횟수 제한 검토.

4. **키 통일**  
   - X33 응답에서 미납 여부는 XML 태그와 VO 맵 키를 동일하게 맞출 것 (예: `coldeLinqStatus` vs `coldLinqStatus`).

5. **동시 변경 불가**  
   - LOST_RESTORE는 이미 `SVC_TGT_CD`에서 동시변경불가(EXPNSN_STR_VAL1='N')로 설정되어 있음. 번호변경·요금상품변경·핸드폰기변과 2개 이상 선택 불가.

이 문서를 기준으로 분실복구/일시정지해제 조회·처리 서비스를 TOBE에 이식하면 됨.
