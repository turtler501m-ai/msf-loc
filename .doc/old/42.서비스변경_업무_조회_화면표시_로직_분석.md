# 서비스 변경 업무 조회·화면 표시 로직 분석

**목적**: 서비스 변경 업무를 조회해서 화면에 뿌리는 전 과정(API → DB → 프론트 렌더)을 단계별로 정리한다.  
**관련 문서**: `40.서비스변경신청서_UI설계서_분석.md`, `tabdoc/MSF_CD_GROUP_BAS_CD_DTL_insert.sql`

---

## 1. 전체 흐름 요약

```
[화면 진입] ChangeTypeCust.vue onMounted
    → getChangeTargets() (GET /api/v1/service-change/base/targets)
[백엔드]   SvcChgBaseController.getChangeTargets()
    → SvcChgBaseSvcImpl.getChangeTargets()
    → ① MSF_CD_GROUP_BAS + MSF_CD_DTL (SVC_TGT_CD) 조회
    → ② 실패 시 msf_svc_chg_ctg / msf_svc_chg_opt 조회
    → ③ 실패 시 상수 fallback
[응답]     SvcChgBaseTargetResVO { categoryList, options, concurrentGroups, imagingTargetCodes }
[화면]     categoryList → 카테고리 뱃지 + 옵션 그리드(체크박스)
            concurrentGroups → 동시 선택 불가 코드(coreChangeCodes) → "(동시 선택 불가)" 표시·검증
```

---

## 2. 프론트엔드 — 조회 시점·호출

| 구분 | 파일 | 내용 |
|------|------|------|
| **진입** | `mform-web/src/components/change/ChangeTypeCust.vue` | 서비스변경 1단계(고객+가입유형) 컴포넌트 마운트 시 조회 |
| **호출 시점** | `onMounted()` | `getChangeTargets()` 1회 호출 (비동기) |
| **API 함수** | `mform-web/src/api/serviceChange.js` | `getChangeTargets()` → `msfGet('/v1/service-change/base/targets')` |
| **실제 요청** | - | `GET {BASE_URL}/v1/service-change/base/targets` (BASE_URL 예: `/api`) |

### 2.1 응답 수신 후 상태 반영

```javascript
// ChangeTypeCust.vue onMounted
const res = await getChangeTargets()
if (res?.categoryList?.length) categoryList.value = res.categoryList
if (Array.isArray(res?.concurrentGroups)) coreChangeCodes.value = res.concurrentGroups.flat()
```

| 응답 필드 | 프론트 상태 | 용도 |
|-----------|--------------|------|
| `categoryList` | `categoryList` (ref) | 카테고리별로 그룹 지어 옵션 그리드 렌더링 (카테고리 라벨 + 옵션 목록) |
| `concurrentGroups` | `coreChangeCodes` (ref) | 동시 변경 불가 코드 목록(평면화). 옵션별 "(동시 선택 불가)" 표시, 전체선택 제외, 조합 검증에 사용 |

- **API 예외 시**: `categoryList`는 빈 배열 유지, `coreChangeCodes`는 초기값 `CORE_CHANGE`(상수) 유지 → **selectableOptions**가 상수 `ServiceChangeOptions`로 폴백.

---

## 3. 프론트엔드 — 화면에 뿌리는 로직

### 3.1 데이터 소스 (옵션 목록)

- **옵션 목록**  
  - API 성공: `selectableOptions = categoryList.flatMap(c => c.options)`  
  - API 실패/빈 목록: `selectableOptions = ServiceChangeOptions` (상수)

- **동시 변경 불가 코드**  
  - API 성공: `coreChangeCodes = res.concurrentGroups.flat()`  
  - API 실패: `coreChangeCodes = CORE_CHANGE` (상수)

### 3.2 렌더 분기

| 조건 | UI |
|------|-----|
| `categoryList.length > 0` | **카테고리별 렌더**: `v-for="cat in categoryList"` → 카테고리 뱃지(`cat.category?.label`) + `cat.options` 3열 그리드 체크박스 |
| `categoryList.length === 0` | **폴백**: 카테고리 없이 `selectableOptions`만 3열 그리드로 렌더 |

### 3.3 옵션 1건 표시

- **체크박스**  
  - `v-model="selectedIds"`, `value="opt.id"`  
  - `opt.label`: 화면 문구 (예: 무선데이터차단 서비스, 요금상품 변경)
- **동시 선택 불가**  
  - `coreChangeCodes.includes(opt.id)` 이면 옵션 옆에 "(동시 선택 불가)" 문구(amber) 표시  
  - 동일 그룹에서 이미 하나 선택된 상태면 다른 CORE 옵션은 `isOptDisabled(opt.id)` 로 비활성화

### 3.4 전체선택

- **대상**: `coreChangeCodes`에 없는 옵션만 (`nonCoreIds`).
- **동작**: "전체 선택" 클릭 시 `nonCoreIds`만 일괄 선택/해제; 동시 변경 불가 4개는 항상 제외.

## 4. 백엔드 — API·서비스

| 구분 | 파일 | 내용 |
|------|------|------|
| **엔드포인트** | `SvcChgBaseController.java` | `GET /api/v1/service-change/base/targets` → `getChangeTargets()` |
| **서비스** | `SvcChgBaseSvcImpl.java` | `getChangeTargets()`: 3단계 데이터 소스 시도 후 `SvcChgBaseTargetResVO` 반환 |

### 4.1 응답 DTO 구조

**SvcChgBaseTargetResVO**

| 필드 | 타입 | 설명 |
|------|------|------|
| categoryList | List\<SvcChgCategoryWithOptionsDto\> | 카테고리별 { category, options } 목록. 화면의 카테고리 뱃지 + 옵션 그리드에 사용 |
| options | List\<SvcChgBaseOptionDto\> | 서비스 변경 옵션 평면 목록 (하위 호환) |
| concurrentGroups | List\<List\<String\>\> | 동시 처리 불가 그룹별 코드 리스트. 보통 1개 그룹 [ NUM_CHANGE, LOST_RESTORE, RATE_CHANGE, PHONE_CHANGE ] |
| imagingTargetCodes | List\<String\> | 신청서 이미징 대상 업무 코드 |

**SvcChgCategoryWithOptionsDto**  
- `category`: SvcChgCategoryDto (id, label, sortOrder)  
- `options`: List\<SvcChgBaseOptionDto\>

**SvcChgBaseOptionDto**  
- id, categoryId, label, concurrentGroupId, imagingRequired, sortOrder

## 5. 백엔드 — 데이터 소스 3단계

### 5.1 1순위: 공통코드 (MSF_CD_GROUP_BAS, MSF_CD_DTL)

- **의미**: 서비스 변경 대상 업무를 공통코드로 관리. **동시 변경 가능 여부**는 `MSF_CD_DTL.EXPNSN_STR_VAL1`(Y/N)로 설정.
- **Mapper**
  - `selectSvcChgTargetFromCodeGroup`: `MSF_CD_GROUP_BAS`에서 `CD_GROUP_ID = 'SVC_TGT_CD'` 1건 → SvcChgCategoryDto (id, label, sortOrder).
  - `selectSvcChgTargetFromCodeDetails`: `MSF_CD_DTL`에서 `CD_GROUP_ID = 'SVC_TGT_CD'` 이고 `USE_YN = 'Y'` 인 행 조회.

**selectSvcChgTargetFromCodeDetails 쿼리 요약**

```sql
SELECT CD_GROUP_ID    AS categoryId,
       DTL_CD         AS id,
       DTL_CD_NM      AS label,
       CASE WHEN TRIM(COALESCE(EXPNSN_STR_VAL1, '')) = 'N' THEN 'CORE' ELSE NULL END AS concurrentGroupId,
       false          AS imagingRequired,
       0              AS sortOrder
  FROM MSF_CD_DTL
 WHERE CD_GROUP_ID = 'SVC_TGT_CD' AND USE_YN = 'Y'
 ORDER BY DTL_CD
```

- **동시 변경 불가**: `EXPNSN_STR_VAL1 = 'N'` 인 행만 `concurrentGroupId = 'CORE'`로 매핑.  
  서비스 레이어에서 이 옵션들의 `id`만 모아 `concurrentGroups`에 1개 리스트로 넣음.

- **화면에 뿌리는 값**  
  - **카테고리**: 그룹 1건 (id=`SVC_TGT_CD`, label=코드그룹명).  
  - **옵션**: 위 쿼리 결과 전체가 한 카테고리 하위 옵션으로 내려감.  
  - **동시 선택 불가**: `concurrentGroupId == 'CORE'` 인 옵션 id 목록이 `concurrentGroups`로 내려감.

### 5.2 2순위: msf_svc_chg_ctg / msf_svc_chg_opt

- **조건**: 1순위에서 그룹/상세 조회 예외 또는 결과 없음.
- **Mapper**
  - `selectSvcChgCategories`: ctg_cd, ctg_nm, sort_ord.
  - `selectSvcChgOptions`: opt_cd, ctg_cd, opt_nm, concurrent_grp_cd, img_yn, sort_ord.
- **결과**: 카테고리별로 옵션 묶어 `buildCategoryList()`로 `categoryList` 구성. 동시 변경 불가는 `FALLBACK_CONCURRENT_GROUP`(4개 코드)로 고정.

### 5.3 3순위: 상수 fallback

- **조건**: 2순위도 실패 또는 데이터 없음.
- **구성**: `buildFallbackCategories()`(요금·부가, 데이터·이용, 번호·기기·유심, 보험·결합) + `buildFallbackOptions()`(11개 옵션, 4개 CORE).  
  동일하게 `buildCategoryList()`로 `categoryList` 채우고, `concurrentGroups`는 4개 코드 1개 그룹.

---

## 6. DB — 공통코드 데이터

- **그룹**: `MSF_CD_GROUP_BAS`  
  - `CD_GROUP_ID = 'SVC_TGT_CD'`, `CD_GROUP_NM` = 서비스변경대상코드 등.
- **상세**: `MSF_CD_DTL`  
  - `CD_GROUP_ID = 'SVC_TGT_CD'`, `DTL_CD`, `DTL_CD_NM`, **EXPNSN_STR_VAL1** (동시변경가능여부 Y/N).
- **초기 데이터**: `ktm/doc/tabdoc/MSF_CD_GROUP_BAS_CD_DTL_insert.sql`  
  - SVC_TGT_CD 상세 11건, 동시 변경 불가 4건(RATE_CHANGE, NUM_CHANGE, LOST_RESTORE, PHONE_CHANGE)에만 `EXPNSN_STR_VAL1 = 'N'`.

## 7. 요약 표

| 단계 | 위치 | 역할 |
|------|------|------|
| 1 | ChangeTypeCust.vue `onMounted` | `getChangeTargets()` 호출 |
| 2 | serviceChange.js | `GET /v1/service-change/base/targets` |
| 3 | SvcChgBaseController | `getChangeTargets()` → 서비스 호출 |
| 4 | SvcChgBaseSvcImpl | ① SVC_TGT_CD 코드 조회 ② ctg/opt ③ 상수 fallback |
| 5 | SvcChgBaseMapper.xml | MSF_CD_GROUP_BAS 1건, MSF_CD_DTL 목록(EXPNSN_STR_VAL1 → CORE 여부) |
| 6 | ChangeTypeCust.vue | `categoryList` → 카테고리+옵션 그리드, `concurrentGroups` → coreChangeCodes → 표시·비활성·전체선택 제외 |

**정리**: 서비스 변경 업무는 **공통코드(SVC_TGT_CD) 조회 결과**를 1순위로 사용해 화면에 뿌리고, **EXPNSN_STR_VAL1**로 동시 변경 불가 여부를 정해 같은 API 한 번에 카테고리·옵션·동시선택 제한까지 전달한다.
