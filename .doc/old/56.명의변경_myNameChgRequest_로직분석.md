# 명의변경 myNameChgRequest 신청 로직 분석

**대상**: AS-IS MCP 포털 `MyNameChgController.myNameChgRequest` → `MyNameChgServiceImpl.myNameChgRequest`  
**참조**: `mcp-portal-was/.../mypage/controller/MyNameChgController.java`, `.../service/MyNameChgServiceImpl.java`, `.../mappers/MyNameChgMapper.xml`

---

## 1. 전체 흐름 요약

| 단계 | 위치 | 내용 |
|------|------|------|
| 1 | Controller | 세션·입력값 세팅, **certAuthChgRequest** 검증, MC0용 고정값(mcnStatRsnCd, usimSuccYn) 세팅 |
| 2 | Service | NICE 본인인증 이력(CI) 조회, 예약번호 발급, 안면인증 처리(해당 시), **DB 3테이블 저장** |
| 3 | Controller | 성공 시 **SCAN 서버** 서식지 전송(custRequestScanService.prodSendScan), 로그 저장 |

## 2. Controller: myNameChgRequest.do

### 2.1 진입 조건

- **세션**: `UserSessionDto` 존재, `userId` 필수. 없으면 `NO_FRONT_SESSION_EXCEPTION`.
- **입력**: `MyNameChgReqDto` (양도인·양수인·약관·납부 등 화면에서 수집한 값).
- Controller에서 `userid`, `customerId`를 세션에서 세팅.

### 2.2 명의변경 최종 STEP 검사 — certAuthChgRequest (private)

신청 전에 **한 번 더** 아래를 검사한다.

1. **개통 정보 일치**
   - `contractNum` 필수.
   - `selectCntrListNmChg(userId, contractNum)`으로 회선 목록 조회.
   - 첫 건 기준: **전화번호(unSvcNo), 요금제(soc), 고객유형(cstmrType), 개인(I)** 가 화면 입력과 일치해야 함.  
   - 불일치 시 `AUTH02`, `F_BIND_EXCEPTION`.

2. **계좌인증 → 신용카드 전환 시**
   - `reqInfoChgYn=Y` 이고 `reqPayType != D` 이면, 계좌 인증 스텝 초기화(`certService.getCertInfo`).

3. **양도인 인증 검증 (STEP01)**
   - CertService `vdlCertInfo("D", ...)` 호출.
   - 키: urlType=saveGrantorForm, ncType(0=양도인/2=양도인 법정대리인), name, birthDate, contractNum, authType, reqSeq, resSeq.
   - 성인 양도인: `grCstmrName`, `grCstmrNativeRrn`. 미성년자: `grMinorAgentName`, `grMinorAgentRrn`.

4. **양수인 인증 검증 (STEP02/STEP03)**
   - 최소 스텝 수 체크(`certService.getStepCnt()` ≥ 8 또는 10).
   - `vdlCertInfo`: urlType=saveAssigneeForm, stepEndYn=Y, ncType(1=양수인/3=양수인 법정대리인), name, birthDate, authType, reqSeq, resSeq, (선택) reqAccountNumber, reqBank.

5. **필수값 세팅**
   - `grOnlineAuthInfo`, `onlineAuthInfo` (ReqNo/ResNo 문자열), `socNm` 세팅.

### 2.3 MC0 연동용 고정값 (Controller에서만)

```java
myNameChgReqDto.setMcnStatRsnCd("RCMCMCN"); // 실사용자 명의변경 고정
myNameChgReqDto.setUsimSuccYn("Y");         // USIM 승계 고정
myNameChgReqDto.setIccId(null);
```

## 3. Service: myNameChgRequest (MyNameChgServiceImpl)

### 3.1 NICE 본인인증 CI 조회

- **NiceLogDto**: `reqSeq`, `resSeq`(양수인 NICE 요청/응답 번호), `limitMinute=60`.
- **nicelog.getMcpNiceHistWithTime(niceLogDto)** → 60분 이내 NICE 이력에서 **connInfo(CI)** 조회.
- 없으면 `NICE_CERT_REQ_NULL_EXCEPTION` 발생.
- 조회된 CI를 **myNameChgReqDto.setSelfCstmrCi(connInfo)** 에 세팅 → 양수인 본인확인 CI로 NMCP_CUST_REQUEST_NAME_CHG.SELF_CSTMR_CI 등에 저장.

### 3.2 예약번호 발급

- **appformDao.generateResNo()** → 명의변경 예약번호 생성.
- **myNameChgReqDto.setMcnResNo(...)** → MCN_RES_NO 등으로 저장.

### 3.3 안면인증(해당 시)

- **fathTrgYn = "Y"** 인 경우:
  - **fathService.validateFathSession()** 으로 안면인증 세션 검증.
  - `fathTransacId`, `fathCmpltNtfyDt`, `fathTelNo`(연락가능 연락처) 세팅.
  - **fathService.updateFathMcpRequestOsst(mcnResNo)** 로 OSST 연동 이력의 MVNO_ORD_NO를 '임시예약번호' → '실제예약번호'로 갱신.

### 3.4 DB 저장 (MyNameChgMapper)

| 순서 | DAO 메서드 | 테이블 | 설명 |
|------|------------|--------|------|
| 1 | insertNmcpCustReqMst | **NMCP_CUST_REQUEST_MST** | selectKey로 **CUST_REQ_SEQ** 생성(YYYYMMDD+5자리 시퀀스). REQ_TYPE='NC', 양도인 정보(이름, 전화, 주민, 계약번호, 고객유형, 본인인증 유형/정보, ETC_MOBILE 등) INSERT. |
| 2 | insertNmcpCustReqNameChg | **NMCP_CUST_REQUEST_NAME_CHG** | 동일 CUST_REQ_SEQ로 양수인·신청 상세 INSERT. 양수인 이름/유형/주민, 본인인증, 약관동의, 주소/명세서/납부(은행·카드), 신분증 정보, 연락처, MCN_STAT_RSN_CD, USIM_SUCC_YN, ICC_ID, **MCN_RES_NO**, MCN_STATE_CODE='00', 안면인증 관련, **SELF_CSTMR_CI** 등. PROC_CD='RQ'. |
| 3 | (조건) insertNmcpCustReqNameChgAgent | **NMCP_CUST_REQ_NAME_CHG_AGENT** | 양도인 또는 양수인 중 **미성년자(NM)** 일 때만. 양도인 미성년자: NOR_AGENT_* (법정대리인). 양수인 미성년자: NEE_AGENT_* (법정대리인). |

### 3.5 미성년자 시 추가 UPDATE

- **양도인 미성년자 (grCstmrType=NM)**  
  - MST에 양도인 본인인증 정보를 넣지 않도록: `grOnlineAuthInfo`, `grOnlineAuthType` 를 빈 문자열로 세팅 후 **updateNmcpCustReqMst** 호출.

- **양수인 미성년자 (cstmrType=NM)**  
  - NAME_CHG에 양수인 본인인증/신분증/CI를 넣지 않도록: `onlineAuthInfo`, `onlineAuthType`, `selfCertType`, `selfIssuExprDt`, `selfIssuNum`, `selfCstmrCi` 를 빈 문자열로 세팅 후 **updateNmcpCustReqNameChg** 호출.

→ 미성년자 구간은 법정대리인 정보만 AGENT 테이블에 두고, 본인 인증 정보는 마스터/명의변경 상세에서는 제거하는 구조.

### 3.6 반환

- 위까지 정상 완료 시 **result = "SUCCESS"** 반환.  
- 예외 시 트랜잭션 롤백 및 Controller에서 RESULT_CODE 0001 또는 FAIL 처리.

## 4. Controller: 신청 후 처리 (SCAN·로그)

- **myNameChgService.myNameChgRequest** 가 SUCCESS 이면:
  - **custRequestScanService.prodSendScan(custReqSeq, userid, "NC")** 호출 → 서식지 데이터 생성·SCAN 서버 전송.
  - **ipstatisticService.insertAdminAccessTrace** 로 CUST_REQUEST_SCAN 처리 로그 저장.
- prodSendScan 예외 시 RESULT_CODE를 FAIL로 바꾸고, 동일한 트레이스 로그를 FAIL로 기록.  
- **custReqSeq** 는 insertNmcpCustReqMst의 selectKey로 채워진 값(MyNameChgMapper에서 keyProperty="custReqSeq").

## 5. 사용 테이블 정리

| 테이블 | 용도 |
|--------|------|
| **NMCP_CUST_REQUEST_MST** | 명의변경 요청 마스터. CUST_REQ_SEQ(PK), REQ_TYPE='NC', 양도인 정보, ETC_MOBILE 등. |
| **NMCP_CUST_REQUEST_NAME_CHG** | 명의변경 상세. 양수인 정보, 약관, 납부, MCN_STAT_RSN_CD, USIM_SUCC_YN, MCN_RES_NO, SELF_CSTMR_CI, 안면인증 등. |
| **NMCP_CUST_REQ_NAME_CHG_AGENT** | 미성년자 법정대리인. NOR_AGENT_*(양도인), NEE_AGENT_*(양수인). |

## 6. TO-BE(mform) 대비 시 유의점

- AS-IS는 **NMCP_CUST_REQUEST_MST / NMCP_CUST_REQUEST_NAME_CHG / NMCP_CUST_REQ_NAME_CHG_AGENT** 를 사용하고,  
  다른 플로우의 **NMCP_NFL_CHG_TRNS / NMCP_NFL_CHG_TRNSFE** (CsTransfer)와는 **테이블이 다름**.
- TO-BE 현재는 NMCP_NFL_CHG_TRNS/TRNSFE 기반 로컬 저장; AS-IS 마이페이지 명의변경은 CUST_REQUEST_* 계열 테이블 사용.
- NICE CI(60분 이내 이력), 예약번호, 안면인증, 미성년자 시 MST/NAME_CHG 업데이트, SCAN 전송까지 흐름을 TO-BE에 반영할 경우 위 순서와 테이블 구분을 참고하면 됨.
