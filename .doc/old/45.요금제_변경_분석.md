## 1. doFarPricePlanView (목록 화면)

### 1.1 개요

대상 소스: MCP `FarPricePlanController.doFarPricePlanView`

- 경로: `mcp/mcp-portal-was/src/main/java/com/ktmmobile/mcp/mypage/controller/FarPricePlanController.java` (118~156행)
- 역할: 마이페이지 > 요금제/부가서비스 > 요금제 변경 목록 화면 처리
- URL: `/mypage/farPricePlanView.do` (PC) / `/m/mypage/farPricePlanView.do` (모바일)



### 1.2 처리 흐름

1. 뷰 경로 결정 (태블릿 / 모바일)
2. 세션 → 회원 정보 조회
3. 회원 소유 회선 목록 조회
4. checkUserType (정회원·선택 회선 검증)
5. selectFarPricePlanYn (요금제 변경 가능 여부)
6. selectFarPricePlan (현재 요금제 정보)
  - ncn(계약번호)로 현재 요금제 코드·명·기본료·적용일 등 조회
7. Model 세팅 후 뷰 반환

    @RequestMapping(value = { "/mypage/farPricePlanView.do", "/m/mypage/farPricePlanView.do" })
    public String doFarPricePlanView(HttpServletRequest request, Model model,
            @ModelAttribute("searchVO") MyPageSearchDto searchVO) {

        String returnUrl = "/portal/mypage/farPricePlanView";

        if ("Y".equals(NmcpServiceUtils.isMobile())) {
            returnUrl = "/mobile/mypage/farPricePlanView";
        }
        //세션 → 회원 정보 조회
        UserSessionDto userSessionDto = SessionUtils.getUserCookieBean(); 
        //회원 소유 회선 목록 조회
        List<McpUserCntrMngDto> cntrList = mypageService.selectCntrList(userSessionDto.getUserId()); 

        // (정회원·선택 회선 검증)
        //searchVO.ncn 과 일치하는 회선이 있으면 해당 회선 정보로 searchVO 갱신
        if (!this.checkUserType(searchVO, cntrList, userSessionDto)) { 
            ResponseSuccessDto responseSuccessDto = getMessageBox();
            model.addAttribute("responseSuccessDto", responseSuccessDto);
            return "/common/successRedirect";
        }

        // 요금제 변경 불가능 socCode
        // 이용중인 부가서비스 조회 체크
        // 요금제 변경 가능 여부
        // 1.공통코드에서 요금제 변경 불가 soc 목록 로드 (`GROUP_CODE_EXCEPTION_LIST_SOC_CODE`)
        // 2.M플랫폼 X20 `getAddSvcInfoDto(ncn, ctn, custId)` 로 가입 부가서비스 조회
        // 3.가입 부가서비스 soc 가 예외 목록에 하나라도 있으면 "FALSE", 없으면 "TRUE" 반환
        String isPriceChange = farPricePlanService.selectFarPricePlanYn(searchVO.getNcn(), searchVO.getCtn(),
                searchVO.getCustId());

        // 요금제 정보
        McpFarPriceDto mcpFarPriceDto = mypageService.selectFarPricePlan(searchVO.getContractNum());
        if (mcpFarPriceDto == null) {
            throw new McpCommonException(NOT_FOUND_DATA_EXCEPTION);
        }

        model.addAttribute("cntrList", cntrList);
        model.addAttribute("searchVO", searchVO);
        model.addAttribute("isPriceChange", isPriceChange);
        model.addAttribute("mcpFarPriceDto", mcpFarPriceDto);

        return returnUrl;

    }


### 1.3 단계별 상세

#### 1.3.1 뷰 경로

java
String returnUrl = "/portal/mypage/farPricePlanView";  // PC
if ("Y".equals(NmcpServiceUtils.isMobile())) {
    returnUrl = "/mobile/mypage/farPricePlanView";     // 모바일
}


#### 1.3.2 세션·회선 조회

- `SessionUtils.getUserCookieBean()` → `UserSessionDto`
- `mypageService.selectCntrList(userId)` → `McpUserCntrMngDto` 회선 목록

#### 1.3.3 checkUserType (회원·회선 검증)

- 정회원 여부: `userSession.getUserDivision() == "01"`
- 회선 목록: null/empty 시 false
- searchVO 미입력 시 첫 번째 회선 기준으로 자동 세팅: ncn, ctn, custId, modelName, contractNum, subStatus
- searchVO.ncn 과 일치하는 회선이 있으면 해당 회선 정보로 searchVO 갱신
- 실패 시 `/common/successRedirect` 로 리다이렉트 (메시지: "정회원 인증 후 이용하실 수 있습니다.")

#### 1.3.4 selectFarPricePlanYn (요금제 변경 가능 여부)

호출: `farPricePlanService.selectFarPricePlanYn(ncn, ctn, custId)`

내부 동작:
1. 공통코드에서 요금제 변경 불가 soc 목록 로드 (`GROUP_CODE_EXCEPTION_LIST_SOC_CODE`)
2. M플랫폼 X20 `getAddSvcInfoDto(ncn, ctn, custId)` 로 가입 부가서비스 조회
3. 가입 부가서비스 soc 가 예외 목록에 하나라도 있으면 "FALSE", 없으면 "TRUE" 반환

- `"FALSE"` → 요금제 변경 불가
- `"TRUE"` → 요금제 변경 가능

#### 1.3.5 selectFarPricePlan (현재 요금제 정보)

호출: `mypageService.selectFarPricePlan(searchVO.getContractNum())`

- MCP API: `POST /mypage/farPricePlan` (body: ncn)
- MypageMapper `selectFarPricePlan`:
  - `MSP_JUO_FEATURE_INFO@DL_MSP`, `MSP_RATE_MST@DL_MSP` 조회
  - ncn(계약번호)로 현재 요금제 코드·명·기본료·적용일 등 조회

결과 DTO: `McpFarPriceDto`
- prvRateCd, prvRateGrpNm, prvBaseAmt, prvRmk
- appStartDd, baseVatAmt, promotionDcAmt 등

#### 1.3.6 Model 세팅 및 반환

model.addAttribute("cntrList", cntrList);           // 회선 목록
model.addAttribute("searchVO", searchVO);            // ncn, ctn, custId, contractNum 등
model.addAttribute("isPriceChange", isPriceChange);  // "TRUE" / "FALSE"
model.addAttribute("mcpFarPriceDto", mcpFarPriceDto);// 현재 요금제 정보
return returnUrl;

### 1.7 mform-api 연동 시 참고

mform-api는 세션 기반 회원 로그인이 없으므로, ncn/ctn/custId 는 휴대폰 인증 결과로만 확보됨.

- 요금제 변경 가능 여부(`selectFarPricePlanYn`)와 현재 요금제 조회(`selectFarPricePlan`)는 ncn(계약번호) 기준으로 수행됨.
- mform용 REST API 구현 시:
  1. X20 부가서비스 조회 → 예외 soc 비교로 요금제 변경 가능 여부 판단
  2. M전산 또는 동등 API로 현재 요금제 정보 조회

## 2. proPriceChg (요금제 즉시변경 처리)

### 2.1 개요

| 항목 | 내용 |
|------|------|
| **진입점** | `FarPricePlanController.proPriceChg()` |
| **경로** | `FarPricePlanController.java` (537~577행) |
| **URL** | `/mypage/farPricePlanChgAjax.do`, `/m/mypage/farPricePlanChgAjax.do` |
| **역할** | 요금제 즉시변경 신청 처리 (M플랫폼 X19, X38, X21 연동) |
| **핵심 로직** | `FarPricePlanService.getAddSvcInfoDto(searchVO, toSocCode, nowPriceSocCode, paraAlterTrace)` |

### 2.2 처리 흐름

```
[컨트롤러 proPriceChg]
    │
    ├─ 1. 사전 검증
    │     ├─ 세션 검증 (SessionUtils.getUserCookieBean)
    │     ├─ 회선 검증 (checkUserType)
    │     ├─ 번호유효성/OTP 검증 (sessionOtp == searchVO.ncn)
    │     └─ 현재 요금제 조회 (mypageService.selectFarPricePlan → nowPriceSocCode)
    │
    └─ 2. farPricePlanService.getAddSvcInfoDto 호출
          │
          ├─ [1단계] 부가서비스 해지 (X38)
          │     ├─ X20 getAddSvcInfoDto → 이용중 부가서비스 조회
          │     ├─ getCloseSubList(contractNum) → 해지 대상 부가서비스 목록 (RCP9030)
          │     ├─ for: getAddSvcInfo ∩ getCloseSubList
          │     │     └─ moscRegSvcCanChgNeTrace → X38 부가서비스 해지
          │     │           └─ 실패 시: SMS 발송, 이력 저장, MSP_SOCFAIL_PROC_MST 저장 후 return
          │     └─ serviceAlterTrace (X38) 저장
          │
          ├─ [2단계] 요금제 변경 (X19)
          │     └─ farPricePlanChgNeTrace(searchVO, toSocCode) → X19 요금 상품 변경
          │           ├─ 성공 → serviceAlterTrace (X19) 저장
          │           └─ 실패 → SMS 발송, 이력 저장, MSP_SOCFAIL_PROC_MST 저장 후 return
          │
          └─ [3단계] 부가서비스 가입 (X21)
                ├─ getromotionDcList(toSocCode) → 신규 가입 부가서비스 목록
                │     (VW_CHRG_PRMT_DTL + MCP_ADDITION, 할인/프로모션 대상)
                ├─ for: serviceInfoList
                │     └─ regSvcChgNeTrace(searchVO, soc, "") → X21 부가서비스 신청
                │           └─ 성공/실패 각각 serviceAlterTrace (X21) 저장
                └─ 최종: FIN 이력 + MSP_SOCFAIL_PROC_MST (SUCCESS/FAIL)
```

### 2.3 처리 흐름 요약

1. 세션·회선·OTP·현재요금제 조회 
  - SessionUtils, checkUserType, selectFarPricePlan
2. X20 이용중 부가 조회 
  - getCloseSubList 해지대상 → 교집합 대상 X38 해지
  - X20 getAddSvcInfoDto, X38 moscRegSvcCanChgNeTrace
3. 요금제 변경 
  - X19 farPricePlanChgNeTrace
4. 부가서비스 가입 
  - getromotionDcList → 각 SOC X21 신청
  - X21 regSvcChgNeTrace

※ 1·2단계 실패 시 SMS, 이력 저장, MSP_SOCFAIL_PROC_MST 후 return

### 2.4 단계별 상세

#### 2.4.1 컨트롤러 사전 검증

| 순서 | 처리 | 내용 |
|------|------|------|
| 1 | 세션 검증 | `SessionUtils.getUserCookieBean()` null/빈 userId → McpCommonJsonException |
| 2 | 회선 검증 | `mypageService.selectCntrList(userId)` 후 `checkUserType(searchVO, cntrList, userSession)` |
| 3 | 번호 유효성 | `SessionUtils.getOtpInfo()` == `searchVO.getNcn()` 일치 필수 |
| 4 | 현재 요금제 | `mypageService.selectFarPricePlan(searchVO.getContractNum())` → `nowPriceSocCode = prvRateCd` |

**파라미터**

- `searchVO`: ncn, ctn, custId, contractNum
- `toSocCode`: 변경할 요금제(SOC)
- `paraAlterTrace`: tSocAmnt, aSocAmnt 등 이력용

#### 2.4.2 1단계: 부가서비스 해지 (X38)

**목적**: 요금제 변경 전, 제거 대상 부가서비스 선처리

1. **X20 조회**: `mPlatFormService.getAddSvcInfoDto(ncn, ctn, custId)` → 이용중 부가서비스
2. **해지 대상 조회**: `getCloseSubList(contractNum)` (MypageMapper)
   - M전산: `MSP_JUO_FEATURE_INFO`, `CMN_GRP_CD_MST` (RCP9030), `MSP_JUO_SUB_INFO`
   - 해지해야 할 SOC 목록 (공통코드 RCP9030 기준)
3. **교집합 처리**: X20 결과 ∩ getCloseSubList, 가입 여부 확인 후 해지 대상이면 X38 호출
4. **X38 호출**: `mPlatFormService.moscRegSvcCanChgNeTrace(searchVO, socCode)`
   - onlineCanYn = 'Y' 등 사전 검증
   - ESB 오류(ITL_SYS_E0001) 시 최대 1회 재시도 (3초 대기)
5. **실패 시**: 카카오 알림톡 발송 → X38 이력 저장 → FIN FAIL 이력 → `MSP_SOCFAIL_PROC_MST` 저장 후 return

#### 2.4.3 2단계: 요금제 변경 (X19)

**목적**: 요금 상품 즉시 변경

1. **X19 호출**: `mPlatFormService.farPricePlanChgNeTrace(searchVO, toSocCode)`
   - M플랫폼 `callServiceNe(param, vo)` with `getParamMap(..., "X19")`
   - 파라미터: ncn, ctn, custId, soc(toSocCode)
2. **성공**: `RESULT_CODE = AJAX_SUCCESS` → X19 이력 저장
3. **실패**: 카카오 알림톡 발송 → X19 이력 저장 → FIN FAIL 이력 → `MSP_SOCFAIL_PROC_MST` 저장 후 return

#### 2.4.4 3단계: 부가서비스 가입 (X21)

**목적**: 변경 후 요금제에 해당하는 프로모션/할인 부가서비스 자동 가입

1. **가입 대상 조회**: `getromotionDcList(toSocCode)` (MypageMapper)
   - `VW_CHRG_PRMT_DTL` + `MCP_ADDITION`
   - 조건: ORGN_ID=직영온라인, REQ_BUY_TYPE=UU/AL, RATE_CD=toSocCode, USG_YN=Y 등
2. **X21 반복 호출**: `mPlatFormService.regSvcChgNeTrace(searchVO, socCode, "")`
   - 각 SOC별 X21 호출
   - 중간 실패해도 다음 SOC 계속 처리
3. **최종**: 전부 성공 시 FIN SUCCESS → `insertSocfailProcMst(succYn='Y')`, 일부 실패 시 FIN FAIL → `insertSocfailProcMst(succYn='N')`, SCNT/FCNT 파라미터 기록

### 2.5 M플랫폼 API

| 단계 | API | 역할 |
|------|-----|------|
| X20 | getAddSvcInfoDto | 현재 가입 부가서비스 조회 |
| X38 | moscRegSvcCanChgNeTrace | 부가서비스 해지 |
| X19 | farPricePlanChgNeTrace | 요금 상품 즉시 변경 |
| X21 | regSvcChgNeTrace | 부가서비스 신청 |

### 2.6 이력·DB 저장

- **이력 저장**: `mypageService.insertServiceAlterTrace`
  - 이벤트 코드: X38(해지), X19(요금제변경), X21(부가서비스신청), FIN(최종)
- **실패 이력**: `mypageService.insertSocfailProcMst` → `MSP_SOCFAIL_PROC_MST`

### 2.8 mform-api 연동 시 참고

mform-api `FarPriceService.applyChange(schedule=immediate)` 구현 시:

1. **X19 호출**: `MplatFormSvc`에서 `farPricePlanChgNeTrace` 또는 X19 동등 규격 구현
2. **선행 X38**: `getCloseSubList` 기준 해지 대상 부가서비스 선처리 필요
3. **후속 X21**: `getromotionDcList` 프로모션 부가서비스 자동 가입 여부 사양 확인
