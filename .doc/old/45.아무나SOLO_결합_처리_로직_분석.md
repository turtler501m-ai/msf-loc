# 아무나 SOLO 결합 처리 로직 분석

**대상**: 아무나 SOLO 결합 **신청·처리** 흐름(프론트 확인 → 백엔드 Y44 호출 → DB 저장).  
**관련**: `CombineController.regCombineSelf`, M플랫폼 **Y44**(서브회선 마스터 결합 신청), `MCP_REQ_COMBINE` 저장.  
**선행 문서**: `44.getCombiSvcList_아무나SOLO_결합가능여부_체크_로직_분석.md` (가입 가능 여부 체크)

---

## 1. 처리 흐름 개요

```
[화면] 아무나 SOLO 전용 화면(combineSelf) → 필수 동의 체크 → [아무나 SOLO 결합 신청] 클릭
  → 확인 다이얼로그 "아무나 SOLO 결합을 신청하시겠습니까?"
  → POST /content/regCombineSelfAjax.do (menu, ncn)
[백엔드] regCombineSelf()
  → ① MasterCombineLineInfo 코드 조회 (mstSvcContId)
  → ② 세션·스텝·인증 검증 (combineSelfSave)
  → ③ Y44 서브회선 마스터 결합 신청 (moscSubMstCombChg)
  → ④ 성공 시 요금제 결합 매핑 확인·MCP_REQ_COMBINE INSERT
[응답] RESULT_CODE=SUCCESS / 9999 / 0001 / 1999 / 0008 등 → 화면 완료 모달 또는 에러 메시지
```

---

## 2. 프론트엔드 — 신청 버튼·호출

**파일**: `combineSelf.js` (portal/mobile 동일 구조)

| 단계 | 내용 |
|------|------|
| 버튼 | `#btnRegCombin` [아무나 SOLO 결합 신청] |
| 검증 | `validator.config['chkAgree'] = 'isNonEmpty'` → 필수 동의(chkAgree) 체크 |
| 확인 | `KTM.Confirm('아무나 SOLO 결합을 신청하시겠습니까?', ...)` |
| 요청 | `POST /content/regCombineSelfAjax.do` |
| 파라미터 | `menu` (메뉴 타입), `ncn` (서비스 계약번호, 결합 대상 회선) |

### 2.1 응답 처리

| RESULT_CODE | 동작 |
|-------------|------|
| **SUCCESS** | 완료 다이얼로그 오픈 (`#combinComplete` 또는 `#combinCompleteTriple`). 기존 결합 리스트에 Internet 포함 시 Triple |
| **9999** | `RESULT_MSG` 또는 "결합 신청에 실패하였습니다." alert |
| **1999** | "결합이 지연되고 있습니다." + 다시 시도 안내 alert |
| 기타 | `RESULT_MSG` + "다시 시도해 주시기 바랍니다." alert |

## 3. 백엔드 — regCombineSelf (regCombineSelfAjax.do)

**진입**: `CombineController.regCombineSelf(AuthSmsDto authSmsDto, MyPageSearchDto searchVO)`

### 3.1 선행 검증

| 순서 | 체크 | 불가 시 |
|------|------|---------|
| 1 | **MasterCombineLineInfo** 공통코드 존재 | 없으면 **0002** "현재 서비스 이용이 불가합니다. 나중에 다시 이용 바랍니다." |
| 2 | `mstSvcContId = materCombineLineList.get(0).getDtlCd()` | Y44 요청 시 **마스터(법인) 서비스 계약 ID** 로 전달 |
| 3 | 세션 | `SessionUtils.getAuthSmsBean(authSmsDto)` 없거나 `svcCntrNo` 없음 → **0001** 예외 |
| 4 | ncn 일치 | `searchVO.getNcn() != rtnDto.getSvcCntrNo()` → **0002** 예외 |
| 5 | **스텝 수** | 로그인 없으면 certStep=4, 있으면 3. `certService.getStepCnt() < certStep` → **STEP02** 예외 |
| 6 | **인증 검증** | `certService.vdlCertInfo("D", certKey, certValue)` urlType=**combineSelfSave**, name, mobileNo, contractNum. 실패 시 **-1** + RESULT_DESC |

### 3.2 Y44 호출 (서브회선 마스터 결합 신청)

- **서비스**: `myCombinationSvc.moscSubMstCombChg(ncn, ctn, custId, mstSvcContId)`
- **실제 연동**: `MplatFormService.moscSubMstCombChg(ncn, ctn, custId, mstSvcContId)`
  - **API 코드**: **Y44** (서브회선 마스터 결합 신청)
  - **파라미터**: `getParamMap(ncn, ctn, custId, "Y44")` + **mstSvcContId** (마스터 결합 대상 서비스 계약 ID, 공통코드 DTL_CD)
- **이력**: `McpServiceAlterTraceDto` — prcsMdlInd=MPY44, eventCode=Y44, 성공/실패·응답 요약 저장 후 `mypageService.insertServiceAlterTrace(serviceAlterTraceSub)`
- **응답**: `MoscSubMstCombChgRes` — `isSuccess()`, `getResultCd()`, `getResultMsg()`.  
  - **resultCd 0000** = 성공(변경 처리 완료)  
  - **1000** = 서브회선 부가서비스 가입 오류 등  
  - **9999** = 실패(공통코드 ErrorMsgReplaceList 로 메시지 치환 시 9999로 통일)

### 3.3 Y44 성공 시 (isSuccess && resultCd == "0000")

| 순서 | 처리 | 비고 |
|------|------|------|
| 1 | 요금제 결합 매핑 재확인 | `selectMspCombRateMapp(rtnDto.getRateCd())`. **null** → **0004** "해당 상품은 결합이 불가합니다." |
| 2 | **MCP_REQ_COMBINE INSERT** | 결합 신청 결과 DB 저장 |

**McpReqCombineDto 세팅 요약**

| 항목 | 값 | 비고 |
|------|-----|------|
| rsltCd | S | 승인완료 |
| combTypeCd | 04 | 결합유형 (아무나 SOLO) |
| combTgtTypeCd | 03 | 결합대상 타인(아무나) |
| mCtn, mCustNm, mCustBirth, mSexCd, mSvcCntrNo, mRateCd, mRateNm | 세션·요금제 정보 | 엠모바일(신청 회선) |
| mRateAdsvcCd / mRateAdsvcNm | myCombination.getrRateCd/Nm | 할인 부가(결합 혜택) |
| combSvcNo | masterCombineInfo.getDtlCdNm() | 결합 상대 표시명 |
| combSvcCntrNo | mstSvcContId | 마스터 서비스 계약 ID |
| combCustNm | "주식회사 케이티엠모바일" | 아무나 결합 상대 |
| combSexCd | 03 | 법인 |
| combSocCd / combSocNm | PL249Q804 / "MVNO마스터결합전용 더미요금제 (엠모바일)" | 무무(아무나) 결합용 더미 |
| combRateAdsvcCd/Nm | EMPTY / "데이터 제공 없음" | |

- **insertMcpReqCombine(reqCombine)** 성공 → **RESULT_CODE=AJAX_SUCCESS**, **REQ_SEQ** 반환.
- INSERT 실패 → **9999** + DB_EXCEPTION 메시지.

### 3.4 Y44 실패·예외 시

| 조건 | RESULT_CODE | RESULT_MSG |
|------|-------------|------------|
| resultCd == "9999" | **9999** | moscSubMstCombChgRes.getResultMsg() (치환된 메시지 가능) |
| 그 외 비성공 | **0001** | getResultMsg() |
| SocketTimeoutException | **1999** | SOCKET_TIMEOUT_EXCEPTION |
| 기타 Exception | **0008** | MPLATFORM_RESPONEXML_EMPTY_EXCEPTION |

## 4. M플랫폼 Y44 (moscSubMstCombChg) 상세

- **역할**: 서브회선(엠모바일 회선)을 **마스터 결합**(법인·아무나 SOLO용 마스터 회선)에 연결하는 신청.
- **입력**: ncn(서비스계약번호), ctn(전화번호), custId, **mstSvcContId**(마스터 서비스 계약 ID, 공통코드 MasterCombineLineInfo의 DTL_CD).
- **이력**: MCP 서비스 변경 이력 테이블에 eventCode=Y44, prcsMdlInd=MPY44 로 저장.
- **응답**: resultCd 0000이면 성공. 1000 등이면 부가 가입 오류(예: 월 2회 제한 등). 실패 시 responseCode/resultCode에 따라 **ErrorMsgReplaceList** 공통코드로 메시지 치환 후 9999로 내려줄 수 있음.

---

## 5. DB 저장 — MCP_REQ_COMBINE

- **테이블**: MCP_REQ_COMBINE  
- **시퀀스**: MCP_REQ_COMBINE_SEQ (REQ_SEQ 채번)  
- **저장 시점**: Y44 성공 후, 요금제 결합 매핑 확인까지 끝난 뒤 1건 INSERT.  
- **용도**: 결합 신청 이력·승인완료(S) 이력 보관. 아무나 SOLO는 combTypeCd=04, combTgtTypeCd=03, combSocCd=PL249Q804 등으로 구분.

## 6. 정리

| 구분 | 내용 |
|------|------|
| **진입** | combineSelf 화면 → 필수 동의 → [아무나 SOLO 결합 신청] → regCombineSelfAjax.do |
| **검증** | MasterCombineLineInfo, 세션, ncn, 스텝 수, combineSelfSave 인증 |
| **처리** | Y44(서브회선 마스터 결합 신청) 호출, 성공 시 selectMspCombRateMapp 확인 후 MCP_REQ_COMBINE INSERT |
| **연동** | M플랫폼 **Y44**, 공통코드 **MasterCombineLineInfo**(mstSvcContId), **msp 결합 매핑**(할인 부가 정보) |
| **결과** | 성공 시 AJAX_SUCCESS + REQ_SEQ, 실패 시 9999/0001/1999/0008 등 + 메시지. 화면은 완료 모달 또는 에러 alert |

아무나 SOLO **가입 가능 여부**는 `checkCombineSelf`(문서 44), **실제 신청 처리**는 `regCombineSelf` + **Y44** + **MCP_REQ_COMBINE** INSERT 로 이루어진다.
