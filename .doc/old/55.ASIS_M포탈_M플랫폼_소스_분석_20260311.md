## 2. M포탈 백엔드 (mcp-portal-was)

### 2.1 패키지 구조 (서식지·M플랫폼 중심)

```
mcp-portal-was/src/main/java/com/ktmmobile/mcp/
├── appform/                    # 서식지
│   ├── controller/AppformController.java
│   ├── service/AppformSvc.java, AppformSvcImpl.java
│   ├── service/FormDtlSvc*.java, ScanSvc*.java, EventCodeSvc*.java, EsimSvc*.java
│   ├── dao/AppformDao*.java, FormDtlDao*.java, EsimDao*.java
│   └── dto/AppformReqDto.java, FormDtlDTO.java, JuoSubInfoDto.java, OsstReqDto.java 등
├── common/
│   ├── mplatform/              # ★ M플랫폼 연동 핵심
│   │   ├── MplatFormService.java
│   │   ├── MplatFormServerAdapter.java
│   │   ├── MplatFormOsstServerAdapter.java
│   │   ├── MplatFormOsstWebService.java
│   │   ├── controller/MplatFormController.java
│   │   ├── dto/                # M플랫폼 응답 DTO (Mosc*)
│   │   └── vo/                 # 요청/응답 VO (Mp*, CommonXmlVO 등)
│   ├── mspservice/             # MSP(요금·정책 등) 연동
│   ├── service/SmsSvc*.java, FCommonSvc*.java
│   └── ...
├── content/                    # 결합·데이터쉐어링
├── mypage/                     # 마이페이지·요금·부가·일시정지
├── phone/                      # 폰·기변
├── usim/                       # 유심
└── ...
```

### 2.2 M플랫폼 연동 흐름

```
[화면/Ajax]
    → AppformController / RegSvcController / PauseController 등
    → MplatFormService.perMyktfInfo() 등 (eventCd: X01, X20, …)
    → MplatFormServerAdapter.callService(param, vo)
        → param: ncn, ctn, custId, userid, appEventCd
        → getURL(param) → encode 후 juice.url 로 POST (getURL=...)
    → [juice/msc-prx 등] → M플랫폼(KOS) 실제 연동
    → responseXml 수신 → CommonXmlVO.toResponseParse()
```

- **연결 설정**: `MplatFormServerAdapter`  
  - `@Value("${juice.url}")` → M플랫폼 연동 게이트웨이 URL  
  - `@Value("${api.interface.server}")` → 이벤트 카운트 등 API 서버
- **LOCAL 모드**: `MplatFormService` 에서 `SERVER_NAME=LOCAL` 이면 `getVo()`/`getVoEtc()` 로 목 데이터 반환 (실제 호출 생략).

## 4. MplatFormServerAdapter (연동 방식)

**파일**: `mcp-portal-was/.../common/mplatform/MplatFormServerAdapter.java`

| 항목 | 내용 |
|------|------|
| **연동 URL** | `juice.url` (properties) |
| **전송 방식** | POST, 파라미터명 `getURL`, 값은 `key=value&...` 형태를 UTF-8 URL 인코딩한 문자열 |
| **타임아웃** | 기본 30초, 오버로드로 지정 가능 |
| **로그** | saveMplateSvcLog(param) 으로 ip, url, mdlInd 등 보강 후 전송 |
| **응답** | responseXml 문자열 수신 → `CommonXmlVO.setResponseXml()` → `toResponseParse()` 로 파싱 |

## 5. mcp-api (REST API · 신청서 이벤트 XML)

**패키지**: `mcp-api/src/main/java/com/ktmmobile/mcp/mplatform/`

| 클래스 | 역할 |
|--------|------|
| **MPlatformController** | REST 컨트롤러. `/mPlatform/*` |
| **MPlatformMapper** | DB/설정 기반으로 **신청서 이벤트별 XML 메시지** 조회 |
| **MPlatformReqVO, MPlatformResVO** | 요청/응답 VO |

### 5.1 주요 API (서식지·명의변경 연동)

| API | 메소드 | 용도 |
|-----|--------|------|
| **POST /mPlatform/getRequestInfo** | getRequestInfo(MPlatformReqVO) | 신청서 등록 정보 조회. **appEventCd**(MC0, MP0 등)에 따라 서비스/오퍼레이션명 등 반환 |
| **POST /mPlatform/getXmlMessageMC0** | getXmlMessageMC0(MPlatformReqVO) | 명의변경 **사전체크** MC0 XML 메시지 조회 |
| **POST /mPlatform/getXmlMessageMP0** | getXmlMessageMP0(String resNo) | 명의변경 **실행** MP0 XML 메시지 조회 |
| **POST /mPlatform/getXmlMessageMcnMP0** | getXmlMessageMcnMP0(MPlatformReqVO) | 명의변경 MP0 (Mcn) |
| getXmlMessageUC0 | 유심 셀프 변경 UC0 | |
| getXmlMessagePC0 | PC0 사전체크 | |
| getXmlMessageOP0, getXmlMessageST1, getXmlMessageEP0 | 기타 신청서 이벤트 | |
| getXmlMessageNP1, NP2, NP3 | 신규/변경 등 | |
| getXmlMessageFS5 | Nstep FS5 | |
| insertRequestOsst, updateRequestAddition | OSST 등록/추가 수정 | |
| insertOsstErrLog | OSST 오류 로그 | |

- **실제 M플랫폼(KOS) 호출**은 msc-prx 등에서 위 XML을 사용해 수행하며, mcp-api는 **XML 메시지 조합·조회** 역할이다.

## 10. 스마트서식지(mform-web) 연동 시 참고 포인트

1. **연동 스펙 동일**: 스마트서식지 백엔드에서 M포탈과 **동일한** appEventCd(X01, X20, X21, X38, X85, X91, X97, X19, X84, X88~X90 등) 및 요청 파라미터(ncn, ctn, custId, userid, appEventCd)로 **juice.url** 에 전달하는 구조를 유지할 것.
2. **mcp-api 역할**: MC0/MP0 등 **신청서 이벤트 XML**은 mcp-api의 `getRequestInfo`, `getXmlMessageMC0`, `getXmlMessageMP0` 등으로 조회하는 구조를 그대로 두고, 스마트서식지 백엔드가 같은 mcp-api를 호출하거나, 동일 스펙으로 자체 구현할지 정책 결정.
3. **portal-was 메소드명**: `perMyktfInfo`(X01), `getAddSvcInfoDto`(X20), `regSvcChg`(X21), `moscRegSvcCanChg`(X38), `moscIntmMgmtSO`(X85), `getAddSvcInfoParamDto`(X97) 등은 **기능목록·연동명세**와 매핑하여 TOBE에서 동일 기능 구현 시 참고.
4. **LOCAL/개발**: MplatFormService 의 `SERVER_NAME=LOCAL` 분기처럼, 스마트서식지 백엔드에서도 목업/로컬 연동 모드를 두면 개발·테스트에 유리함.
