## 5. 스텝 진행 아키텍처 (MsfFormView)

### 동작 원리

```
route.params.domain (예: "newchange")
  → stepStore.steps["newchange"] = ['NewChangeCustomer','NewChangeProduct','NewChangeAgreement']
  → getFormComponent("newchange", "NewChangeCustomer")
  → defineAsyncComponent(() => import('@/components/form/newchange/NewChangeCustomer.vue'))
```

### visibleSteps 누적 표시 방식

```javascript
// 완료한 스텝 + 현재 스텝까지 아래로 쌓이는 UI
visibleSteps = allSteps.slice(0, currentStepIndex + 1)
```

각 스텝 패널이 쌓이면서 스크롤로 진행 — `currentStepIndex` watch → 새 스텝 `.page-step-panel`로 smooth scroll.

### 스텝 컴포넌트 expose 패턴

모든 스텝 컴포넌트는 `defineExpose`로 두 함수를 노출해야 한다:

```javascript
// 각 스텝 컴포넌트 내부
defineExpose({
  save() { /* 현재 스텝 데이터를 스토어에 저장 */ },
  data() { /* 현재 스텝 수집 데이터 반환 */ }
})

// MsfFormView에서 호출
stepRefs[i].save()   // 다음 스텝 진입 전 저장
stepRefs[i].data()   // 최종 제출 시 데이터 수집
```

### 스텝 완료 흐름

```
스텝 컴포넌트 @complete 이벤트 emit
  → onComplete(result, stepIndex)
  → currentStepIndex++
  → visibleSteps에 다음 스텝 추가
  → smooth scroll to 새 스텝
```

### 하단 버튼 구성

| 버튼 | 조건 | 동작 |
|------|------|------|
| 초기화 | newchange 도메인만 표시 | newchange 스토어 리셋 |
| 다음 | 마지막 스텝이 아닐 때 | 현재 스텝 유효성 검사 후 진행 |
| 작성완료 | 마지막 스텝 | 전체 데이터 제출 |

### 임시저장 복원

`window.history.state.code` — 임시저장 복원 시 코드 전달 방식 사용.

## 6. 동적 컴포넌트 로딩 (`comp.utils.js`)

### Form 컴포넌트 (Lazy Load)

```javascript
// /src/libs/utils/comp.utils.js
export const getFormComponent = (dir, file) =>
  markRaw(defineAsyncComponent({
    loader: () => import(`@/components/form/${dir}/${file}.vue`),
    loadingComponent: MsfLoadingComp,
    errorComponent: MsfErrorComp,
    delay: 200,
    timeout: 10000
  }))
```

### Extra 컴포넌트 (사전 스캔 방식)

```javascript
// Vite 빌드 시 import.meta.glob으로 전체 경로 사전 스캔
const modules = Object.fromEntries(
  Object.entries(import.meta.glob('@/components/extra/**/*.vue'))
    .map(([path, fn]) => [path.replace('/src', '@'), fn])
)

export const getExtraComponent = (pathes) =>
  markRaw(defineAsyncComponent({
    loader: modules[`@/components/extra/${pathes.join('/')}.vue`],
    ...
  }))
```

> Form은 runtime `import()`, Extra는 빌드타임 `import.meta.glob` 방식으로 차이가 있음.

## 13. SCSS 시스템

### 토큰 시스템 (`abstracts/_tokens.scss`)

KT 가이드 기반 Sass 토큰 맵 (`$tokens`) — CSS 변수로 자동 생성.

```scss
// 주요 색상 토큰
primary:   #191a1b  (기본 텍스트/UI)
accent1:   #e0282f  (KT 레드, 강조)
accent2:   #00a39b  (KT 그린, 보조 강조)
secondary: #e9eaee  (배경/비활성)
```

### 자동 주입

`vite.config.js`에서 모든 `.vue` 파일 SCSS에 자동 주입:
```javascript
// vite.config.js
css: {
  preprocessorOptions: {
    scss: {
      additionalData: `@use '@/assets/styles/abstracts' as *;`
    }
  }
}
```
→ 별도 import 없이 `@include flex()`, `$tokens` 등 abstracts 전체 사용 가능.

### 전역 스타일 (`global.scss`)

main.js에서 import — CSS Reset, 전역 기본 스타일, Tailwind 기본 레이어.

`unplugin-vue-components`가 자동 등록하는 경로:
- `src/components/` 하위 전체
- `src/libs/ui/` 하위 전체

이 경로 외부의 컴포넌트는 수동 import 필요.
