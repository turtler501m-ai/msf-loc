# getCombiSvcList / 아무나 SOLO 결합 가능여부 체크 로직 분석

**대상**: MCP 포탈 결합 메뉴 중 **getCombiSvcList**(결합 서비스 목록 조회) 및 **아무나 SOLO 결합 가능 여부** 판단 흐름.  
**관련 컨트롤러**: `mcp-portal-was` `CombineController.java`

---

## 1. API 구분

| API | 용도 | 비고 |
|-----|------|------|
| **getCombiSvcListAjax.do** | 결합 서비스 목록 조회 (X87) / 아무나 SOLO 화면 시 SOLO 여부·할인명 | `menuPara`에 따라 분기 |
| **checkCombineSelfAjax.do** | 아무나 SOLO 결합 **가입 가능 여부** 체크 (인증 후 1회) | combineSelf.js에서 호출 |
| **getCombineSoloTypeAjax.do** | 요금제 단위 아무나 SOLO 결합 가능 여부 (신청서 등 조회용) | rateCd만으로 결합 매핑 조회 |

## 2. getCombiSvcList (getCombiSvcListAjax.do) 전체 흐름

**진입**: `CombineController.getCombiSvcList(MyPageSearchDto searchVO, String menuPara)`  
**파라미터**: `searchVO.ncn`, `menuPara` (기본 "combine", 아무나 SOLO 전용 시 **"myCombineSelf"**)

### 2.1 공통 선행 처리

- 세션: `SessionUtils.getAuthSmsBean(menuPara)` → 모회선 정보(`pareSession`).
- `pareSession == null` 또는 `svcCntrNo` 없음 → **0001** 예외.
- `searchVO.getNcn() != pareSession.getSvcCntrNo()` → **0002** 예외.
- **요금제 결합 매핑**: `myCombinationSvc.selectMspCombRateMapp(pareSession.getRateCd())`.
  - **null** → `AdsvcNm = ""` (할인 부가서비스명 없음).
  - **있음** → `AdsvcNm = myCombination.getrRateNm()` (할인 요금제명).

### 2.2 분기 1: menuPara == "myCombineSelf" (아무나 SOLO 전용)

**목적**: 아무나 SOLO(마스터 결합 SOLO) **이미 가입 여부** 확인 및 화면용 플래그·할인명 설정.

| 순서 | 처리 | 설명 |
|------|------|------|
| 1 | 이용중 부가 조회 | `mPlatFormService.getAddSvcInfoDto(svcCntrNo, ctn, custId)` → **X20** 호출, 이용중 부가 목록 |
| 2 | PL249Q800 여부 | `addSvcInfoDto.getList()`에서 **soc == "PL249Q800"** 인 항목이 있으면 `isService = true` |
| 3 | PL249Q800 의미 | **PL249Q800** = "MVNO마스터결합전용 더미부가서비스(엠모바일)". 아무나 SOLO 가입 시 자동 부가되는 내부 관리용 SOC. |
| 4 | 할인명 처리 | **isService == false** 이면 `AdsvcNm = ""` 로 덮어씀. (이미 SOLO 부가가 없으면 할인명 미표시) |
| 5 | 응답 | `rtnMap.put("IS_SOLO_SERVICE", isService)` |

**정리 (myCombineSelf)**  
- **IS_SOLO_SERVICE = true**: 이용중 부가에 PL249Q800 있음 → **이미 아무나 SOLO(마스터 결합 SOLO) 가입 상태**.  
- **IS_SOLO_SERVICE = false**: PL249Q800 없음. 이때는 `AdsvcNm`을 비워서, 화면에서 “대상 요금제 아님” 또는 “미가입”으로 보이게 함.

### 2.3 분기 2: menuPara != "myCombineSelf" (일반 결합 목록)

- **X87** 결합 서비스 계약 조회: `mplatFormService.moscCombSvcInfoList(custId, svcCntrNo, ctn)`.
- 성공 시 결합 리스트에서 **본인 회선(item.svcNo == pareSession.ctn) 제외**, 모바일/인터넷 구분 마스킹 후 `RESULT_LIST` 로 반환.
- `CombTypeNm`, `CombProdNm`, `EngtPerdMonsNum`, `CombDcTypeNm`, `CombDcTypeDtl` 등 메타 반환.
- **결합 진행 현황**: `myCombinationSvc.selectMcpReqCombine(mSvcCntrNo)` 조회 후 리스트 반환.

## 3. 아무나 SOLO 결합 가능여부 체크 — checkCombineSelf (checkCombineSelfAjax.do)

**용도**: 아무나 SOLO 전용 화면(combineSelf)에서 인증 후 **“지금 이 회선으로 아무나 SOLO 결합 가입 가능한지”** 한 번에 판단.  
**호출**: `combineSelf.js` → `checkCombineSelf()` → `GET/POST /content/checkCombineSelfAjax.do` (menu, ncn 등).

### 3.1 처리 순서

| 순서 | 체크 항목 | 불가 시 |
|------|-----------|---------|
| 1 | 세션 존재, `searchVO.ncn == rtnDto.svcCntrNo` | 0001, 0002 예외 |
| 2 | certService.vdlCertInfo("combineSelfAut") | 실패 시 RESULT_CODE "-1", RESULT_DESC |
| 3 | MasterCombineLineInfo 코드 존재 | 없으면 **0002** "현재 서비스 이용이 불가합니다." |
| 4 | **이용중 부가 (X20)** | getAddSvcInfoDto 호출 |
| 5 | **PL249Q800 존재 여부** | list에 **soc == "PL249Q800"** 있으면 **0001** "이미 신청 이력이 존재합니다." → **가입 불가** |
| 6 | 주민번호 형식 (숫자 7자리 이상) | **10008** "주민번호 형식이 일치하지 않습니다." |
| 7 | 회선 상태 subStatus == "A" (사용중) | **00003** "현재 회선을 사용 중인 고객만 결합이 가능합니다." |
| 8 | **요금제 결합 매핑** | selectMspCombRateMapp(rateCd) **null** → **0004** "해당 상품은 결합이 불가합니다." |
| 9 | | **rRateCd == "EMPTY"** → **0004** "해당 상품은 결합이 불가합니다.(EMPTY)" |
| 10 | **X87 결합 서비스 조회** | moscCombSvcInfoList(custId, svcCntrNo, ctn) → 결합 리스트 있으면 `isCombSvc = true` |
| 11 | 성공 저장 | certService.vdlCertInfo("C", "combineSelfCheck", ...) |
| 12 | 성공 응답 | RESULT_CODE=AJAX_SUCCESS, **IS_COMBIN**=isCombSvc, **RESULT_SERVICE_NM**=myCombination.getrRateNm(), RESULT_SERVICE_CD, RESULT_COMBINE_LIST |

### 3.2 “가능” 조건 요약

- **이미 신청 이력 없음**: 이용중 부가(X20)에 **PL249Q800이 없음**.
- **요금제가 결합 대상**: `selectMspCombRateMapp(rateCd)` 결과가 존재하고, `rRateCd`가 "EMPTY"가 아님.
- 회선 사용중(subStatus=A), 주민번호 형식 정상, MasterCombineLineInfo 코드 존재, 인증(combineSelfAut) 통과.

### 3.3 IS_COMBIN / RESULT_SERVICE_NM

- **IS_COMBIN**: X87 결합 리스트가 1건 이상이면 true. “이미 아무나 결합/가족결합 등 다른 결합에 가입된 회선”이면 true → 화면에서 “결합은 가능하나 결합 혜택이 추가로 제공되지 않습니다” 등 중복 혜택 안내.
- **RESULT_SERVICE_NM**: 요금제 결합 매핑에서 오는 할인 요금제명. 가입 시 제공될 혜택명으로 화면에 표시.

## 4. getCombineSoloType (getCombineSoloTypeAjax.do)

**용도**: 신청서 등에서 **요금제(rateCd)** 만으로 아무나 SOLO 결합 가능 여부를 물을 때.

- `myCombinationSvc.selectMspCombRateMapp(rateCd)` 만 수행.
- **null** → RESULT_CODE **00001** "해당 상품은 결합이 불가합니다."
- **rRateCd == "EMPTY"** → RESULT_CODE **00002** "해당 상품은 결합이 불가합니다.(EMPTY)"
- 성공 시 RESULT_CODE=AJAX_SUCCESS, **R_RATE_NM**=할인 요금제명.

## 5. 연동·데이터 정리

| 구분 | API/DB | 역할 |
|------|--------|------|
| M플랫폼 | **X20** (getAddSvcInfoDto) | 이용중 부가서비스 목록 조회 → **PL249Q800** 존재 여부로 “이미 아무나 SOLO 가입” 판단 |
| msp | selectMspCombRateMapp(rateCd) | 요금제별 결합(할인) 매핑 조회 → 결합 가능 요금제 여부·할인 요금제명(rRateNm) |
| M플랫폼 | **X87** (moscCombSvcInfoList) | 결합 서비스 계약 조회 → 기존 결합 유무(IS_COMBIN), 결합 목록 |
| DB | MasterCombineLineInfo 코드 | 아무나 SOLO(마스터 결합) 서비스 오픈 여부 |

## 6. PL249Q800 정리

- **SOC 코드**: PL249Q800  
- **의미**: MVNO 마스터 결합 전용 더미 부가(엠모바일). 아무나 SOLO 가입 시 자동 부가되는 내부 관리용.
- **역할**  
  - **있음** → 이미 아무나 SOLO 가입 완료 → checkCombineSelf에서 **0001** “이미 신청 이력이 존재합니다.” 반환, 가입 불가.  
  - getCombiSvcList(myCombineSelf)에서는 **IS_SOLO_SERVICE = true** 로 “이미 SOLO 사용 중” 표시.

---

## 7. 요약

- **getCombiSvcList**:  
  - **myCombineSelf**일 때: X20으로 PL249Q800 여부 확인 → **IS_SOLO_SERVICE** 설정, PL249Q800 없으면 AdsvcNm 비움.  
  - 그 외: X87로 결합 목록·진행 현황 조회.
- **아무나 SOLO 결합 가능 여부(가입 가능)** 는 **checkCombineSelf**에서 판단:  
  - **PL249Q800 없음** + **요금제 결합 매핑 있음** + 회선 사용중·인증·기타 조건 통과 시 가입 가능.  
  - X87로 **IS_COMBIN**을 내려서, 이미 다른 결합에 가입된 경우 “혜택 중복 불가” 안내에 사용.
