# MCP 폴더 구조 분석

## 1. 전체 개요

MCP(Mobile Customer Portal) 프로젝트는 **멀티 모듈** 구조로, 포털·관리자·API·배치·프록시 등 기능별로 분리된 7개 하위 프로젝트로 구성됩니다.

```
mcp/
├── mcp-portal-was      # 포털 백엔드 (Spring MVC, WAR)
├── mcp-portal-web      # 포털 프론트엔드 (정적 MPA, Webpack)
├── mcp-admin-was       # 관리자 백엔드 (Spring MVC, WAR)
├── mcp-admin-web       # 관리자 프론트엔드 (정적 MPA)
├── mcp-api             # REST API 서버 (Spring MVC, WAR)
├── msc-batch           # 배치 작업 (Spring Boot 2.6, WAR)
└── msc-prx             # 프록시/인증 서버 (Spring 3.2, Java 6, WAR)
```

---

## 2. 모듈별 상세

### 2.1 mcp-portal-was (포털 백엔드)

| 항목 | 내용 |
|------|------|
| **역할** | 모바일/웹 포털 서버 (고객용) |
| **빌드** | Maven, WAR 패키징 |
| **기술** | Spring 4.3, Java 11, MyBatis |
| **groupId** | com.ktmmobile.mcp |

**디렉토리 구조**

```
mcp-portal-was/
├── devops/jenkins/           # Jenkins 파이프라인 (mcp-portal, mcp-widget)
├── lib/                      # 로컬 JAR (CNP, crypto, ipin, NiceID 등)
├── src/main/
│   ├── java/com/ktmmobile/mcp/
│   │   ├── app/              # 앱 컨트롤러
│   │   ├── appform/          # 서식지
│   │   ├── common/           # mplatform, mspservice, cti, util
│   │   ├── content/          # 결합(MyCombination), 데이터쉐어링
│   │   ├── cs/               # 고객상담, 공지, 문의
│   │   ├── join/             # 가입
│   │   ├── mypage/           # 마이페이지, 요금조회, 결제수단
│   │   ├── order/            # 주문
│   │   ├── phone/            # 폰 프론트/모바일
│   │   ├── usim/             # 유심
│   │   └── xxxx_*/           # wire, search, event, etc, ktPolicy 등
│   ├── resources/
│   │   ├── mappers/          # MyBatis XML
│   │   ├── context/          # Spring 설정
│   │   └── configuration/properties/
│   └── webapp/
│       ├── META-INF/
│       └── WEB-INF/views/portal/
├── src/main/resources-{dev|local|prd|stg}/  # 환경별 설정
└── pom.xml
```

**주요 패키지**
- `common.mplatform` : KT M플랫폼 API 연동 (X78/X79 등)
- `content.controller.MyCombinationController` : 아무나 가족결합+
- `mypage.controller` : 요금조회, 결제수단, 마이올레 등

## 3. 공통 구조

### 3.1 환경별 설정

대부분 모듈에서 다음 프로파일을 사용합니다.

| 프로파일 | 용도 |
|----------|------|
| resources-dev | 개발 |
| resources-local | 로컬 |
| resources-stg | 스테이징 |
| resources-prd | 운영 |

### 3.2 Jenkins 배포

- `devops/jenkins/{dev|stg|prd}/` 에 Jenkinsfile 또는 파이프라인 정의
- Nexus: `http://211.253.137.5:8081/nexus/repository/maven-public/`

### 3.3 Java 패키지 규칙

- **기본 패키지**: `com.ktmmobile.mcp`
- **xxxx_* / xxxxx_***: 예약어·키워드 회피용 폴더명 (실제 패키지는 `requestReview`, `wire`, `member` 등)
- **레이어**: `controller`, `service`, `dao`, `dto`, `mapper`
