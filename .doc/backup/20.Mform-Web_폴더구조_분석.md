# mform-web 폴더 구조 분석

## 1. 개요

**mform-web**은 MCP 서식지(모바일 신청/변경/해지 등) 전용 **Vue 3 SPA** 프로젝트입니다.  
프로젝트명은 `mcp-mform-web`이며, Vite 7 + Vue 3 + Pinia + Tailwind CSS 4 기반으로 구성됩니다.

| 항목 | 내용 |
|------|------|
| **역할** | 서식지 웹 앱 (신규/변경/명의변경/해지/부가기능) |
| **빌드** | Vite 7 |
| **기술** | Vue 3.5, Vue Router 5, Pinia 3, Tailwind CSS 4 |
| **Node** | ^20.19.0 \|\| >=22.12.0 |

## 4. 라우팅

| 경로 | 이름 | 설명 |
|------|------|------|
| `/` | mcp-main | McpMainLayout (자식: home, service-detail) |
| `` | home | McpHomeView (홈) |
| `mobile/:domain/:service` | service-detail | McpServiceDetailView (동적 서비스) |

- **domain**: `regist`, `change`, `ident`, `cancel`, `addition`
- **service**: 해당 도메인 내 컴포넌트명 (예: `RegistType`, `RegistCust`)

동적 컴포넌트는 `utils/mcp_components.js`의 `getRoutingComponent` / `getStepComponent`로  
`@/components/{domain}/{service}.vue`를 동적 import 합니다.

## 5. 메뉴/스텝 구조 (mcp_menu.js)

| id | 제목 | 도메인 | 스텝 |
|----|------|--------|------|
| 01 | 신규/변경 | regist | 가입유형 → 고객 → 상품 → 동의 |
| 02 | 서비스 변경 | change | 가입유형 → 고객 → 상품 → 동의 |
| 03 | 명의변경 | ident | 고객 → 상품 → 동의 |
| 04 | 서비스 해지 | cancel | 고객 → 상품 → 동의 |
| 05 | 부가기능 | addition | 임시저장, 접수완료, 간편신청서, 로그인설정 |

## 6. 주요 의존성

| 구분 | 패키지 |
|------|--------|
| **프레임워크** | vue, vue-router, pinia |
| **스타일** | tailwindcss, @tailwindcss/vite, tw-animate-css |
| **UI/아이콘** | reka-ui, lucide-vue-next, @fortawesome/* |
| **유틸** | @vueuse/core, class-variance-authority, clsx, tailwind-merge |
| **빌드/도구** | vite, @vitejs/plugin-vue, unplugin-vue-components |

- **unplugin-vue-components**: `src/components/ui` 자동 임포트 (디렉티브 사용 가능)
