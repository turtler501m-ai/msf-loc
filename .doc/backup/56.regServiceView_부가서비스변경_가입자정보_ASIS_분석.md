# regServiceView.do ASIS 부가서비스 변경 – 가입자정보 변동 처리 분석

**목적**: 부가서비스 신청/해지 화면(regServiceView.do)에서 **가입자정보(가입자 정보 변동)** 를 어떻게 확보·사용하는지 화면부터 백엔드까지 분석  
**참조**: 54.가입자정보조회_X01_Y04_ASIS_로직분석.md, MypageServiceImpl.checkUserType, RegSvcController

## 2. 서버 진입 – 가입자정보가 채워지는 흐름

### 2.1 최초 화면 로딩: regServiceView.do GET

**Controller**: `RegSvcController.regServiceView()`

```
1. UserSessionDto userSession = SessionUtils.getUserCookieBean();
2. cntrList = mypageService.selectCntrList(userSession.getUserId());
3. checkUserType(searchVO, cntrList, userSession);
4. model.addAttribute("cntrList", cntrList);
   model.addAttribute("searchVO", searchVO);
5. return "/portal/mypage/regServiceView" (또는 mobile 경로);
```

- **cntrList**: 로그인 사용자 소유 **회선 목록**. mcp-api `/mypage/...` 호출로 조회(예: MCP_USER_CNTR_MNG 기반). 각 항목에 `svcCntrNo`, `cntrMobileNo`, `custId`, `contractNum` 등 포함.
- **searchVO**: 최초 요청 시 비어 있을 수 있음. **checkUserType** 내부에서 기본값/선택 회선으로 세팅.

### 2.2 checkUserType – 가입자정보( ncn / ctn / custId ) 세팅

**파일**: `MypageServiceImpl.checkUserType(MyPageSearchDto searchVO, List<McpUserCntrMngDto> cntrList, UserSessionDto userSession)`

역할: **요청으로 넘어온 ncn(선택 회선)** 과 **cntrList** 를 매칭해, 해당 회선의 **ncn, ctn, custId, contractNum** 등을 searchVO에 넣는다.

1. **정회원·회선 보유 검사**  
   - `userSession.getUserDivision()` 이 "01"이 아니거나, cntrList가 null/empty면 false 반환.

2. **선택 회선 결정**  
   - `paramNcn` = `searchVO.getNcn()` (요청 파라미터, Ajax에서는 선택한 svcCntrNo)  
   - `sesNcn` = `SessionUtils.getCurrPhoneNcn()` (세션에 저장된 현재 회선)  
   - `selNcn` = paramNcn이 있으면 paramNcn, 없으면 sesNcn.

3. **selNcn이 비어 있는 경우**  
   - 첫 번째 회선을 기본 선택:  
     `searchVO.setNcn(cntrList.get(0).getSvcCntrNo())`,  
     `setCtn(cntrList.get(0).getCntrMobileNo())`,  
     `setCustId(cntrList.get(0).getCustId())`,  
     `setContractNum(cntrList.get(0).getContractNum())`,  
     `setSoc(...)`  
   - `SessionUtils.setCurrPhoneNcn(cntrList.get(0).getSvcCntrNo())` 로 세션 갱신.

4. **selNcn이 있는 경우**  
   - `cntrList`를 순회하며 `mcpUserCntrMngDto.getSvcCntrNo()` 와 `selNcn` 이 같으면  
     `searchVO.setNcn(ncn)`, `setCtn(ctn)`, `setCustId(custId)`, `setContractNum(contractNum)`, `setSoc(soc)`  
   - `SessionUtils.setCurrPhoneNcn(ncn)` 로 세션 갱신.

**정리**:  
- **가입자정보 “변동”** = 사용자가 드롭다운에서 **다른 회선을 선택**할 때마다, 클라이언트는 **ncn(= svcCntrNo)** 만 보내고,  
- 서버는 **이미 보유한 cntrList**에서 해당 회선을 찾아 **searchVO에 ncn, ctn, custId, contractNum**을 채운다.  
- 이 화면에서는 **X01 가입정보조회나 join-info(이름+ctn→ncn/custId)를 호출하지 않는다.**

## 3. 회선 목록(cntrList)의 가입자정보 출처

- **selectCntrList(userId)**  
  - MCP 포털-WAS에서는 **mcp-api** 로 회선 목록 API 호출 (예: `apiInterfaceServer + "/mypage/..."` 에 userId, customerId 등 전달).  
  - mcp-api에서 **MCP_USER_CNTR_MNG** 등에 저장된 회선을 조회해, 회선별 **SVC_CNTR_NO(svcCntrNo), CNTR_MOBILE_NO(ctn), CUSTOMER_ID(custId), CONTRACT_NUM(contractNum)** 등을 반환.

- **회선이 저장되는 시점**  
  - **회선 추가** (예: `/mypage/userMultiReg.do` 등):  
    - 이름·휴대폰 입력 → **휴대폰 인증** (SMS 등)  
    - **M전산 조회**: `mypageUserService.selectContractNum(name, phone)` → 계약번호(contractNum) 획득  
    - 이후 회선 등록 API 호출 시 해당 회선의 **contractNum, 휴대폰번호, custId** 등이 mcp-api를 통해 MCP_USER_CNTR_MNG(또는 동일 역할 테이블)에 저장됨.  
  - 즉, **가입자정보( ncn, ctn, custId )는 “회선 추가” 시점에 한 번 확보·저장**되고, regServiceView에서는 **그 저장된 값을 회선 선택만으로 재사용**한다.

## 5. 요약 – 가입자정보 변동 처리

| 구분 | 내용 |
|------|------|
| **화면** | regServiceView.jsp + myCommCtn.jsp. 회선 선택 select는 **svcCntrNo**를 value로 사용. |
| **클라이언트** | 조회/신청/해지 시 **ncn**(= 선택 option의 value, 즉 svcCntrNo)만 전송. ctn, custId는 전송하지 않음. |
| **서버** | **checkUserType(searchVO, cntrList, userSession)** 에서 요청의 ncn(또는 세션 ncn)과 **cntrList**를 매칭해, 해당 회선의 **ncn, ctn, custId, contractNum**을 searchVO에 세팅. |
| **cntrList** | 로그인 사용자의 **등록 회선 목록**(mcp-api, MCP_USER_CNTR_MNG 등). 회선 추가 시 휴대폰 인증·M전산 조회로 확보한 contractNum·custId·휴대폰번호가 저장된 값. |
| **X01/join-info** | regServiceView **화면·Ajax 경로에서는 호출하지 않음**. 가입자정보는 “회선 추가” 시점에 한 번 확보·저장하고, 부가서비스 변경 시에는 **저장된 회선 정보를 선택만으로 사용**. |

즉, **regServiceView.do ASIS 부가서비스 변경**에서의 가입자정보 변동은  
- **“회선 선택” 변경 시**  
- **이미 등록된 회선 목록(cntrList)** 에서 선택한 회선의 **ncn, ctn, custId**를 **checkUserType**으로 searchVO에 채워 넣고,  
- 그 searchVO로 X97/X20/X21/X38을 호출하는 구조이다.  
화면에서는 **회선 선택 드롭다운**만 바꾸면 되고, 서버는 **저장된 가입자정보를 조회해 재사용**한다.
