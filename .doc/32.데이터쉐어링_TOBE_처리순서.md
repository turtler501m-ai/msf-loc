## B. 데이터쉐어링 신규 개통 — 미구현

> ASIS: AppformController `saveDataSharingSimple(4001-03)` + `saveDataSharing(4001-07)` + `saveDataSharingStep1~3(4001-08,10,11)` + `conPreCheck(4001-09)` + `sendScan(4001-04)`
> TOBE: 단계별 API로 분리 구현

### 전체 흐름

```
[Step0] 신청서 저장 : POST /apply (4001-03 / 4001-07)
[Step1] PC0 사전체크 : POST /apply/step1 (4001-08)
[Step2] PC2 폴링 + Y39 : POST /apply/step2 (4001-09)
[Step3] NU1/NU2 번호조회/예약 : POST /apply/step3 (4001-10)
[Step4] OP0 개통 + 수납 : POST /apply/step4 (4001-11)
[SCAN]  SCAN 서버 전송 : POST /apply/scan (4001-04)
```

ㅇ 신청서 저장 — POST /data-sharing/apply
1) 입력값 검증 - contractNum(NCN), reqUsimSn 필수
2) 가입자 조회 - selectCntrListNoLogin (비회원만)
3) 청구계정번호 조회 - selectBanSel
4) 기존 신청서 조회 - getCopyMcpRequest → MSF_REQUEST, CSTMR, REQ, SALEINFO, AGENT
5) 공통코드 조회 - getCodeNmDto (공통모듈)
6) 유심비 조회 - selectJoinUsimPriceNew
7) 대리점코드 조회 - getAgentCode
8) 프로모션ID 조회 - getChrgPrmtId
9) MSP 상품코드 조회 - getMspPrdtCode
10) USIM 그룹코드 조회 - getCodeList(USIM_PROD_DTL_ID_GROP_CODE)
11) 인증 STEP 검증 - vdlCertInfo → MSF 인증로그 적재로 대체 (본인인증 제외)
12) 신청서 저장 - generateRequestKey/ResNo(MSF 시퀀스), saveAppform, updateFathMcpRequestOsst → MSF_REQUEST_OSST
13) SMS 발송 - sendKakaoNoti (4001-07 전용, onOffType=0,3, 현행 ITO API 그대로)
14) 응답 - { resNo, requestKey }


### [Step0] 신청서 저장 — POST /api/v1/service-change/data-sharing/apply

**ASIS**: `saveDataSharingSimple(4001-03)`, `saveDataSharing(4001-07)`

```
[1] 입력값 검증

[2] 가입자 조회 (비회원)
    - mypageService.selectCntrListNoLogin
      [TOBE] M포탈/전산 테이블링크 사용 → MSF 동일하게 적용
    ※ 제외: selectCntrList (회원 로직 불필요)

[3] 청구계정번호 조회
    - mypageService.selectBanSel
      [TOBE] M포탈/전산 API 그대로 사용 → /mypage/selectBanSel

[5] 공통코드 조회
    - NmcpServiceUtils.getCodeNmDto
      [TOBE] MSF 공통모듈 적용 (담당: 홍성민C)

[6] 유심비 조회
    - usimService.selectJoinUsimPriceNew
      [TOBE] M포탈/전산 테이블링크 사용 → /storeUsim/joinUsimPriceNew

[7] 대리점코드 조회
    - appformSvc.getAgentCode → /appform/agentCode
      [TOBE] M포탈/전산 테이블링크 사용

[8] 프로모션ID 조회
    - appformSvc.getChrgPrmtId → /appform/getChrgPrmtId
      [TOBE] M포탈/전산 테이블링크 사용

[9] MSP 상품코드 조회
    - appformDao.getMspPrdtCode → /appform/mspPrdtCode
      [TOBE] M포탈/전산 테이블링크 사용

[10] USIM 상품 그룹코드 조회
    - NmcpServiceUtils.getCodeList(USIM_PROD_DTL_ID_GROP_CODE)
      [TOBE] MSF 공통모듈 적용

[11] 인증 STEP 검증
    - certService.getModuTypeStepCnt → NMCP_CRT_VLD_DTL
      [TOBE] 본인인증 불필요 → MSF 인증 로그 적재로 대체
    - certService.vdlCertInfo
      [TOBE] 인증로그 공통모듈로 공통화
    ※ 제외: certService.getCertInfo (본인인증)

[12] 신청서 저장 (saveAppform)
    - AppformMapper.generateRequestKey → SQ_RCP_REQUEST_KEY_01
      [TOBE] SQL Copy → MSF 시퀀스로 변경
    - AppformMapper.generateResNo → SQ_RCP_RES_NO_01
      [TOBE] SQL Copy → MSF 시퀀스 통합 (임시번호 미사용, 실제 신청서번호 동일 사용)
    - crtSaveAppFormStep (최종 step 확인)
      [TOBE] M플랫폼 전송 API 호출로 대체
    - fathService.validateFathSession → MSF 적용
    - fathService.updateFathMcpRequestOsst → FathMapper.updateFathMcpRequestOsst
      [TOBE] SQL Copy → MCP_REQUEST_OSST → MSF_REQUEST_OSST
    - appformSvc.saveAppform 전체
      [TOBE] M플랫폼 전송 API 호출로 대체
    ※ 제외: 배송(insertRequestSelfDlvry), 선불, 사은품, 자급제, 상담사, 결제, 포인트, 추천인

    { "resNo": "", "requestKey": "" }
```

ㅇ 신청서 저장 — POST /apply [Step0] 저장
- ASIS saveDataSharingSimple(4001-03) / saveDataSharing(4001-07)
1. 입력값 검증 - contractNum(NCN), reqUsimSn 필수

3) PC0 사전체크 요청

  - sendOsstSt1Service → MSF_REQUEST_OSST UPDATE

  - MoscSvcContService(osstOrdNo) [M플랫폼] Y39

ㅇ 번호조회/예약 — POST /apply/step3 [Step3] NU1/NU2

### [Step1] PC0 사전체크 — POST /api/v1/service-change/data-sharing/apply/step1

**ASIS**: `saveDataSharingStep1(4001-08)`

```
[1] fnSetDataOfdataSharing 재호출
    → rtnAppformReqDto, certContractNum, customerId 획득

[2] 중복 PC0 방어로직
    - appformSvc.requestOsstCount({mvnoOrdNo=resNo, prgrStatCd=PC2}) > 0
      → 이미 PC2 성공 → 즉시 SUCCESS 반환
    - appformSvc.requestOsstCount({prgrStatCd=PC0, rsltCd=ITL_SST_E0002}) > 0
      → MVNO 접수번호 존재 → 즉시 SUCCESS 반환

[3] PC0 사전체크 요청 (Thread.sleep 3초 후)
    - appformSvc.sendOsstService(
        {resNo, prntsContractNo=certContractNum, custNo=customerId},
        EVENT_CODE_PRE_CHECK=PC0
      )
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용

[4] 응답
    성공: { RESULT_CODE: "SUCCESS", RES_NO, REQUEST_KEY }
    실패: { RESULT_CODE: 1001~1005, EVENT_CODE, ERROR_MSG }
```

### [Step2] PC2 폴링 + ST1 + Y39 — POST /api/v1/service-change/data-sharing/apply/step2

**ASIS**: `conPreCheck(4001-09)`

```
[1] PC2 결과 폴링
    - appformSvc.getRequestOsst
      [TOBE] SQL Copy → MCP_REQUEST_OSST → MSF_REQUEST_OSST
      조건: mvnoOrdNo=resNo, prgrStatCd=PC2(EVENT_CODE_PC_RESULT)
      5초 × 50회 반복 (최대 250초)
      성공(rsltCd=0000): 다음 진행
      실패: { RESULT_CODE: 2001 }
      타임아웃: { RESULT_CODE: 2002 }

[2] PC2 완료상태 확인
    - appformSvc.chkRealPc2Result(resNo, contractNum)
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용
      미완료 시: { RESULT_CODE: PRE_SCH_ERR, EVENT_CODE_PRE_SCH }

[3] ST1 (사전체크 작업 완료 상태 조회)
    - sendOsstSt1Service [M플랫폼] ST1
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용

[4] Y39 — 아이핀 CI 조회
    - mPlatFormService.MoscSvcContService(osstOrdNo) [M플랫폼] Y39
      [TOBE] M포탈/M전산 소스 로직 MSF에 적용
      LOCAL/DEV/STG: NICE 본인인증 CI로 대체

[5] CI STEP 검증
    - certService.vdlCertInfo("D", [urlType=chkPreOpenCi, stepEndYn=Y, connInfo=CI])
      [TOBE] MSF 인증로그 공통모듈 적용
      실패: { RESULT_CODE: STEP03 }

[6] 응답
    성공: { RESULT_CODE: "SUCCESS" }
```
