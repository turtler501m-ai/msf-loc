# MSF 백엔드 패키지 구조 변경 계획

> 작성일: 2026-04-02
> 대상: `msf/msf-api` 백엔드 패키지 재구성 (ASIS mcp → TOBE msf)

---

## 1. 패키지 매핑표

### 시스템 공통

| ASIS (mcp) | TOBE (msf) | 설명 |
|------------|------------|------|
| `mcp/app` | `msf/login` | 로그인 / 앱 진입 |
| `mcp/cs` | `msf/main` | 메인 화면 / CS |
| `mcp/common` | `msf/common` | 시스템 공통 |
| `mcp/fath` | `msf/fath` | FATH 연동 |
| `mcp/cert` | `msf/cert` | 인증 |

### 업무 서식 (form)

| ASIS (mcp) | TOBE (msf) | 설명 |
|------------|------------|------|
| `mcp/order` | `msf/form/common` | 업무공통 통합 |
| `mcp/phone` | `msf/form/common` | 업무공통 통합 |
| `mcp/usim` | `msf/form/common` | 업무공통 통합 |
| `mcp/appform` | `msf/form/newchange` | 신규가입 변경 |
| `mcp/content` | `msf/form/servicechange` | 서비스 변경 |
| `mcp/mypage` | `msf/form/servicechange` | 서비스 변경 (마이페이지 일부) |
| `mcp/mypage` | `msf/form/ownerchange` | 명의 변경 (마이페이지 일부) |
| `mcp/mypage` | `msf/form/termination` | 서비스 해지 (마이페이지 일부) |

## 2. 레이어 구조 규칙

### 2-1. 네이밍 — `Sf` 접두사

Controller와 Service 클래스명 앞에 **`Sf`** 접두사를 붙인다.

```
SfXxxController.java
SfXxxService.java
SfXxxServiceImpl.java
```

### 2-2. Mapper 위치

| 분류 | 패키지 위치 |
|------|------------|
| 업무공통 Mapper | `form/comm/mapper/` |
| 각 업무 도메인 Mapper | `form/{업무}/mapper/` |
| 시스템공통 Mapper | `common/mapper/` |

### 3-2. MyBatis XML 매퍼 (`src/main/resources/mapper/`)

ASIS `.doc/asis/mcp-api-mapper/`, `.doc/asis/mcp-web-mappers/` 참조하여 각 업무 하위로 이관.

```
resources/mapper/
│
├── login/                        ← LoginMapper.xml, AppMapper.xml 등
├── main/                         ← MainMapper.xml 등
├── common/                       ← CommonMapper.xml, CommCodeMapper.xml 등
├── fath/                         ← FathMapper.xml 등
├── cert/                         ← CertMapper.xml 등
│
└── form/
    ├── comm/                     ← OrderMapper.xml, PhoneMapper.xml, UsimMapper.xml 등
    ├── newChg/                   ← AppFormMapper.xml, AppformMapper.xml 등
    ├── svcChg/                   ← RateMapper.xml, RegSvcMapper.xml 등
    ├── ownChg/                   ← MyNameChgMapper.xml, MypageMapper.xml 일부
    └── svcCncl/                  ← CancelConsultMapper.xml 등
```

## 4. @MapperScan 등록 목록

신규 패키지 추가 시 `MsfApplication.java`의 `@MapperScan`에 반드시 등록.

```java
@MapperScan({
    "com.ktmmobile.msf.common.mapper",
    "com.ktmmobile.msf.form.comm.mapper",
    "com.ktmmobile.msf.form.newChg.mapper",
    "com.ktmmobile.msf.form.svcChg.mapper",
    "com.ktmmobile.msf.form.ownChg.mapper",
    "com.ktmmobile.msf.form.svcCncl.mapper"
})
```

> 누락 시 `NoSuchBeanDefinitionException` 발생으로 서버 기동 실패

## 5. 주의사항

| 항목 | 내용 |
|------|------|
| 클래스명 충돌 | `form/comm` 통합 시 order·phone·usim 간 중복 클래스명 사전 확인 필요 |
| mypage 분리 기준 | 기능 성격(변경/명의/해지)에 따라 소속 패키지 명확히 결정 후 이관 |
| 의존성 방향 | `form/{업무}` → `form/comm` 단방향 허용 / 업무 패키지 간 직접 참조 금지 |
| HTTP 메서드 | GET(단순조회) / POST(나머지) 두 가지만 사용 — PUT·PATCH·DELETE 금지 |
